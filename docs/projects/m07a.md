## What You'll Build

A character-level GPT trained on Shakespeare's complete works (~1.1M characters). You'll implement a custom character tokenizer (no BPE, no HuggingFace), a 4-layer causal transformer decoder in raw PyTorch, an AdamW training loop with cosine LR decay and gradient clipping, and a text generation function with temperature scaling and top-k sampling. Target: validation loss below 1.5 nats, generated output recognizably Shakespearean in syntax and rhythm.

## The Mental Model

A language model learns $P(x_t \mid x_1, \ldots, x_{t-1})$ — the probability of the next token given all prior tokens. At character level, the vocabulary is tiny (~65 symbols for Shakespeare) but sequences are long, so the model must learn long-range dependencies to produce coherent verse. Self-attention solves this: every token attends to every prior token in a single layer, with $O(1)$ path length between any two positions. The causal mask enforces the autoregressive constraint — position $t$ can see positions $1 \ldots t$, never $t+1$ onward.

The core equation: $\text{Attn}(Q, K, V) = \text{softmax}\!\left(\dfrac{QK^T}{\sqrt{d_k}} + M\right)V$, where $M$ is $-\infty$ in upper-triangle entries (future tokens) and $0$ elsewhere. Multi-head attention runs $h$ parallel attention heads, concatenates outputs, and projects back to $d_\text{model}$. The FFN after each attention block acts as per-token memory: it stores and retrieves patterns that attention cannot (attention mixes tokens; FFN transforms each independently).

Training minimizes cross-entropy loss: $\mathcal{L} = -\sum_t \log P(x_t \mid x_{<t})$. Temperature at generation time divides the logits before softmax, sharpening ($T < 1$) or flattening ($T > 1$) the distribution. Top-k restricts sampling to the $k$ most probable tokens, eliminating long-tail garbage.

## Phase 1 — Data Loading and Character Tokenizer

**Why this first:** The tokenizer defines the vocabulary size, which sets the shape of every tensor in the model. Fix this contract before writing any network code.

```python
# ── Tokenizer ─────────────────────────────────────────────────────────────────

def build_tokenizer(text: str) -> tuple[dict[str, int], dict[int, str]]:
    """
    Character-level tokenizer: one token per unique character.
    Returns (char2idx, idx2char) dictionaries.
    Vocabulary is sorted for reproducibility.
    """
    chars = sorted(set(text))
    char2idx = {ch: i for i, ch in enumerate(chars)}
    idx2char = {i: ch for ch, i in char2idx.items()}
    return char2idx, idx2char


def encode(text: str, char2idx: dict[str, int]) -> list[int]:
    return [char2idx[ch] for ch in text]


def decode(indices: list[int], idx2char: dict[int, str]) -> str:
    return "".join(idx2char[i] for i in indices)


def load_shakespeare(data_path: str = "shakespeare.txt") -> str:
    """Download Shakespeare corpus if not present. ~1.1M chars."""
    p = Path(data_path)

        print(f"Downloading Shakespeare corpus from {url} ...")
        urllib.request.urlretrieve(url, data_path)
        print(f"Saved to {data_path}")
    return p.read_text(encoding="utf-8")
```

## Phase 2 — Dataset with Context Windows

**Why this second:** PyTorch's `DataLoader` needs a `Dataset` that serves `(input, target)` pairs. For language modeling the target is the input shifted right by one — predicting the next character at every position simultaneously.

```python
# ── Dataset ───────────────────────────────────────────────────────────────────

class CharDataset(torch.utils.data.Dataset):
    """
    Returns overlapping windows of length block_size.
    Input:  tokens[i : i+block_size]
    Target: tokens[i+1 : i+block_size+1]  (shifted by one)
    Training sees every subsequence: parallel next-token prediction.
    """

    def __init__(self, data: torch.Tensor, block_size: int) -> None:
        self.data = data
        self.block_size = block_size

    def __len__(self) -> int:
        # Last valid start index: len(data) - block_size - 1
        return len(self.data) - self.block_size

    def __getitem__(self, idx: int) -> tuple[torch.Tensor, torch.Tensor]:
        x = self.data[idx : idx + self.block_size]
        y = self.data[idx + 1 : idx + self.block_size + 1]
        return x, y


# ── Hyperparameters ───────────────────────────────────────────────────────────

class Config:

    block_size:  int   = 256    # context length (tokens)
    vocab_size:  int   = 65     # set after tokenizer build

    d_model:     int   = 128    # embedding dimension
    d_ff:        int   = 512    # FFN hidden dim = 4 * d_model

    batch_size:  int   = 64
    max_steps:   int   = 5000
    lr:          float = 3e-4
    lr_min:      float = 3e-5
    warmup_steps:int   = 100
    grad_clip:   float = 1.0
    eval_interval: int = 200
    eval_iters:  int   = 50
    device: str = "cuda" if torch.cuda.is_available() else (
        "mps" if torch.backends.mps.is_available() else "cpu"
    )
```

## Phase 3 — Transformer Block (Causal MHA + FFN + LayerNorm)

**Why this third:** The transformer block is the core computation unit. Build and test it in isolation before assembling the full model.

```python
# ── Transformer components ────────────────────────────────────────────────────

class CausalSelfAttention(nn.Module):
    """
    Multi-head causal self-attention.
    
    For each head h:
        Q_h, K_h, V_h = Linear projections of input X    shape: (B, T, d_k)
        scores = Q_h K_h^T / sqrt(d_k)                  shape: (B, T, T)
        scores[i, j] = -inf if j > i   (causal mask)
        attn = softmax(scores)                           shape: (B, T, T)
        out_h = attn @ V_h                               shape: (B, T, d_k)
    Concatenate all heads → project to d_model.
    """

    def __init__(self, d_model: int, n_heads: int, dropout: float) -> None:
        super().__init__()
        assert d_model % n_heads == 0, "d_model must be divisible by n_heads"
        self.n_heads = n_heads
        self.d_k = d_model // n_heads

        # Fused QKV projection — one matrix multiply instead of three
        self.qkv_proj = nn.Linear(d_model, 3 * d_model, bias=False)
        self.out_proj  = nn.Linear(d_model, d_model, bias=False)
        self.attn_drop = nn.Dropout(dropout)
        self.resid_drop= nn.Dropout(dropout)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        B, T, C = x.shape      # batch, sequence length, d_model

        # Project and split into Q, K, V
        qkv = self.qkv_proj(x)                          # (B, T, 3*C)
        Q, K, V = qkv.split(C, dim=-1)                  # each (B, T, C)

        # Reshape to (B, n_heads, T, d_k)
        def split_heads(t: torch.Tensor) -> torch.Tensor:
            return t.view(B, T, self.n_heads, self.d_k).transpose(1, 2)

        Q, K, V = split_heads(Q), split_heads(K), split_heads(V)

        # Scaled dot-product attention
        scale = math.sqrt(self.d_k)
        scores = (Q @ K.transpose(-2, -1)) / scale      # (B, h, T, T)

        # Causal mask: upper triangle (j > i) → -inf
        mask = torch.triu(torch.ones(T, T, device=x.device), diagonal=1).bool()
        scores = scores.masked_fill(mask, float("-inf"))

        attn = F.softmax(scores, dim=-1)
        attn = self.attn_drop(attn)

        # Weighted sum of values
        out = attn @ V                                   # (B, h, T, d_k)
        out = out.transpose(1, 2).contiguous().view(B, T, C)  # (B, T, C)
        return self.resid_drop(self.out_proj(out))


class FeedForward(nn.Module):
    """
    Position-wise FFN: Linear → GELU → Linear
    Expands d_model → d_ff → d_model.
    Each token processed independently (no cross-token mixing).
    """

    def __init__(self, d_model: int, d_ff: int, dropout: float) -> None:
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model),
            nn.Dropout(dropout),
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.net(x)


class TransformerBlock(nn.Module):
    """
    Pre-LayerNorm transformer block (GPT-2 convention):
        x = x + Attn(LN(x))
        x = x + FFN(LN(x))
    Pre-LN stabilizes training: gradients reach early layers without
    passing through attention softmax saturation.
    """

    def __init__(self, cfg: Config) -> None:
        super().__init__()
        self.ln1  = nn.LayerNorm(cfg.d_model)
        self.attn = CausalSelfAttention(cfg.d_model, cfg.n_heads, cfg.dropout)
        self.ln2  = nn.FFN  = nn.LayerNorm(cfg.d_model)
        self.ffn  = FeedForward(cfg.d_model, cfg.d_ff, cfg.dropout)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        x = x + self.attn(self.ln1(x))
        x = x + self.ffn(self.ln2(x))
        return x


class GPT(nn.Module):
    """
    Character-level GPT decoder.
    Embedding: token_embed + learned positional_embed
    Body: n_layers TransformerBlocks
    Head: LayerNorm → Linear(d_model, vocab_size)
    """

    def __init__(self, cfg: Config) -> None:
        super().__init__()
        self.block_size = cfg.block_size

        self.tok_emb = nn.Embedding(cfg.vocab_size, cfg.d_model)
        self.pos_emb = nn.Embedding(cfg.block_size, cfg.d_model)
        self.drop    = nn.Dropout(cfg.dropout)
        self.blocks  = nn.ModuleList([TransformerBlock(cfg) for _ in range(cfg.n_layers)])
        self.ln_f    = nn.LayerNorm(cfg.d_model)
        self.head    = nn.Linear(cfg.d_model, cfg.vocab_size, bias=False)

        # Weight tying: input embedding and output projection share weights
        # Reduces parameters and empirically improves language model perplexity
        self.head.weight = self.tok_emb.weight

        # Initialize weights
        self.apply(self._init_weights)

    def _init_weights(self, module: nn.Module) -> None:
        if isinstance(module, nn.Linear):
            nn.init.normal_(module.weight, mean=0.0, std=0.02)
            if module.bias is not None:
                nn.init.zeros_(module.bias)
        elif isinstance(module, nn.Embedding):
            nn.init.normal_(module.weight, mean=0.0, std=0.02)
        elif isinstance(module, nn.LayerNorm):
            nn.init.ones_(module.weight)
            nn.init.zeros_(module.bias)

    def forward(
        self, idx: torch.Tensor, targets: torch.Tensor | None = None
    ) -> tuple[torch.Tensor, torch.Tensor | None]:
        B, T = idx.shape
        assert T <= self.block_size, f"Sequence length {T} exceeds block_size {self.block_size}"

        positions = torch.arange(T, device=idx.device)       # (T,)
        x = self.drop(self.tok_emb(idx) + self.pos_emb(positions))  # (B, T, d_model)

        for block in self.blocks:
            x = block(x)

        x = self.ln_f(x)                                     # (B, T, d_model)
        logits = self.head(x)                                # (B, T, vocab_size)

        loss = None
        if targets is not None:
            # Flatten for cross-entropy: (B*T, vocab_size) vs (B*T,)
            loss = F.cross_entropy(
                logits.view(-1, logits.size(-1)),
                targets.view(-1),
            )

        return logits, loss

    def count_parameters(self) -> None:
        """Print parameter count breakdown by component."""
        total = 0
        rows = []
        for name, param in self.named_parameters():
            n = param.numel()
            total += n
            rows.append((name, tuple(param.shape), n))

        print(f"\n{'Component':<35} {'Shape':<20} {'Params':>10}")
        print("-" * 67)
        for name, shape, n in rows:
            print(f"{name:<35} {str(shape):<20} {n:>10,}")
        print("-" * 67)
        print(f"{'TOTAL':<56} {total:>10,}")
        # Theoretical: 12 * L * d^2 (attention + FFN per block) + embeddings
        cfg_approx = 12 * 4 * 128**2
        print(f"Theoretical attention+FFN: ~{cfg_approx:,}  (12Ld²)")
```

**Checkpoint:** `GPT(cfg).count_parameters()` prints ~400K–600K total parameters. Run a forward pass: `model(torch.zeros(2, 64, dtype=torch.long))` returns logits of shape `(2, 64, 65)` with no errors. `loss` is approximately `ln(65) ≈ 4.17` on random initialization (uniform distribution).

## Phase 4 — Training Loop with Cosine LR Decay and Validation

**Why this fourth:** A properly scheduled learning rate and gradient clipping are the difference between a model that converges in 5000 steps and one that diverges at step 200.

```python
# ── Training utilities ────────────────────────────────────────────────────────

def cosine_lr(step: int, max_steps: int, lr: float, lr_min: float, warmup: int) -> float:
    """
    Linear warmup for `warmup` steps, then cosine decay to lr_min.
    
    Warmup:  lr_t = lr * t / warmup
    Cosine:  lr_t = lr_min + 0.5 * (lr - lr_min) * (1 + cos(π * progress))
    """
    if step < warmup:
        return lr * step / warmup
    progress = (step - warmup) / (max_steps - warmup)
    return lr_min + 0.5 * (lr - lr_min) * (1.0 + math.cos(math.pi * progress))


@torch.no_grad()
def estimate_loss(
    model: GPT,
    train_ds: CharDataset,
    val_ds: CharDataset,
    cfg: Config,
) -> dict[str, float]:
    """Evaluate average loss over eval_iters random batches of train and val."""
    model.eval()
    results: dict[str, float] = {}

    for split_name, dataset in [("train", train_ds), ("val", val_ds)]:
        losses = []
        loader = torch.utils.data.DataLoader(
            dataset, batch_size=cfg.batch_size, shuffle=True, drop_last=True
        )
        for i, (xb, yb) in enumerate(loader):
            if i >= cfg.eval_iters:
                break
            xb, yb = xb.to(cfg.device), yb.to(cfg.device)
            _, loss = model(xb, yb)
            losses.append(loss.item())
        results[split_name] = float(np.mean(losses))

    model.train()
    return results


def train(
    model: GPT,
    train_ds: CharDataset,
    val_ds: CharDataset,
    cfg: Config,
) -> list[dict]:
    """
    Main training loop.
    Returns list of log dicts: {step, train_loss, val_loss, lr, elapsed}.
    """
    section("Phase 4 — Training Loop")

    model = model.to(cfg.device)
    optimizer = torch.optim.AdamW(
        model.parameters(),
        lr=cfg.lr,
        betas=(0.9, 0.95),
        weight_decay=0.1,
    )

    loader = torch.utils.data.DataLoader(
        train_ds, batch_size=cfg.batch_size, shuffle=True, drop_last=True
    )
    data_iter = iter(loader)

    log: list[dict] = []
    t0 = time.time()

    for step in range(1, cfg.max_steps + 1):
        # Fetch next batch, reset iterator if exhausted
        try:
            xb, yb = next(data_iter)
        except StopIteration:
            data_iter = iter(loader)
            xb, yb = next(data_iter)

        xb, yb = xb.to(cfg.device), yb.to(cfg.device)

        # Set learning rate for this step
        lr_now = cosine_lr(step, cfg.max_steps, cfg.lr, cfg.lr_min, cfg.warmup_steps)
        for param_group in optimizer.param_groups:
            param_group["lr"] = lr_now

        # Forward + backward
        _, loss = model(xb, yb)
        optimizer.zero_grad(set_to_none=True)
        loss.backward()

        # Gradient clipping: prevents exploding gradients in early training
        grad_norm = torch.nn.utils.clip_grad_norm_(model.parameters(), cfg.grad_clip)

        optimizer.step()

        # Periodic evaluation
        if step % cfg.eval_interval == 0 or step == 1:
            losses = estimate_loss(model, train_ds, val_ds, cfg)
            elapsed = time.time() - t0
            log.append({
                "step": step,
                "train_loss": losses["train"],
                "val_loss": losses["val"],
                "lr": lr_now,
                "elapsed": elapsed,
                "grad_norm": float(grad_norm),
            })
            print(
                f"step {step:5d}/{cfg.max_steps} | "
                f"train={losses['train']:.4f} | val={losses['val']:.4f} | "
                f"lr={lr_now:.2e} | gnorm={grad_norm:.3f} | {elapsed:.1f}s"
            )

    print(f"\nTraining complete. Final val loss: {log[-1]['val_loss']:.4f}")
    return log
```

**Checkpoint:** After 200 steps, val loss should drop from ~4.17 to ~2.5. After 2000 steps it should be below 2.0. After 5000 steps it should be below 1.6. If val loss stagnates above 2.5 past step 1000, check that `yb` is `xb` shifted by one (Phase 2 `__getitem__`).

## Phase 5 — Text Generation with Temperature and Top-k Sampling

**Why this last:** Generation is the visible proof of learning. Temperature and top-k control the exploration-exploitation trade-off in the output distribution.

```python
# ── Generation ────────────────────────────────────────────────────────────────

@torch.no_grad()
def generate(
    model: GPT,
    prompt_ids: list[int],
    max_new_tokens: int,
    temperature: float = 1.0,
    top_k: int | None = None,
    device: str = "cpu",
) -> list[int]:
    """
    Autoregressive generation from a prompt.
    
    At each step:
      1. Feed current context (up to block_size) through the model.
      2. Take logits at the final position.
      3. Divide by temperature (lower T → sharper, higher T → flatter).
      4. Optionally zero out all but top-k logits.
      5. Sample from softmax distribution.
      6. Append sampled token to context.
    
    temperature=1.0  → unmodified distribution
    temperature=0.7  → more confident, less diverse
    temperature=1.3  → more random, more surprising
    top_k=40         → restrict to 40 most probable tokens at each step
    """
    model.eval()
    idx = torch.tensor([prompt_ids], dtype=torch.long, device=device)  # (1, T)

    generated = list(prompt_ids)

    for _ in range(max_new_tokens):
        # Crop context to block_size
        idx_cond = idx[:, -model.block_size:]          # (1, ≤block_size)

        logits, _ = model(idx_cond)                    # logits: (1, T, vocab_size)
        logits = logits[:, -1, :]                      # last position: (1, vocab_size)

        # Apply temperature
        logits = logits / temperature

        # Top-k filtering: set all logits outside top-k to -inf
        if top_k is not None:
            values, _ = torch.topk(logits, min(top_k, logits.size(-1)))
            threshold = values[:, -1].unsqueeze(-1)    # k-th largest value
            logits = logits.masked_fill(logits < threshold, float("-inf"))

        probs = F.softmax(logits, dim=-1)              # (1, vocab_size)
        next_token = torch.multinomial(probs, num_samples=1)  # (1, 1)

        idx = torch.cat([idx, next_token], dim=1)      # (1, T+1)
        generated.append(next_token.item())

    return generated


def demo_generation(
    model: GPT,
    char2idx: dict[str, int],
    idx2char: dict[int, str],
    cfg: Config,
) -> None:
    section("Phase 5 — Text Generation")

    prompt = "HAMLET:\n"
    prompt_ids = encode(prompt, char2idx)

    configs = [
        ("Temperature 1.0 (baseline)",     1.0, None),
        ("Temperature 0.7 (sharpened)",    0.7, None),
        ("Temperature 1.0 + top-k=10",     1.0, 10),
        ("Temperature 0.7 + top-k=40",     0.7, 40),
    ]

    for label, temp, k in configs:
        print(f"\n--- {label} ---")
        ids = generate(
            model, prompt_ids,
            max_new_tokens=200,
            temperature=temp,
            top_k=k,
            device=cfg.device,
        )
        text = decode(ids, idx2char)
        print(text)
        print()
```

## Debugging Guide

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| Loss stuck at ~4.17 after 500 steps | Causal mask not applied — model sees future tokens and trivially predicts | Verify `torch.triu(..., diagonal=1)` produces `True` above diagonal in `scores.masked_fill` |
| NaN loss after step 1 | Learning rate too high or weight initialization std too large | Confirm `std=0.02` in `_init_weights`; confirm `lr=3e-4`, not `3e-3` |
| `RuntimeError: size mismatch` in `head.weight = tok_emb.weight` | Weight tying requires identical shapes — head output must equal vocab_size | Verify `nn.Linear(d_model, vocab_size, bias=False)` and `nn.Embedding(vocab_size, d_model)` use the same vocab_size |
| Generated text is all one repeated character | Top-k too small (e.g. top_k=1 is greedy decoding) or temperature too low | Start with temperature=1.0, top_k=None; add constraints after confirming model learned |
| Val loss diverges from train loss after step 2000 | Overfitting — model memorized training windows | Increase dropout to 0.2, reduce d_model to 64, or reduce max_steps |
| CUDA out of memory | batch_size * block_size * d_model creates large intermediate tensors | Reduce batch_size to 32 or block_size to 128; use `torch.cuda.empty_cache()` |

## What This Project Teaches You to Say in an Interview

1. "Causal masking in self-attention is implemented by adding $-\infty$ to the upper triangle of the attention score matrix before softmax — this collapses to zero probability for future tokens while leaving the mathematical form of softmax unchanged."
2. "Weight tying between the token embedding and the output projection reduces parameters by `vocab_size × d_model` and constrains the model to represent the same semantic space at input and output — empirically it lowers perplexity by 5–15% on character-level models."
3. "Cosine LR decay with linear warmup prevents loss spikes during early training (when gradients are large and random) and allows finer updates near convergence. Warmup is critical because AdamW's adaptive estimates are unreliable at step 1."
4. "Pre-LayerNorm (LN before attention) vs Post-LayerNorm matters: pre-LN gradients flow more directly to early layers because they bypass the attention softmax, making training stabler at large depth and high learning rates."
5. "Temperature $T$ controls entropy of the output distribution: $H(p_T) = -\sum_i \frac{e^{z_i/T}}{\sum_j e^{z_j/T}} \log \frac{e^{z_i/T}}{\sum_j e^{z_j/T}}$. Top-k truncates the support of the distribution, eliminating very low probability tokens that tend to produce incoherent text regardless of temperature."
