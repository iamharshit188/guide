# Mini Seq2Seq Translation Model — Thinking Guide

> **Module:** 07 — Transformers from Scratch · **Difficulty:** Expert · **Type:** THINKING · **Time:** ~6 hours

## The Problem

**Input:** Sentence pairs from Multi30k or a Tatoeba English→French subset (~30K–150K pairs). Each source sentence is a sequence of English tokens $\mathbf{x} = (x_1, \ldots, x_S)$; each target is a French sequence $\mathbf{y} = (y_1, \ldots, y_T)$.

**Output:** A trained encoder-decoder transformer that, given an English sentence at inference time, generates a French translation token by token. Report BLEU-4 on the Multi30k test set (target: ≥ 25 BLEU with 2-layer encoder/decoder on Multi30k).

**Constraints:** Training uses teacher forcing. Inference uses beam search with beam width $\geq 4$. Both source and target use shared or separate BPE vocabularies of 8K–16K tokens. Model fits in 8 GB VRAM (a 2-layer, 256-dim transformer does so comfortably).

## Before You Write Any Code: Build The Mental Model

The fundamental question is: why encoder-decoder and not a decoder-only model? A decoder-only model for translation would concatenate source and target and do causal generation — this works, but it forces the model to re-attend to the source through a fixed causal window. The encoder-decoder architecture processes the full source bidirectionally (no causal mask in the encoder), producing rich contextual representations $\mathbf{H}^{enc} \in \mathbb{R}^{S \times d}$, then uses cross-attention in the decoder to selectively read from those representations at each generation step. Cross-attention is the architectural expression of the linguistic fact that translating word $t$ requires looking at specific source words — not the entire source indiscriminately.

Cross-attention mechanics: the decoder at layer $l$, position $t$ computes $Q = W_Q h_t^{dec}$, $K = W_K H^{enc}$, $V = W_V H^{enc}$. The query comes from the decoder (what the decoder needs), the keys and values come from the encoder (what the source offers). The attention weights $\alpha_{t,s} = \text{softmax}(QK^T / \sqrt{d_k})$ form a soft alignment matrix: high $\alpha_{t,s}$ means the $t$-th target token heavily depends on the $s$-th source token. This is a continuous, learned version of the word alignment matrices from classical statistical MT.

Teacher forcing during training feeds the ground-truth target prefix at each step: given $(y_1, \ldots, y_{t-1})$ as input, predict $y_t$. This is computationally efficient (all positions predicted in parallel via a causal mask on the decoder) but introduces exposure bias — at inference, the model sees its own (possibly wrong) predictions as prefix, not ground truth. Scheduled sampling partially fixes this by interpolating between teacher-forced and autoregressive inputs as training progresses.

BLEU-4 computes the geometric mean of 1-gram through 4-gram precision multiplied by a brevity penalty. It rewards exact n-gram overlap with references — it does not reward paraphrase, synonymy, or structural equivalence. A model that generates "the large feline sat on the mat" gets zero BLEU-4 overlap with "le grand chat était assis sur le tapis" unless those exact French n-grams appear. Use sacreBLEU for standardized, reproducible BLEU computation.

## Decompose It

### Phase 1 — Tokenization and Vocabulary

**The question this phase answers:** What are the token vocabularies for source and target, and how are out-of-vocabulary words handled?

BPE with 8K merges on each language independently is the standard approach for Multi30k. Alternatively, use shared BPE (joint vocabulary over both languages) which forces the model to discover cross-lingual subword overlap — useful for related languages like English/French where many words share Latin roots. Shared vocabulary reduces total vocab size and may improve alignment learning, but it conflates English and French tokens that happen to share surface form.

**Key decisions:**
- Separate vocabularies (8K EN + 8K FR = 16K total) vs. shared vocabulary (8K joint): separate vocabularies allow each language's BPE to learn its own morphological patterns; shared vocabulary aids low-resource languages where sharing helps data efficiency.
- SentencePiece vs. BPE via HuggingFace tokenizers: SentencePiece handles whitespace as part of the token, making detokenization trivial; HuggingFace tokenizers are faster for large corpora.
- Special tokens: you need `<bos>` (decoder start), `<eos>` (generation stop), `<pad>` (batch padding), `<unk>`. The `<bos>` token is the first input to the decoder at inference time — the model generates $P(y_1 \mid \text{<bos>}, H^{enc})$.

**Trap:** Tokenizing source and target with the same vocabulary trained only on source text. French subwords that don't appear in English will map to `<unk>` — the model cannot learn to generate them. Always train the BPE on the language it tokenizes.

**Done when:** `src_vocab_size` and `tgt_vocab_size` are both set. `encode("Hello world") → decode(encode("Hello world"))` round-trips cleanly. Every token in the training set maps to a known ID — no `<unk>` tokens in the training data (only in validation/test on unseen words).

### Phase 2 — Encoder Architecture

**The question this phase answers:** How does the encoder produce $H^{enc}$ from source tokens, and why is bidirectional attention valid here?

The encoder is a stack of $N$ identical layers, each with multi-head self-attention (no causal mask) and a position-wise FFN. Every source token attends to every other source token: position $s$ can attend to positions $1 \ldots S$ in both directions. This is valid because the full source sentence is known at translation time — unlike the decoder, the encoder does not need to generate tokens causally.

Sinusoidal positional encoding $PE(pos, 2i) = \sin(pos / 10000^{2i/d})$, $PE(pos, 2i+1) = \cos(pos / 10000^{2i/d})$ is the standard choice for seq2seq. It generalizes to sequence lengths longer than seen during training (unlike learned PE, which has no representation for position 513 if you trained on sequences of at most 512). For very long sequences (document-level translation), relative positional encodings (RoPE, ALiBi) are superior because sinusoidal PE degrades past the training length.

**Key decisions:**
- Number of encoder layers $N$: 2 layers suffice for Multi30k (small dataset); 6 layers (original Transformer) for large corpora. Increasing $N$ without proportionally increasing training data risks overfitting.
- $d_{model}$ and $n_{heads}$: $d_{model}=256$, $n_{heads}=8$ (so $d_k = 32$) is a practical minimum. Below $d_k = 16$ per head, attention heads cannot differentiate meaningful patterns.

**Trap:** Applying a causal mask to the encoder's self-attention. This makes encoding essentially left-to-right (like a unidirectional RNN) and severely limits the quality of encoder representations. The encoder's cross-entropy-style supervision (from decoder gradients) will still train, but BLEU will be 5–10 points lower than with full bidirectional attention.

**Done when:** A batch of padded source sequences of shape $(B, S)$ produces encoder output of shape $(B, S, d_{model})$ with no NaN values. The padding mask correctly zeroes attention over `<pad>` positions.

### Phase 3 — Decoder Architecture and Cross-Attention

**The question this phase answers:** What are the three attention operations in each decoder layer, and how do they interact?

Each decoder layer has three sublayers:
1. **Causal self-attention** over target prefix: decoder position $t$ attends to target positions $1 \ldots t$ only. The causal mask prevents future target tokens from influencing past predictions during teacher-forced training.
2. **Cross-attention** over encoder output: $Q$ from decoder hidden state, $K$ and $V$ from $H^{enc}$. No mask needed — the decoder may attend to any source position.
3. **Position-wise FFN**: same as in the encoder.

The causal mask on decoder self-attention is what makes teacher forcing parallel. Without the mask, position $t$ would see $y_{t+1}, \ldots, y_T$ and trivially predict the next token by copying. With the mask, all $T$ positions are predicted simultaneously but each only sees its valid prefix — $T$ independent autoregressive steps in a single matrix multiply.

**Key decisions:**
- Pre-LayerNorm vs. Post-LayerNorm: pre-LN (normalize before attention) trains more stably, especially at depth > 4. Post-LN matches the original Transformer paper and converges to slightly better final quality on large datasets but diverges more easily.
- Number of decoder layers: must match or exceed encoder layers for good translation. Empirically, equal depth (both $N=2$ or both $N=6$) outperforms mismatched depths on standard benchmarks.

**Trap:** Using the same mask for cross-attention as for self-attention (applying causal masking to the cross-attention over encoder outputs). The cross-attention mask should only mask out `<pad>` source tokens, not future positions, because the encoder has already processed the full source — there are no "future" source positions from the decoder's perspective.

**Done when:** Decoder forward pass with shape $(B, T)$ target tokens and encoder memory $(B, S, d_{model})$ produces logits $(B, T, \text{tgt\_vocab\_size})$ without error. Verify the causal mask by checking that attention weights at position $t$ are zero for positions $> t$.

### Phase 4 — Training with Teacher Forcing and Label Smoothing

**The question this phase answers:** How do you compute the loss, and why does label smoothing prevent overconfident softmax?

Teacher-forced training: decoder input is `<bos> y_1 y_2 ... y_{T-1}`, decoder target is `y_1 y_2 ... y_T <eos>`. The model predicts all $T$ tokens in parallel. Loss is cross-entropy averaged over non-padding positions:

$$\mathcal{L} = -\frac{1}{|\mathcal{T}|} \sum_{t \in \mathcal{T}} \log P(y_t \mid y_{<t}, \mathbf{x})$$

where $\mathcal{T}$ excludes `<pad>` positions. Weight the loss by position count (not batch count) to handle variable-length sequences fairly.

Label smoothing with $\epsilon = 0.1$ replaces the one-hot target distribution with $(1 - \epsilon) \cdot \text{one\_hot} + \epsilon / V$. This prevents the model from assigning probability approaching 1.0 to the teacher-forced target — a pathology that occurs because the model never has to recover from its own mistakes during training. Overconfident models generalize worse at inference because they assume the prefix is always correct.

**Key decisions:**
- How to handle pad tokens in loss: mask them with `ignore_index=pad_id` in `nn.CrossEntropyLoss`. Failing to mask pads makes the model learn to predict `<pad>` after `<eos>`, which wastes capacity and inflates gradient estimates.
- Optimizer: AdamW with $\beta_1=0.9$, $\beta_2=0.98$, $\epsilon=10^{-9}$ (original Transformer values). Learning rate schedule: linear warmup for 4000 steps then $lr = d_{model}^{-0.5} \cdot \min(\text{step}^{-0.5}, \text{step} \cdot \text{warmup}^{-1.5})$.

**Trap:** Computing loss over the `<bos>` position. The decoder input starts with `<bos>`, but the target starts with `y_1` — there is no loss for predicting `<bos>`. If you include it, you're computing cross-entropy between the model's prediction at position 0 and `y_1`, which is a valid signal but not part of the standard training objective. Shift targets correctly: `decoder_input = tgt[:, :-1]`, `decoder_target = tgt[:, 1:]`.

**Done when:** Training loss decreases from ~$\log(V)$ at initialization to below 2.0 after 10 epochs on Multi30k. Validation perplexity (exp of loss) decreases monotonically for at least the first 15 epochs. Label-smoothed loss values are slightly higher than non-smoothed for the same model — this is expected.

### Phase 5 — Beam Search and BLEU Evaluation

**The question this phase answers:** Why does greedy decoding systematically underperform beam search for translation, and how do you implement beam search correctly?

Greedy decoding at each step selects $\hat{y}_t = \argmax P(y_t \mid y_{<t}, \mathbf{x})$. The problem: a greedy choice at step $t$ closes off high-probability continuations that require a slightly suboptimal choice at $t$ followed by a very high-probability $t+1$ and $t+2$. Beam search maintains the $k$ highest-probability partial hypotheses (beams) at each step, expanding all of them and keeping only the top $k$.

Beam search with width $k$ explores $k$ paths simultaneously, selecting the top-$k$ sequences by total log-probability $\sum_t \log P(y_t \mid y_{<t}, \mathbf{x})$. Length normalization divides by sequence length to prevent the model from preferring shorter translations (which accumulate fewer log-probability terms). Length penalty: divide by $((5 + T) / 6)^\alpha$ where $\alpha \in [0.6, 0.8]$ is common.

BLEU-4 failures to anticipate: (1) it rewards fluent but wrong translations if they happen to share 4-grams with the reference; (2) it penalizes valid paraphrases with zero credit; (3) brevity penalty is asymmetric — extra words are penalized by precision, missing words only by brevity penalty; (4) a single reference is much harsher than multi-reference BLEU, which is what the original Transformer paper used. Use sacrebleu's `corpus_bleu` with `tokenize="13a"` for standard comparability.

**Key decisions:**
- Beam width $k$: $k=4$ is the minimum for good results; $k=8$ gives marginal improvement; diminishing returns beyond $k=8$ on multi30k.
- When to terminate a beam: when `<eos>` is generated, move that beam to the completed set and continue with the remaining beams. A beam that generates `<eos>` early should not block other beams from generating longer (potentially better) translations.
- Maximum generation length: set as $1.5 \times S$ (source length) or a hard cap. Without a maximum, a degenerate model can loop indefinitely.

**Trap:** Comparing beams by sum of log-probabilities without length normalization. Shorter translations always have higher total log-probability (fewer terms, each bounded by $\leq 0$). Without normalization, beam search consistently produces one-word translations that maximize log-probability by terminating early.

**Done when:** BLEU-4 on Multi30k test set is ≥ 25 with beam width 4. Greedy BLEU should be 2–5 points lower — this gap demonstrates the value of beam search. Running sacrebleu in signature mode (`sacrebleu --test-set multi30k/16 -l en-fr`) provides a standardized baseline for comparison.

## Data Flow

```
Source sentence (English tokens)
        │  src_ids: (B, S)
        ▼
Source embedding + sinusoidal PE
        │  (B, S, d_model)
        ▼
Encoder — N × [Self-Attn (no mask) → FFN]
        │  H_enc: (B, S, d_model)  ← frozen during decoder forward
        │
        │  Target prefix (teacher-forced): <bos> y_1 ... y_{T-1}
        │  tgt_ids: (B, T)
        ▼
Target embedding + sinusoidal PE
        │  (B, T, d_model)
        ▼
Decoder — N × [Causal Self-Attn → Cross-Attn(Q=dec, K/V=H_enc) → FFN]
        │  (B, T, d_model)
        ▼
Linear projection → logits (B, T, tgt_vocab_size)
        ▼
Label-smoothed cross-entropy vs. y_1 ... y_T <eos>
        │  scalar loss
        ▼
Backprop → AdamW update

── Inference Path ──────────────────────────────────────────
H_enc = encoder(src_ids)          [computed once]
beams = [([<bos>], log_prob=0)]
for step in range(max_len):
    for each beam:
        logits = decoder(beam_tokens, H_enc)[-1]   [last position]
        top_k candidates → extend beam
    keep top-k beams by cumulative log_prob / length_penalty
    if all beams ended with <eos>: break
return best completed beam → decode → French string → BLEU
```

## Edge Cases to Handle Before You Code

1. **Variable-length batching with padding:** Source sentences in a batch have different lengths. Pad shorter sequences to `max_len` in the batch. The encoder self-attention mask must mark pad positions as $-\infty$ in attention scores so pad tokens don't receive meaningful attention — otherwise the encoder's `<pad>` representations leak into cross-attention and corrupt decoder queries at those positions.

2. **Identical source and target lengths causing mask confusion:** When source and target lengths coincidentally match, it is tempting to reuse the same mask tensor for both self-attention and cross-attention. They are different masks: the decoder self-attention mask is $(T \times T)$ causal; the cross-attention mask is $(T \times S)$ with `<pad>` marking. Reusing the causal mask for cross-attention blocks all source positions beyond $T$, which is catastrophically wrong when $S > T$.

3. **`<eos>` token in the source sentence:** If the source tokenizer adds `<eos>` at the end, the encoder processes it — this is fine and common. But if the decoder cross-attention mask is computed from source lengths (excluding `<eos>`), there is an off-by-one that unintentionally masks the final meaningful source token. Always compute masks from the actual padded length, not from `len - 1`.

4. **Beam search: duplicate completed hypotheses:** If two different beams both generate `<eos>` at the same step and have similar log-probabilities, they may decode to the same French string. Deduplicate completed hypotheses before selecting the final output to avoid returning the same translation twice in an n-best list.

## The Hard Design Decisions

| Decision | Option A | Option B | When to pick A |
|----------|---------|---------|----------------|
| Shared vs. separate BPE vocabularies | Shared 8K joint | Separate 8K EN + 8K FR | Shared when languages are closely related and dataset is small; separate when languages are morphologically distant or dataset is large enough to learn both independently |
| Sinusoidal vs. relative PE | Sinusoidal (fixed, extrapolates) | Relative PE / RoPE (learned relative distances) | Sinusoidal for Multi30k (short sentences, fixed training length); relative PE for document-level translation where test sequences are longer than training |
| Teacher forcing ratio | 1.0 (always teacher-forced) | Scheduled sampling (decay from 1.0 to ~0.5) | 1.0 for first 80% of training; scheduled sampling only if you observe severe performance collapse at inference vs. training (exposure bias manifesting as high train BLEU, low test BLEU gap > 8 points) |
| Label smoothing $\epsilon$ | 0.1 (standard) | 0.0 (hard targets) | 0.1 almost always; 0.0 only if training set is tiny (< 5K pairs) and you cannot afford to dilute the signal |

## What Makes This Non-Trivial

The cross-attention mask interaction is subtle: during beam search with batch size $>1$ (where different beams may correspond to different source sentences), you must replicate each source's encoder output $k$ times (one per beam), and the padding mask must be replicated correspondingly. Failing to replicate the mask while replicating encoder outputs leads to silent correctness errors: beams for source sentence 0 attend to source sentence 1's encoder outputs without masking.

Exposure bias is not theoretical for translation — on Multi30k, a model trained purely with teacher forcing typically shows a 4–6 BLEU gap between forced (training-time) and free-running (inference) evaluation. Scheduled sampling reduces this gap by 1–3 BLEU but requires careful annealing to avoid training instability during the transition period.

The interaction between label smoothing and BLEU is non-monotonic: models trained with $\epsilon = 0.1$ achieve higher BLEU than those with $\epsilon = 0.0$ despite having higher training loss, because they generalize better at inference. This is one of the clearest examples in NLP where lower training loss does not imply better downstream performance.

## Interview Angles This Project Develops

1. **"Why cross-attention and not self-attention over the concatenated source and target?"** — Strong answer: cross-attention is $O(T \cdot S)$ and processes source bidirectionally in the encoder; concatenating and applying causal attention is $O((S+T)^2)$ and forces the decoder to re-read the source causally. The encoder-decoder split also enables caching the encoder output once and reusing it for all $T$ decoding steps, which is critical for beam search efficiency.

2. **"What is the exposure bias problem and how does your architecture address it?"** — Strong answer: exposure bias is the train/inference distribution mismatch where training conditions on ground-truth prefixes but inference conditions on model predictions. Scheduled sampling interpolates between the two during training. Describe when it matters (large translation datasets with many similar sentences) and when it doesn't (small datasets where overfitting is the bigger risk).

3. **"How does beam search interact with length normalization?"** — Strong answer: without normalization, beam search score $= \sum_t \log P(y_t)$ is negative and decreasing in $T$ (each term $\leq 0$), so shorter sequences always score higher. Length normalization $= \text{score} / T^\alpha$ corrects this bias. $\alpha = 0$ recovers raw score; $\alpha = 1$ fully normalizes by length; $\alpha \in [0.6, 0.8]$ is empirically optimal for most translation pairs.

4. **"What does BLEU-4 measure precisely, and why is it insufficient as a sole metric?"** — Strong answer: BLEU-4 is the geometric mean of precision of 1- through 4-gram matches between hypothesis and reference, multiplied by a brevity penalty $e^{1 - r/h}$ where $r$ is reference length and $h$ is hypothesis length. It fails on synonymy (zero credit for valid paraphrases), on reordering (French SOV constructions vs. English SVO), and on low-frequency but correct translations. Supplement with chrF (character F-score), BERTScore (embedding similarity), and human evaluation.

5. **"How would you extend this architecture to many-to-one multilingual translation (10 languages → French)?"** — Strong answer: prepend a language ID token to the source (`<en>`, `<de>`, `<es>`, etc.), use a shared encoder and decoder across all language pairs, and rely on the model to learn language-specific patterns via the language token. This is the mBART/M2M-100 approach. The shared vocabulary must cover all source languages — use joint BPE trained on multilingual data.
