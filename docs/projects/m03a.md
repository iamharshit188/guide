# Semantic Code Search Engine — Step-by-Step Guide

> **Module:** 03 — Databases & Vector DBs · **Difficulty:** Intermediate · **Type:** GUIDED · **Time:** ~3 hours

## What You'll Build

A CLI tool that crawls a Python codebase, parses every `.py` file with the `ast` module to extract function/class signatures and docstrings, embeds them using `sentence-transformers`, and upserts them into a persistent ChromaDB collection. A REPL query loop accepts natural-language questions and returns the top-5 semantically closest code units with `file:line` references and cosine similarity scores.

## The Mental Model

Semantic search replaces exact-string matching with a geometry problem: embed both the query and every code chunk into the same high-dimensional space, then find the nearest neighbours by cosine distance. The search quality depends entirely on embedding quality — `all-MiniLM-L6-v2` was trained on hundreds of millions of sentence pairs, so it understands that "read a file line by line" and `open(...).readlines()` live in similar directions in $\mathbb{R}^{384}$.

ChromaDB handles the HNSW index, persistence, and metadata filtering so you can concentrate on the pipeline. The hard part is the AST extraction: you need enough context per chunk (signature + docstring + sometimes first few lines) for the embedding to be informative, but not so much that chunks become semantically noisy.

## Phase 1 — Crawl and Parse the Codebase

**Why this first:** Garbage in, garbage out. The extractor determines chunk granularity and metadata richness. Every downstream phase depends on what comes out here.

```python
# extractor.py
import ast
import textwrap
from pathlib import Path
from dataclasses import dataclass, field
from typing import Iterator


@dataclass
class CodeChunk:
    chunk_id: str          # "path/to/file.py::ClassName.method_name::42"
    text: str              # signature + docstring (what gets embedded)
    filepath: str
    line: int
    kind: str              # "function" | "class" | "method"
    name: str


def _build_text(node: ast.FunctionDef | ast.AsyncFunctionDef | ast.ClassDef,
                source_lines: list[str]) -> str:
    """Combine signature line(s) + docstring into a single embeddable string."""
    # Signature: reconstruct from source rather than unparse to preserve style
    sig_end = node.body[0].lineno - 1  # first statement in body
    sig_lines = source_lines[node.lineno - 1 : sig_end]
    signature = " ".join(l.strip() for l in sig_lines).strip()

    docstring = ast.get_docstring(node) or ""
    # Truncate docstring so chunks stay under ~300 tokens
    docstring = textwrap.shorten(docstring, width=512, placeholder="...")
    return f"{signature}\n{docstring}".strip()


def extract_chunks(filepath: Path, root: Path) -> Iterator[CodeChunk]:
    """Yield one CodeChunk per function/class in filepath."""
    source = filepath.read_text(encoding="utf-8", errors="ignore")
    source_lines = source.splitlines()
    try:
        tree = ast.parse(source, filename=str(filepath))
    except SyntaxError:
        return

    rel = filepath.relative_to(root)

    for node in ast.walk(tree):
        if not isinstance(node, (ast.FunctionDef, ast.AsyncFunctionDef, ast.ClassDef)):
            continue
        if node.lineno is None:
            continue

        # Skip dunder-only private helpers unless they have docstrings
        if node.name.startswith("__") and not ast.get_docstring(node):
            continue

        kind = "class" if isinstance(node, ast.ClassDef) else "function"
        # Detect methods (parent is a ClassDef)
        # ast.walk loses parent info — use a separate traversal for methods
        chunk_id = f"{rel}::{node.name}::{node.lineno}"
        text = _build_text(node, source_lines)
        if len(text) < 10:          # skip trivial one-liners with no docstring
            continue

        yield CodeChunk(
            chunk_id=chunk_id,
            text=text,
            filepath=str(rel),
            line=node.lineno,
            kind=kind,
            name=node.name,
        )


def crawl(root: Path) -> list[CodeChunk]:
    chunks: list[CodeChunk] = []
    for py_file in sorted(root.rglob("*.py")):
        # Skip virtual envs, caches
        parts = py_file.parts
        if any(p in parts for p in {"__pycache__", ".venv", "venv", "env", "site-packages"}):
            continue
        chunks.extend(extract_chunks(py_file, root))
    print(f"[extractor] {len(chunks)} chunks from {root}")
    return chunks
```

**Checkpoint:** Run `python -c "from extractor import crawl; from pathlib import Path; chunks = crawl(Path('.')); print(chunks[0])"` against your own codebase. You should see a `CodeChunk` with a non-empty `text` field containing the function signature and docstring.

## Phase 2 — Embed with sentence-transformers

**Why this phase:** The embedding model transforms variable-length text into fixed-length vectors in $\mathbb{R}^{384}$. Batching matters — encoding 1000 chunks one at a time is ~10× slower than a single `.encode()` call with a large batch.

```python
# embedder.py
from sentence_transformers import SentenceTransformer
import numpy as np
from extractor import CodeChunk

MODEL_NAME = "all-MiniLM-L6-v2"   # 80 MB, 384-dim, fast on CPU


class Embedder:
    def __init__(self) -> None:
        self.model = SentenceTransformer(MODEL_NAME)
        self.dim = self.model.get_sentence_embedding_dimension()
        print(f"[embedder] loaded {MODEL_NAME}, dim={self.dim}")

    def embed_chunks(self, chunks: list[CodeChunk], batch_size: int = 64) -> np.ndarray:
        """Return (N, dim) float32 array, one row per chunk."""
        texts = [c.text for c in chunks]
        embeddings = self.model.encode(
            texts,
            batch_size=batch_size,
            show_progress_bar=True,
            normalize_embeddings=True,   # L2-normalize → cosine sim == dot product
            convert_to_numpy=True,
        )
        return embeddings.astype(np.float32)

    def embed_query(self, query: str) -> np.ndarray:
        """Return (1, dim) float32 array."""
        vec = self.model.encode(
            [query],
            normalize_embeddings=True,
            convert_to_numpy=True,
        )
        return vec.astype(np.float32)
```

**Checkpoint:** `Embedder().embed_chunks(chunks[:10])` should return shape `(10, 384)`. Every row should have L2-norm ≈ 1.0 — verify with `np.linalg.norm(emb, axis=1)`.

## Phase 3 — Upsert to ChromaDB

**Why this phase:** ChromaDB persists vectors and metadata to disk so you don't re-index on every run. The `upsert` (not `add`) call is idempotent — running the indexer twice won't duplicate chunks.

```python
# indexer.py
import chromadb
from chromadb.config import Settings
from pathlib import Path
from extractor import crawl, CodeChunk
from embedder import Embedder
import numpy as np

COLLECTION_NAME = "code_index"
PERSIST_DIR = ".chroma_db"


def get_collection(persist_dir: str = PERSIST_DIR) -> chromadb.Collection:
    client = chromadb.PersistentClient(path=persist_dir)
    collection = client.get_or_create_collection(
        name=COLLECTION_NAME,
        metadata={"hnsw:space": "cosine"},   # ChromaDB uses this for distance metric
    )
    return collection


def index_codebase(root: Path, force: bool = False) -> None:
    collection = get_collection()

    if not force and collection.count() > 0:
        print(f"[indexer] {collection.count()} chunks already indexed. Use --reindex to rebuild.")
        return

    if force and collection.count() > 0:
        # Delete all existing documents before re-indexing
        existing_ids = collection.get()["ids"]
        if existing_ids:
            collection.delete(ids=existing_ids)
        print(f"[indexer] cleared existing index")

    chunks = crawl(root)
    if not chunks:
        print("[indexer] no chunks found — check the root path")
        return

    embedder = Embedder()
    embeddings = embedder.embed_chunks(chunks)   # (N, 384)

    # ChromaDB upsert takes lists, not numpy arrays directly
    collection.upsert(
        ids=[c.chunk_id for c in chunks],
        embeddings=embeddings.tolist(),
        documents=[c.text for c in chunks],
        metadatas=[
            {
                "filepath": c.filepath,
                "line": c.line,
                "kind": c.kind,
                "name": c.name,
            }
            for c in chunks
        ],
    )
    print(f"[indexer] upserted {len(chunks)} chunks → {PERSIST_DIR}")


if __name__ == "__main__":
    import argparse

    parser = argparse.ArgumentParser(description="Index a Python codebase into ChromaDB")
    parser.add_argument("root", type=Path, help="Root directory to index")
    parser.add_argument("--reindex", action="store_true", help="Delete existing index and rebuild")
    args = parser.parse_args()

    index_codebase(args.root, force=args.reindex)
```

**Checkpoint:** Run `python indexer.py /path/to/some/python/project`. The `.chroma_db/` directory should appear. Re-run without `--reindex` — it should say "already indexed" and skip. Run with `--reindex` — it should re-crawl and re-embed.

## Phase 4 — Query Loop: Embed → Search → Display

**Why this phase:** This is the product. The query embedding must be produced with the same model and normalization as the indexed embeddings — mixing models or forgetting `normalize_embeddings=True` silently degrades results.

```python
# search.py
import textwrap
from indexer import get_collection
from embedder import Embedder

TOP_K = 5


def search(query: str, embedder: Embedder, kind_filter: str | None = None) -> list[dict]:
    """
    Return top-K results as dicts with keys:
    score, name, kind, filepath, line, snippet
    """
    collection = get_collection()
    query_vec = embedder.embed_query(query)   # (1, 384)

    where_clause = {"kind": kind_filter} if kind_filter else None

    results = collection.query(
        query_embeddings=query_vec.tolist(),
        n_results=TOP_K,
        where=where_clause,
        include=["documents", "metadatas", "distances"],
    )

    hits = []
    for doc, meta, dist in zip(
        results["documents"][0],
        results["metadatas"][0],
        results["distances"][0],
    ):
        # ChromaDB cosine distance = 1 - cosine_similarity (when space="cosine")
        similarity = 1.0 - dist
        hits.append({
            "score": round(similarity, 4),
            "name": meta["name"],
            "kind": meta["kind"],
            "filepath": meta["filepath"],
            "line": meta["line"],
            "snippet": doc,
        })
    return hits


def display(hits: list[dict]) -> None:
    if not hits:
        print("No results.")
        return
    for i, h in enumerate(hits, 1):
        ref = f"{h['filepath']}:{h['line']}"
        header = f"[{i}] {h['kind'].upper()} {h['name']}  ({ref})  score={h['score']}"
        snippet = textwrap.indent(
            textwrap.shorten(h["snippet"], width=200, placeholder="..."),
            prefix="    ",
        )
        print(header)
        print(snippet)
        print()


def repl() -> None:
    embedder = Embedder()
    collection = get_collection()
    count = collection.count()
    if count == 0:
        print("Index is empty. Run: python indexer.py <root> first.")
        return
    print(f"Code search ready — {count} chunks indexed. Ctrl-C to quit.\n")

    while True:
        try:
            query = input("search> ").strip()
        except (KeyboardInterrupt, EOFError):
            print("\nBye.")
            break
        if not query:
            continue

        # Optional filter: prefix query with "class:" or "function:"
        kind_filter = None
        if query.startswith("class:"):
            kind_filter = "class"
            query = query[6:].strip()
        elif query.startswith("function:"):
            kind_filter = "function"
            query = query[9:].strip()

        hits = search(query, embedder, kind_filter=kind_filter)
        display(hits)


if __name__ == "__main__":
    repl()
```

**Checkpoint:** Start the REPL and try:
- `"read a file and return its lines"` — should surface file-reading functions
- `"class: data container with fields"` — should surface dataclass-like classes
- `"function: compute moving average"` — if your codebase has one, it should appear in top-3

## Phase 5 — CLI Entrypoint and Re-index Support

**Why this phase:** A single `main.py` with subcommands (`index`, `search`, `stats`) makes the tool usable as a real CLI, not just a REPL you have to remember how to start.

```python
# main.py
import argparse
from pathlib import Path


def cmd_index(args: argparse.Namespace) -> None:
    from indexer import index_codebase
    index_codebase(args.root, force=args.reindex)


def cmd_search(args: argparse.Namespace) -> None:
    from embedder import Embedder
    from search import search, display
    embedder = Embedder()
    hits = search(args.query, embedder)
    display(hits)


def cmd_stats(args: argparse.Namespace) -> None:
    from indexer import get_collection
    collection = get_collection()
    count = collection.count()
    if count == 0:
        print("Index is empty.")
        return

    sample = collection.get(limit=5, include=["metadatas"])
    kinds: dict[str, int] = {}
    # Full kind breakdown requires fetching all metadata
    all_meta = collection.get(include=["metadatas"])["metadatas"]
    for m in all_meta:
        kinds[m["kind"]] = kinds.get(m["kind"], 0) + 1

    print(f"Total chunks : {count}")
    for k, v in sorted(kinds.items()):
        print(f"  {k:12s}: {v}")
    print(f"Sample IDs   : {sample['ids']}")


def main() -> None:
    parser = argparse.ArgumentParser(prog="codesearch", description="Semantic code search")
    sub = parser.add_subparsers(dest="cmd", required=True)

    p_index = sub.add_parser("index", help="Index a codebase")
    p_index.add_argument("root", type=Path)
    p_index.add_argument("--reindex", action="store_true")
    p_index.set_defaults(func=cmd_index)

    p_search = sub.add_parser("search", help="One-shot query (non-interactive)")
    p_search.add_argument("query")
    p_search.set_defaults(func=cmd_search)

    p_stats = sub.add_parser("stats", help="Show index statistics")
    p_stats.set_defaults(func=cmd_stats)

    sub.add_parser("repl", help="Interactive query REPL").set_defaults(
        func=lambda _: __import__("search").repl()
    )

    args = parser.parse_args()
    args.func(args)


if __name__ == "__main__":
    main()
```

**Checkpoint:** Full workflow:
```bash
python main.py index /path/to/project
python main.py stats
python main.py search "parse command line arguments"
python main.py repl
```

## Debugging Guide

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| `collection.query()` returns very low scores (< 0.3) for obviously relevant queries | Embeddings were indexed without `normalize_embeddings=True` but queried with normalization (or vice versa) | Delete the collection, re-index with `--reindex`, keep normalization consistent in both `embed_chunks` and `embed_query` |
| `SyntaxError` during crawl halts the whole run | A `.py` file uses a newer syntax (e.g., match-case on Python 3.9) | The `try/except SyntaxError: return` in `extract_chunks` already handles this — verify it's present |
| ChromaDB raises `DuplicateIDError` on second run | Using `collection.add()` instead of `collection.upsert()` | Switch to `upsert` — it inserts new IDs and updates existing ones |
| REPL hangs on first query | Model not yet loaded (SentenceTransformer downloads on first import) | Normal — first run downloads ~80 MB. Subsequent runs use the local cache at `~/.cache/huggingface/` |
| Results look good for code but bad for English phrases | The index was built only on signatures without docstrings | Ensure `_build_text` includes the docstring; skip chunks where `len(text) < 10` |

## What This Project Teaches You to Say in an Interview

1. "ChromaDB stores embeddings in an HNSW index — approximate nearest neighbour search that trades a small recall penalty for sub-linear query time. For exact search you'd use a flat index, which is $O(N)$ per query."
2. "I normalize embeddings to unit length before upserting so that cosine similarity reduces to a dot product — ChromaDB's HNSW then optimises for that metric."
3. "Chunking strategy is the biggest lever on retrieval quality. I chunk at the function/class level because a method has a coherent semantic unit — splitting mid-function mixes two concepts in one embedding."
4. "Re-indexing is idempotent because I use `upsert` keyed on a stable `filepath::name::line` ID. If a function moves, its old chunk stays; the next `--reindex` wipes and rebuilds."
5. "sentence-transformers' `all-MiniLM-L6-v2` is a bi-encoder — query and document are encoded independently, so retrieval is fast. A cross-encoder re-ranker (e.g., `cross-encoder/ms-marco-MiniLM-L-6-v2`) could re-score the top-20 results for higher precision at the cost of 20 forward passes."
