# Full-Stack AI Blog Platform — Step-by-Step Guide

> **Module:** 14 — Frontend & Full-Stack · **Difficulty:** Advanced · **Type:** GUIDED · **Time:** ~8 hours

## What You'll Build

A Next.js 14 blog where posts are markdown files served from a Flask API, each post page has an on-demand AI summary button that caches the result in `localStorage`, and a client-side TF-IDF search engine that ranks all posts in real time without hitting the backend. The whole stack is deployable: Next.js to Vercel, Flask to any Python host (Render, Railway, Fly.io).

## The Mental Model

The architecture is intentionally split: Flask owns all data access (file I/O, markdown parsing, AI calls) while Next.js owns presentation and client-side intelligence. The AI summary sits at the boundary — Flask does the OpenAI API call, Next.js caches the result. The TF-IDF search is purely client-side: on mount, the React app fetches all post metadata, builds an inverted index in memory, and never touches the server again for search queries. This means zero search latency after the initial fetch — every keystroke is a pure JavaScript operation over an in-memory data structure.

Understanding TF-IDF is necessary before you implement it: term frequency $\text{TF}(t, d) = \frac{f_{t,d}}{\sum_k f_{k,d}}$ (fraction of document that is term $t$), inverse document frequency $\text{IDF}(t) = \log\frac{N}{|\{d : t \in d\}|}$ (log of inverse document frequency — rare terms score higher). The final score $\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \text{IDF}(t)$ down-weights common words (the, is, a) and up-weights distinctive terms.

## Phase 1 — Flask API: Markdown File Server

**Why this first:** The frontend is useless without data. The Flask API is the single source of truth for post content — get it right before building anything on top.

#!/usr/bin/env python3

from __future__ import annotations
import os
import re
from pathlib import Path

import frontmatter
from flask import Flask, jsonify, request, abort
from flask_cors import CORS
from openai import OpenAI

CONTENT_DIR = Path(__file__).parent / "content"
app = Flask(__name__)
CORS(app)  # Allow Next.js dev server (localhost:3000) to call Flask (localhost:5000)

openai_client = OpenAI()  # reads OPENAI_API_KEY from environment


def section(title: str) -> None:
    width = 60
    print(f"\n{'=' * width}\n  {title}\n{'=' * width}")


def load_post(slug: str) -> dict | None:
    """Load a single post by slug. Returns None if not found."""
    path = CONTENT_DIR / f"{slug}.md"
    if not path.exists():
        return None
    post = frontmatter.load(str(path))
    return {
        "slug": slug,
        "title": post.metadata.get("title", slug.replace("-", " ").title()),
        "date": str(post.metadata.get("date", "")),
        "tags": post.metadata.get("tags", []),
        "excerpt": post.metadata.get("excerpt", post.content[:200].strip()),
        "content": post.content,
    }


def load_all_posts() -> list[dict]:
    """Load all posts, sorted by date descending."""
    posts = []
    for path in CONTENT_DIR.glob("*.md"):
        slug = path.stem
        post_data = load_post(slug)
        if post_data:
            posts.append(post_data)
    posts.sort(key=lambda p: p["date"], reverse=True)
    return posts


@app.route("/posts", methods=["GET"])
def get_posts() -> tuple:
    """Return list of post metadata (no full content — keep payload small)."""
    posts = load_all_posts()
    metadata = [
        {k: v for k, v in p.items() if k != "content"}
        for p in posts
    ]
    return jsonify(metadata)


@app.route("/posts/<slug>", methods=["GET"])
def get_post(slug: str) -> tuple:
    """Return full post including markdown content."""
    # Sanitize slug: only alphanumeric, hyphens, underscores
    if not re.fullmatch(r"[a-zA-Z0-9_-]+", slug):
        abort(400, description="Invalid slug format")
    post = load_post(slug)
    if post is None:
        abort(404, description=f"Post '{slug}' not found")
    return jsonify(post)


@app.route("/summarize", methods=["POST"])
def summarize() -> tuple:
    """
    Receive {slug, content}, call OpenAI to summarize, return {summary}.
    The content is passed from the client to avoid a second Flask DB read.
    """
    data = request.get_json(silent=True)
    if not data or "content" not in data:
        abort(400, description="Request body must include 'content'")

    content = data["content"][:4000]  # Truncate to avoid token limit
    slug = data.get("slug", "unknown")

    try:
        response = openai_client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[
                {
                    "role": "system",
                    "content": (
                        "You are a technical blog summarizer. "
                        "Summarize the following blog post in exactly 2 sentences. "
                        "Be precise and dense — omit filler phrases."
                    ),
                },
                {"role": "user", "content": content},
            ],
            max_tokens=120,
            temperature=0.3,
        )
        summary = response.choices[0].message.content.strip()
        return jsonify({"slug": slug, "summary": summary})
    except Exception as exc:
        abort(500, description=f"OpenAI error: {exc}")

**Why this third:** The summary feature demonstrates the caching pattern that eliminates redundant API calls — the most important performance pattern in this stack.

The summary button logic is already wired in Phase 2's `PostPage`. What this phase adds: the complete `localStorage` caching strategy and the Flask `POST /summarize` endpoint (already written in Phase 1). The client-side flow:

```javascript
// Extracted as a reusable hook — pages/posts/[slug].js
function useSummary(slug, content) {
  const CACHE_KEY = `summary_${slug}`
  const [summary, setSummary] = useState(null)
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState(null)

  // Load from cache on mount
  useEffect(() => {
    try {
      const cached = localStorage.getItem(CACHE_KEY)
      if (cached) setSummary(cached)
    } catch {
      // localStorage can throw in private browsing mode — handle gracefully
    }
  }, [CACHE_KEY])

  async function generate() {
    // Check cache again (race condition: two tabs could both request)
    const cached = localStorage.getItem(CACHE_KEY)
    if (cached) { setSummary(cached); return }

    setLoading(true)
    setError(null)
    try {
      const res = await fetch(`${API_BASE}/summarize`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ slug, content }),
      })
      if (!res.ok) throw new Error(`Server error ${res.status}`)
      const { summary: text } = await res.json()
      setSummary(text)
      localStorage.setItem(CACHE_KEY, text)
    } catch (err) {
      setError(err.message)
    } finally {
      setLoading(false)
    }
  }

  return { summary, loading, error, generate }
}
```

## Phase 4 — Client-Side TF-IDF Search

**Why this fourth:** This phase demonstrates that full-text search over small corpora does not require a backend — and teaches the TF-IDF algorithm concretely.

```javascript
// lib/tfidf.js — TF-IDF implementation
export function tokenize(text) {
  // Lowercase, remove punctuation, split on whitespace, filter stopwords
  const STOPWORDS = new Set([
    'the', 'a', 'an', 'is', 'it', 'in', 'of', 'to', 'and',
    'for', 'on', 'at', 'by', 'are', 'be', 'this', 'that',
    'with', 'from', 'as', 'or', 'was', 'but', 'not', 'we',
    'he', 'she', 'they', 'have', 'has', 'had', 'will', 'would',
  ])
  return text
    .toLowerCase()
    .replace(/[^a-z0-9\s]/g, ' ')
    .split(/\s+/)
    .filter(w => w.length > 2 && !STOPWORDS.has(w))
}

export function buildIndex(posts) {
  // posts: [{slug, title, content, excerpt, tags}, ...]
  const N = posts.length

  // Term frequency per document: {slug: {term: tf}}
  const tf = {}
  // Document frequency: {term: count of docs containing term}
  const df = {}

  for (const post of posts) {
    const text = [post.title, post.excerpt, post.content, ...(post.tags || [])].join(' ')
    const tokens = tokenize(text)
    const termCount = {}
    for (const token of tokens) {
      termCount[token] = (termCount[token] || 0) + 1
    }
    const totalTerms = tokens.length || 1
    tf[post.slug] = {}
    for (const [term, count] of Object.entries(termCount)) {
      tf[post.slug][term] = count / totalTerms  // TF: fraction of doc
      df[term] = (df[term] || 0) + 1           // DF: docs containing term
    }
  }

  // IDF: log(N / df(term))  — add 1 to df to avoid division by zero
  const idf = {}
  for (const [term, docFreq] of Object.entries(df)) {
    idf[term] = Math.log(N / docFreq)
  }

  return { tf, idf, posts }
}

export function search(query, index) {
  // Returns posts sorted by descending TF-IDF score
  const { tf, idf, posts } = index
  const queryTerms = tokenize(query)
  if (queryTerms.length === 0) return posts  // empty query → return all

  const scores = {}
  for (const post of posts) {
    scores[post.slug] = 0
    for (const term of queryTerms) {
      const termTF = (tf[post.slug] || {})[term] || 0
      const termIDF = idf[term] || 0  // 0 for terms not in any doc
      scores[post.slug] += termTF * termIDF
    }
  }

  return [...posts].sort((a, b) => scores[b.slug] - scores[a.slug])
}
```

// components/SearchBar.js — Replace the placeholder in pages/index.js
import { useState, useEffect, useRef } from 'react'
import { buildIndex, search } from '../lib/tfidf'

const API_BASE = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:5000'

export default function SearchBar({ onResults, initialPosts }) {
  const [query, setQuery] = useState('')
  const indexRef = useRef(null)  // Persist index across re-renders without state

  // Build index once on mount (after fetching full content for all posts)
  useEffect(() => {
    async function initIndex() {
      try {
        // Fetch full content for all posts (needed for content-based search)
        const res = await fetch(`${API_BASE}/posts`)
        const metadata = await res.json()
        // Use metadata only — avoid fetching full content of each post
        // For full content search, fetch each post; for demo, use excerpt + title
        indexRef.current = buildIndex(metadata)
        onResults(metadata)  // Initial state: show all posts
      } catch {
        indexRef.current = buildIndex(initialPosts)
      }
    }
    initIndex()
  }, [])  // Run once on mount

  function handleChange(e) {
    const q = e.target.value
    setQuery(q)
    if (!indexRef.current) return
    const results = search(q, indexRef.current)
    onResults(results)
  }

  return (
    <div className="search-bar">
      <input
        type="search"
        value={query}
        onChange={handleChange}
        placeholder="Search posts by keyword..."
        aria-label="Search posts"
      />
      {query && (
        <span className="search-hint">
          {/* Results are updated above via onResults callback */}
          Searching for: {query}
        </span>
      )}
    </div>
  )
}

// components/ThemeToggle.js
import { useState, useEffect } from 'react'

export default function ThemeToggle() {
  const [dark, setDark] = useState(false)

  // Initialize from localStorage on mount
  useEffect(() => {
    const saved = localStorage.getItem('theme')
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches
    const isDark = saved === 'dark' || (!saved && prefersDark)
    setDark(isDark)
    document.documentElement.classList.toggle('dark', isDark)
  }, [])

  function toggle() {
    const next = !dark
    setDark(next)
    document.documentElement.classList.toggle('dark', next)
    localStorage.setItem('theme', next ? 'dark' : 'light')
  }

  return (
    <button
      onClick={toggle}
      aria-label={dark ? 'Switch to light mode' : 'Switch to dark mode'}
      className="theme-toggle"
    >
      {dark ? '☀ Light' : '◑ Dark'}
    </button>
  )
}

| Symptom | Likely Cause | Fix |
|---|---|---|
| CORS error in browser console | Flask is not sending CORS headers | Verify `CORS(app)` is called after app creation, before routes; check that the origin matches (localhost:3000 vs 127.0.0.1:3000) |
| `getServerSideProps` returns empty posts | Flask not running or wrong port | Start Flask first (`python server.py`); verify `NEXT_PUBLIC_API_URL` in `.env.local`; check Flask logs for incoming request |
| TF-IDF search returns all posts in wrong order | Stopword list too aggressive or tokenizer strips useful terms | Log `tokenize(query)` — if it returns empty array, query is all stopwords; reduce stopword list |
| AI summary button always shows loading | Flask throws 500 on `/summarize` | Check that `OPENAI_API_KEY` env var is set; check Flask console for traceback; test with `curl -X POST http://localhost:5000/summarize -H 'Content-Type: application/json' -d '{"slug":"test","content":"test content"}'` |
| Theme flashes on page load | `useEffect` runs after hydration, briefly showing wrong theme | Move theme initialization to `_document.js` as an inline script to run before React hydration |

## What This Project Teaches You to Say in an Interview

1. **"I implemented client-side TF-IDF search that requires zero backend calls after initial data load."** Explain the inverted index structure, IDF formula, and why `useRef` (not `useState`) stores the index — mutation doesn't need to trigger a re-render.

2. **"The AI summary uses a cache-then-fetch pattern with localStorage."** The component checks the cache before making any API call, and writes to cache on success. Mention the private browsing edge case where `localStorage.getItem` throws — you wrapped it in try/catch.

3. **"Flask-CORS is essential when Next.js (port 3000) calls Flask (port 5000) — browsers enforce same-origin policy on all cross-origin fetches."** Describe what happens without CORS: the preflight OPTIONS request fails with 403, and the browser blocks the actual request.

4. **"CSS custom properties (variables) enable theme switching without JavaScript-managed style strings."** Toggling a class on `html` element changes all `var()` references in the cascade simultaneously — no per-component style injection, no re-render cycle.

5. **"`getServerSideProps` fetches post metadata on the server before the page HTML is generated, so the post list is in the HTML — search-engine crawlable and instant to render."** Contrast with `useEffect` fetching: that runs after the page loads and produces a flash of empty content.
