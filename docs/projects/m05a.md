# Neural Network Training Dashboard — Step-by-Step Guide

> **Module:** 05 — Deep Learning & MLOps · **Difficulty:** Intermediate · **Type:** GUIDED · **Time:** ~4 hours

## What You'll Build

A 2-layer NumPy neural network trained on a synthetic multi-class dataset, with a live per-epoch terminal table showing train/val loss, val accuracy, and learning rate. Every run logs params, metrics, and the final weight artifact to MLflow; the trained model is registered in MLflow Model Registry with a printed model card. No PyTorch, no TensorFlow.

---

## The Mental Model

A neural network is a composition of affine maps and nonlinearities. The forward pass computes $\hat{y} = \text{softmax}(W_2 \cdot \text{ReLU}(W_1 x + b_1) + b_2)$. The backward pass propagates the gradient of the cross-entropy loss $\mathcal{L} = -\sum_c y_c \log \hat{y}_c$ through each layer via the chain rule. Because softmax and cross-entropy compose cleanly, the gradient at the output layer simplifies to $\delta_2 = \hat{y} - y$. MLflow wraps the whole experiment: every hyperparameter, every epoch metric, and the final weight file are stored so runs are reproducible and comparable.

## Phase 1 — Dataset and Utilities

**Why this first:** Clean data and consistent helpers prevent silent bugs in later phases.

```python
import numpy as np
import mlflow
import mlflow.pyfunc
import json
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

def section(title: str) -> None:
    width = 60
    print(f"\n{'=' * width}")
    print(f"  {title}")
    print(f"{'=' * width}")

def make_dataset(n_samples: int = 2000, n_features: int = 16,
                 n_classes: int = 4, seed: int = 42) -> tuple:
    """Gaussian clusters — one per class, no sklearn dependency for generation."""
    rng = np.random.default_rng(seed)
    X_parts, y_parts = [], []
    for c in range(n_classes):
        center = rng.uniform(-4, 4, size=n_features)
        X_c = rng.normal(loc=center, scale=1.2,
                         size=(n_samples // n_classes, n_features))
        X_parts.append(X_c)
        y_parts.append(np.full(n_samples // n_classes, c, dtype=int))
    X = np.vstack(X_parts).astype(np.float64)
    y = np.concatenate(y_parts)
    # Standardize: zero mean, unit variance per feature
    mu, sigma = X.mean(axis=0), X.std(axis=0) + 1e-8
    X = (X - mu) / sigma
    return train_test_split(X, y, test_size=0.2, random_state=seed, stratify=y)

def to_onehot(y: np.ndarray, n_classes: int) -> np.ndarray:
    oh = np.zeros((len(y), n_classes), dtype=np.float64)
    oh[np.arange(len(y)), y] = 1.0
    return oh
```

**Checkpoint:** `X_train.shape == (1600, 16)`, `X_val.shape == (400, 16)`. `X_train.mean(axis=0)` ≈ 0 (within ±0.2). `np.unique(y_train)` returns `[0, 1, 2, 3]`.

## Phase 2 — NumPy Neural Network (Forward & Backward)

**Why this phase:** Getting the math right here makes the training loop trivial.

```python
# ── Activations ────────────────────────────────────────────
def relu(z: np.ndarray) -> np.ndarray:
    return np.maximum(0.0, z)

def relu_grad(z: np.ndarray) -> np.ndarray:
    """Derivative of ReLU: 1 where z > 0, else 0."""
    return (z > 0).astype(np.float64)

def softmax(z: np.ndarray) -> np.ndarray:
    """Numerically stable row-wise softmax."""
    e = np.exp(z - z.max(axis=1, keepdims=True))
    return e / e.sum(axis=1, keepdims=True)

# ── Loss ────────────────────────────────────────────────────
def cross_entropy(y_oh: np.ndarray, y_hat: np.ndarray) -> float:
    """
    L = -sum_c y_c * log(y_hat_c)
    Clip y_hat to avoid log(0).
    """
    return -np.sum(y_oh * np.log(np.clip(y_hat, 1e-12, 1.0))) / len(y_oh)

# ── Parameter init ──────────────────────────────────────────
def init_params(n_in: int, n_hidden: int, n_out: int,
                seed: int = 42) -> dict:
    """He initialisation for ReLU layers."""
    rng = np.random.default_rng(seed)
    return {
        "W1": rng.normal(0, np.sqrt(2.0 / n_in),  (n_in, n_hidden)),
        "b1": np.zeros(n_hidden),
        "W2": rng.normal(0, np.sqrt(2.0 / n_hidden), (n_hidden, n_out)),
        "b2": np.zeros(n_out),
    }

# ── Forward pass ────────────────────────────────────────────
def forward(X: np.ndarray, params: dict) -> dict:
    z1 = X @ params["W1"] + params["b1"]          # (N, hidden)
    a1 = relu(z1)                                  # (N, hidden)
    z2 = a1 @ params["W2"] + params["b2"]          # (N, out)
    a2 = softmax(z2)                               # (N, out)
    return {"z1": z1, "a1": a1, "z2": z2, "a2": a2}

# ── Backward pass ───────────────────────────────────────────
def backward(X: np.ndarray, y_oh: np.ndarray,
             cache: dict, params: dict) -> dict:
    N = X.shape[0]
    # Output layer: fused softmax + cross-entropy gradient
    dz2 = (cache["a2"] - y_oh) / N                # (N, out)
    dW2 = cache["a1"].T @ dz2                      # (hidden, out)
    db2 = dz2.sum(axis=0)                          # (out,)

    # Hidden layer: backprop through ReLU
    da1 = dz2 @ params["W2"].T                     # (N, hidden)
    dz1 = da1 * relu_grad(cache["z1"])             # (N, hidden)
    dW1 = X.T @ dz1                               # (in, hidden)
    db1 = dz1.sum(axis=0)                          # (hidden,)

    return {"dW1": dW1, "db1": db1, "dW2": dW2, "db2": db2}
```

**Checkpoint:** At random initialisation, `cross_entropy(y_oh, a2)` ≈ `log(n_classes) ≈ 1.386`. If it's far from this, initialisation is wrong.

## Phase 3 — DataLoader and Training Loop

**Why this phase:** Mini-batch SGD generalises better than full-batch; shuffling each epoch prevents order-dependent convergence.

```python
def make_batches(X: np.ndarray, y: np.ndarray,
                 batch_size: int, rng: np.random.Generator):
    idx = rng.permutation(len(X))
    for start in range(0, len(X), batch_size):
        b = idx[start:start + batch_size]
        yield X[b], y[b]

def sgd_update(params: dict, grads: dict, lr: float) -> None:
    for key in params:
        params[key] -= lr * grads[f"d{key}"]

def train(X_train, y_train, X_val, y_val,
          n_hidden: int = 64, lr: float = 0.05,
          batch_size: int = 128, n_epochs: int = 60,
          seed: int = 42) -> tuple:

    rng = np.random.default_rng(seed)
    n_features = X_train.shape[1]
    n_classes  = len(np.unique(y_train))
    params = init_params(n_features, n_hidden, n_classes, seed)
    y_train_oh = to_onehot(y_train, n_classes)

    history = []

    # Print header
    print(f"\n{'Epoch':>6} {'LR':>9} {'Train Loss':>11} "
          f"{'Val Loss':>10} {'Val Acc':>9}")
    print("-" * 50)

    for epoch in range(1, n_epochs + 1):
        # Cosine LR decay (no warmup — simple schedule)
        current_lr = lr * 0.5 * (1 + np.cos(np.pi * epoch / n_epochs))

        # Mini-batch training
        epoch_loss = 0.0
        n_batches = 0
        for X_b, y_b in make_batches(X_train, y_train, batch_size, rng):
            y_b_oh = to_onehot(y_b, n_classes)
            cache  = forward(X_b, params)
            loss_b = cross_entropy(y_b_oh, cache["a2"])
            grads  = backward(X_b, y_b_oh, cache, params)
            sgd_update(params, grads, current_lr)
            epoch_loss += loss_b
            n_batches  += 1

        train_loss = epoch_loss / n_batches

        # Validation
        val_cache = forward(X_val, params)
        val_loss  = cross_entropy(to_onehot(y_val, n_classes), val_cache["a2"])
        val_preds = val_cache["a2"].argmax(axis=1)
        val_acc   = accuracy_score(y_val, val_preds)

        history.append({
            "epoch": epoch, "lr": current_lr,
            "train_loss": train_loss, "val_loss": val_loss,
            "val_acc": val_acc,
        })

        if epoch % 10 == 0 or epoch == 1:
            print(f"{epoch:>6} {current_lr:>9.5f} {train_loss:>11.4f} "
                  f"{val_loss:>10.4f} {val_acc:>8.2%}")

    return params, history
```

**Checkpoint:** Loss at epoch 1 should be < 1.386. Loss should strictly decrease for the first 20 epochs with `lr=0.05`. Val accuracy > 70% by epoch 30 on this dataset.

## Phase 4 — MLflow Tracking

**Why this phase:** Reproducibility requires logging hyperparams and metrics alongside the artifact — not separately.

```python
def log_to_mlflow(params_dict: dict, history: list,
                  weights: dict, model_name: str = "numpy-nn") -> str:
    """
    Log an experiment run: hyperparams → params, metrics per epoch,
    weights as JSON artifact.
    Returns the run_id.
    """
    mlflow.set_experiment("numpy-nn-dashboard")

    with mlflow.start_run() as run:
        # Hyperparameters
        for k, v in params_dict.items():
            mlflow.log_param(k, v)

        # Per-epoch metrics
        for row in history:
            step = row["epoch"]
            mlflow.log_metric("train_loss", row["train_loss"], step=step)
            mlflow.log_metric("val_loss",   row["val_loss"],   step=step)
            mlflow.log_metric("val_acc",    row["val_acc"],    step=step)
            mlflow.log_metric("lr",         row["lr"],         step=step)

        # Serialize weights as JSON artifact
        weight_path = "/tmp/numpy_nn_weights.json"
        serializable = {k: v.tolist() for k, v in weights.items()}
        with open(weight_path, "w") as f:
            json.dump(serializable, f)
        mlflow.log_artifact(weight_path, artifact_path="model")

        run_id = run.info.run_id

    return run_id
```

**Checkpoint:** `mlflow ui` shows one experiment named `numpy-nn-dashboard`. The run has 4 params and 4 metric series. The `model/numpy_nn_weights.json` artifact is visible.

## Phase 5 — Model Registry and Summary Card

**Why this phase:** Registering a model version creates a governance trail — which weights produced which accuracy.

```python
def register_model(run_id: str, model_name: str,
                   history: list, hparams: dict) -> None:
    """Register model in MLflow Model Registry and print a model card."""
    artifact_uri = f"runs:/{run_id}/model"

    # Register (creates version 1 if model doesn't exist)
    result = mlflow.register_model(artifact_uri, model_name)
    version = result.version

    best = min(history, key=lambda r: r["val_loss"])
    final = history[-1]

    card = f"""
╔══════════════════════════════════════════════════════╗
║              MODEL CARD — {model_name:<24}║
╠══════════════════════════════════════════════════════╣
║  Registry version : {version:<33}║
║  Run ID           : {run_id[:8]}...{' ' * 23}║
╠══════════════════════════════════════════════════════╣
║  HYPERPARAMETERS                                     ║
║  n_hidden   : {hparams['n_hidden']:<38}║
║  lr         : {hparams['lr']:<38}║
║  batch_size : {hparams['batch_size']:<38}║
║  n_epochs   : {hparams['n_epochs']:<38}║
╠══════════════════════════════════════════════════════╣
║  PERFORMANCE                                         ║
║  Best val_loss  : {best['val_loss']:.4f} at epoch {best['epoch']:<19}║
║  Final val_acc  : {final['val_acc']:.2%}{' ' * 30}║
╚══════════════════════════════════════════════════════╝"""
    print(card)


def main() -> None:
    section("Dataset")
    X_train, X_val, y_train, y_val = make_dataset()
    print(f"Train: {X_train.shape}  Val: {X_val.shape}  "
          f"Classes: {len(np.unique(y_train))}")

    hparams = {
        "n_hidden": 64, "lr": 0.05,
        "batch_size": 128, "n_epochs": 60,
    }

    section("Training")
    weights, history = train(
        X_train, y_train, X_val, y_val, **hparams
    )

    section("MLflow Logging")
    run_id = log_to_mlflow(hparams, history, weights)
    print(f"Run ID: {run_id}")

    section("Model Registry")
    register_model(run_id, "numpy-nn", history, hparams)

    section("Final Summary")
    best = min(history, key=lambda r: r["val_loss"])
    print(f"Best val_loss: {best['val_loss']:.4f} at epoch {best['epoch']}")
    print(f"Final val_acc: {history[-1]['val_acc']:.2%}")
    print("\nView experiment: mlflow ui --host 0.0.0.0 --port 5000")


if __name__ == "__main__":
    main()
```

**Checkpoint:** Running `python3.14 solution.py` produces a training table, prints the model card, and exits cleanly. `mlflow.search_registered_models()` returns the `numpy-nn` model.

## Debugging Guide

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| Loss is `nan` from epoch 1 | LR too large → exploding gradients | Reduce `lr` to `1e-2`; check `log(0)` in cross-entropy |
| Val accuracy stuck at 25% | Softmax not normalising | Assert `a2.sum(axis=1).mean() ≈ 1.0` |
| Loss decreases but val_acc doesn't | Onehot mismatch between train and val | Use the same `n_classes` for both `to_onehot` calls |
| MLflow `RestException` | No tracking URI set, filesystem collision | Run `mlflow.set_tracking_uri("./mlruns")` before `set_experiment` |
| `register_model` raises `MlflowException` | Artifact path doesn't match `log_artifact` path | Check `artifact_path="model"` matches URI fragment |

## What This Project Teaches You to Say in an Interview

1. "The fused softmax + cross-entropy gradient is $\hat{y} - y$, which avoids a numerically unstable explicit Jacobian of the softmax."
2. "He initialisation sets variance to $2/n_{in}$, compensating for the dead-neuron effect of ReLU cutting half the activation space."
3. "MLflow tracks runs immutably — each run_id maps to a fixed artifact URI, so you can reproduce any historical model by loading its weights from that URI."
4. "The Model Registry adds a governance layer on top of runs: you can promote a version to Staging or Production, tag it with a champion/challenger label, and roll back with a single API call."
5. "Mini-batch SGD converges faster than full-batch because the gradient noise acts as implicit regularisation and escapes shallow local minima."

## Hints

<details><summary>Hint 1 — Why clip in cross-entropy?</summary>
<code>np.log(0)</code> returns <code>-inf</code>, making the loss <code>nan</code>. <code>np.clip(y_hat, 1e-12, 1.0)</code> prevents this without changing gradients meaningfully because softmax outputs are never exactly 0 on real data — only with dead neurons.
</details>

<details><summary>Hint 2 — Dividing gradients by N</summary>
The backward pass divides <code>dz2</code> by <code>N</code> (batch size). This makes the gradient scale invariant to batch size — otherwise, doubling the batch size doubles the gradient magnitude and requires halving the LR.
</details>

<details><summary>Hint 3 — ReLU gradient is a mask</summary>
<code>relu_grad(z) = (z > 0)</code> — ones where the pre-activation was positive, zeros elsewhere. Multiply element-wise against the upstream gradient. Neurons where <code>z1 <= 0</code> receive zero gradient (they are "dead" for that sample).
</details>

<details><summary>Hint 4 — JSON serialisation of NumPy arrays</summary>
<code>json.dump</code> cannot serialise <code>np.ndarray</code>. Convert with <code>.tolist()</code> before dumping. To reload: <code>np.array(json.load(f)["W1"])</code>.
</details>

<details><summary>Hint 5 — MLflow local vs remote</summary>
Without configuration, MLflow writes to <code>./mlruns</code> in the current directory. Set <code>MLFLOW_TRACKING_URI=http://localhost:5000</code> to point at a tracking server. For this project, local file-based tracking is sufficient.
</details>
