# Domain-Specific LoRA Fine-Tuner — Step-by-Step Guide

> **Module:** 09 — Fine-Tuning LLMs · **Difficulty:** Expert · **Type:** GUIDED · **Time:** ~4 hours

## What You'll Build

A complete LoRA fine-tuning pipeline that loads Mistral-7B (or Llama-3-8B) with 4-bit BitsAndBytes quantization, formats a domain dataset (medical QA, legal text, or coding) into instruction format, trains using PEFT's SFTTrainer with a LoRA adapter (r=16, alpha=32), evaluates perplexity before and after training on a held-out split, then merges the adapter back into the base weights and saves the final model. The end product is a domain-shifted model stored on disk, ready for inference or further evaluation.

## The Mental Model

LoRA (Low-Rank Adaptation) freezes the pretrained weight matrix $W_0 \in \mathbb{R}^{d \times k}$ and injects two trainable matrices $B \in \mathbb{R}^{d \times r}$ and $A \in \mathbb{R}^{r \times k}$, where $r \ll \min(d, k)$. The forward pass becomes:

$$h = (W_0 + \Delta W)x = \left(W_0 + \frac{\alpha}{r} \cdot BA\right)x$$

The scaling factor $\frac{\alpha}{r}$ controls how strongly the adapter signal overrides the base model. With $r=16$ on a 7B model's attention projections, you train roughly 20M parameters instead of 7B — a 350× reduction. $A$ is initialized with random Gaussian noise; $B$ is initialized to zero so $\Delta W = 0$ at the start of training, meaning the adapter starts as an identity perturbation and learns from there.

Why attention weights specifically? The query, key, and value projection matrices ($W_Q, W_K, W_V$) encode how the model attends to context — they're where domain shift manifests most directly. Modifying FFN layers often destabilizes training at low rank. Target `q_proj` and `v_proj` as the minimum effective set; add `k_proj` and `o_proj` if you have headroom.

Perplexity $\text{PPL} = \exp\left(-\frac{1}{N}\sum_{i=1}^N \log P(x_i \mid x_{<i})\right)$ measures how surprised the model is by held-out text. A drop of 20–40% after LoRA training on domain data is a healthy signal. Larger drops suggest the base model was genuinely underfit on your domain; smaller drops suggest your data is either redundant with pretraining or too small.

## Phase 1 — Base Model Load with 4-Bit Quantization

**Why this first:** Loading at full precision would OOM a 16GB GPU. 4-bit NF4 quantization via BitsAndBytes lets the base model occupy ~4.5 GB, leaving headroom for the LoRA adapter, optimizer states, and activations.

```python
#!/usr/bin/env python3
"""Domain-specific LoRA fine-tuner using PEFT + SFTTrainer."""

from __future__ import annotations
import os
import math
import torch
from transformers import (
    AutoModelForCausalLM,
    AutoTokenizer,
    BitsAndBytesConfig,
)

# ── constants ──────────────────────────────────────────────────────────────────
MODEL_ID = "mistralai/Mistral-7B-v0.3"   # swap for "meta-llama/Meta-Llama-3-8B"
MAX_SEQ_LEN = 1024
DEVICE_MAP = "auto"                        # spreads layers across available GPUs


def section(title: str) -> None:
    width = 72
    print(f"\n{'─' * width}")
    print(f"  {title}")
    print(f"{'─' * width}")


def load_base_model() -> tuple:
    """Load model in 4-bit NF4 + tokenizer. Returns (model, tokenizer)."""
    section("Phase 1 — Loading base model (4-bit NF4)")

    bnb_config = BitsAndBytesConfig(
        load_in_4bit=True,
        bnb_4bit_quant_type="nf4",          # NF4 beats INT4 on LLMs (~0.3 PPL)
        bnb_4bit_compute_dtype=torch.bfloat16,   # bfloat16 stable; float16 can NaN
        bnb_4bit_use_double_quant=True,     # nested quantization saves ~0.5 GB
    )

    model = AutoModelForCausalLM.from_pretrained(
        MODEL_ID,
        quantization_config=bnb_config,
        device_map=DEVICE_MAP,
        trust_remote_code=False,
        attn_implementation="flash_attention_2",  # remove if not on Ampere+
    )
    model.config.use_cache = False          # required for gradient checkpointing
    model.config.pretraining_tp = 1        # avoids tensor parallel split issues

    tokenizer = AutoTokenizer.from_pretrained(MODEL_ID, trust_remote_code=False)
    tokenizer.pad_token = tokenizer.eos_token   # Mistral has no pad token
    tokenizer.padding_side = "right"            # left-padding causes issues in SFT

    # report memory
    if torch.cuda.is_available():
        used = torch.cuda.memory_allocated() / 1e9
        total = torch.cuda.get_device_properties(0).total_memory / 1e9
        print(f"  GPU memory after load: {used:.1f} GB / {total:.1f} GB")

    param_count = sum(p.numel() for p in model.parameters())
    print(f"  Total parameters: {param_count / 1e9:.2f}B")
    return model, tokenizer
```

**Checkpoint:** Model loads without OOM. `torch.cuda.memory_allocated()` reports ~4–5 GB used. The model's `dtype` is `torch.uint8` for quantized layers and `torch.bfloat16` for unquantized (norms, embeddings).

## Phase 2 — LoRA Configuration

**Why this second:** The LoRA config must be applied before any optimizer or dataset setup. It freezes base weights, injects adapter modules, and determines which parameters will receive gradients.

```python
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training, TaskType


def build_lora_model(model, r: int = 16, alpha: int = 32, dropout: float = 0.05):
    """Wrap base model with LoRA adapters and return trainable model."""
    section("Phase 2 — LoRA Configuration")

    # prepare_model_for_kbit_training:
    #   1. casts LayerNorm to float32 (prevents NaN in bfloat16)
    #   2. enables gradient checkpointing
    #   3. marks embeddings as requiring no grad
    model = prepare_model_for_kbit_training(model, use_gradient_checkpointing=True)

    # Target modules: q_proj + v_proj is the canonical minimum set.
    # Adding k_proj and o_proj increases parameter count by ~2x but rarely
    # improves final eval loss by more than 0.5–1% on domain tasks.
    lora_config = LoraConfig(
        r=r,
        lora_alpha=alpha,
        target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
        lora_dropout=dropout,
        bias="none",            # "none" is fastest; "all" rarely helps
        task_type=TaskType.CAUSAL_LM,
    )

    model = get_peft_model(model, lora_config)

    # Parameter audit
    total_params = sum(p.numel() for p in model.parameters())
    trainable_params = sum(p.numel() for p in model.parameters() if p.requires_grad)
    pct = 100.0 * trainable_params / total_params

    print(f"  rank r           : {r}")
    print(f"  alpha            : {alpha}  (effective scale = {alpha/r:.1f})")
    print(f"  trainable params : {trainable_params:,} ({pct:.4f}% of total)")
    print(f"  total params     : {total_params:,}")

    # Theoretical LoRA param count per attention projection:
    # (d_model * r) + (r * d_model) = 2 * d_model * r
    # Mistral-7B: d_model = 4096, so per proj = 2 * 4096 * 16 = 131,072
    # 4 projections * 32 layers = 4 * 32 * 131,072 ≈ 16.7M trainable params
    print(f"\n  Expected ≈ 16–20M trainable params for Mistral-7B r=16, 4 proj")
    return model
```

**Checkpoint:** `model.print_trainable_parameters()` shows ~0.24% trainable. Every linear layer in `target_modules` now has a `.lora_A` and `.lora_B` attribute. Confirm with `[n for n, _ in model.named_modules() if 'lora' in n][:5]`.

## Phase 3 — Dataset Formatting

**Why this third:** The tokenizer and model are fixed after Phase 2. Formatting converts raw Q&A pairs into the instruction template the model was pretrained with, then tokenizes to fixed length. Mismatch between training template and inference template is the most common cause of degraded fine-tune quality.

```python
from datasets import load_dataset, DatasetDict
from typing import Any


# Alpaca instruction format — compatible with most 7B instruction-tuned bases
ALPACA_TEMPLATE = (
    "### Instruction:\n{instruction}\n\n"
    "### Input:\n{input}\n\n"
    "### Response:\n{output}"
)

# ChatML format — preferred for Mistral-based models
CHATML_TEMPLATE = (
    "<|im_start|>user\n{instruction}\n<|im_end|>\n"
    "<|im_start|>assistant\n{output}<|im_end|>"
)


def format_example(example: dict[str, Any], template: str = "chatml") -> dict:
    """Map a dataset row to a formatted prompt string."""
    if template == "alpaca":
        text = ALPACA_TEMPLATE.format(
            instruction=example.get("instruction", ""),
            input=example.get("input", ""),
            output=example.get("output", ""),
        )
    else:
        text = CHATML_TEMPLATE.format(
            instruction=example.get("instruction", example.get("question", "")),
            output=example.get("output", example.get("answer", "")),
        )
    return {"text": text}


def load_and_format_dataset(
    dataset_name: str = "medalpaca/medical_meadow_medqa",
    template: str = "chatml",
    train_size: int = 5000,
    val_size: int = 500,
    seed: int = 42,
) -> DatasetDict:
    """
    Load a HuggingFace dataset, format to instruction template, split.

    Alternatives for domain-specific fine-tuning:
    - Medical: "medalpaca/medical_meadow_medqa" (4-option MCQ with explanations)
    - Legal:   "nguyen-brat/legal_contracts" (contract clause QA)
    - Coding:  "iamtarun/python_code_instructions_18k_alpaca" (code generation)
    """
    section("Phase 3 — Dataset Formatting")

    raw = load_dataset(dataset_name, split="train")
    print(f"  Raw dataset size: {len(raw):,} examples")

    # Shuffle and subsample (full dataset often too large for a short run)
    raw = raw.shuffle(seed=seed)
    raw = raw.select(range(min(train_size + val_size, len(raw))))

    # Apply format
    formatted = raw.map(
        lambda ex: format_example(ex, template=template),
        remove_columns=raw.column_names,
        desc="Formatting",
    )

    # Split
    split = formatted.train_test_split(test_size=val_size, seed=seed)
    train_ds = split["train"]
    val_ds = split["test"]

    # Token length audit — critical for setting max_seq_len
    # SFTTrainer will silently truncate; long examples lose their answer
    sample_lengths = []
    for ex in train_ds.select(range(min(200, len(train_ds)))):
        sample_lengths.append(len(ex["text"].split()))
    avg_words = sum(sample_lengths) / len(sample_lengths)
    max_words = max(sample_lengths)
    print(f"  Train examples   : {len(train_ds):,}")
    print(f"  Val examples     : {len(val_ds):,}")
    print(f"  Avg words/example: {avg_words:.0f}  (≈{avg_words*1.3:.0f} tokens)")
    print(f"  Max words/example: {max_words}       (≈{max_words*1.3:.0f} tokens)")
    print(f"  Template          : {template}")

    # Show one formatted example
    print(f"\n  Sample formatted example (truncated):\n")
    print("  " + train_ds[0]["text"][:300].replace("\n", "\n  ") + "...")
    return DatasetDict({"train": train_ds, "validation": val_ds})
```

**Checkpoint:** Dataset prints correct train/val sizes. The formatted text contains the instruction markers (e.g., `<|im_start|>user`). Average token length is below `MAX_SEQ_LEN=1024`; if not, reduce `max_seq_len` or filter long examples with `dataset.filter(lambda x: len(x["text"].split()) < 700)`.

## Phase 4 — SFTTrainer Training Loop

**Why this fourth:** SFTTrainer (from TRL) wraps HuggingFace Trainer with SFT-specific defaults: completion-only loss masking, packing support, and dataset text formatting. Gradient checkpointing trades compute for memory — recomputing activations on the backward pass instead of caching them.

```python
from trl import SFTTrainer, SFTConfig
from transformers import TrainingArguments
import wandb  # optional — set WANDB_DISABLED=true to skip


def train(model, tokenizer, dataset: DatasetDict, output_dir: str = "./lora-output"):
    """Configure and run SFTTrainer."""
    section("Phase 4 — Training")

    training_args = SFTConfig(
        output_dir=output_dir,
        num_train_epochs=3,
        per_device_train_batch_size=2,          # increase if VRAM permits
        per_device_eval_batch_size=2,
        gradient_accumulation_steps=8,          # effective batch = 2*8 = 16
        gradient_checkpointing=True,
        optim="paged_adamw_32bit",              # paged optimizer reduces VRAM spikes
        save_strategy="epoch",
        logging_steps=25,
        learning_rate=2e-4,                     # LoRA LRs are higher than full FT
        weight_decay=0.001,
        fp16=False,
        bf16=True,                              # bfloat16 preferred on Ampere GPUs
        max_grad_norm=0.3,                      # clip gradients — LoRA is sensitive
        warmup_ratio=0.03,                      # 3% warmup stabilizes early steps
        lr_scheduler_type="cosine",
        evaluation_strategy="epoch",
        load_best_model_at_end=True,
        metric_for_best_model="eval_loss",
        report_to="none",                       # set to "wandb" for experiment tracking
        max_seq_length=MAX_SEQ_LEN,
        dataset_text_field="text",
        packing=False,                          # packing concatenates short examples;
                                                # disable for QA (avoids answer leakage)
    )

    trainer = SFTTrainer(
        model=model,
        args=training_args,
        train_dataset=dataset["train"],
        eval_dataset=dataset["validation"],
        tokenizer=tokenizer,
    )

    print(f"  Steps per epoch  : {len(trainer.get_train_dataloader()):,}")
    print(f"  Total steps      : {trainer.args.max_steps if trainer.args.max_steps > 0 else 'auto'}")
    print(f"  Effective batch  : {training_args.per_device_train_batch_size * training_args.gradient_accumulation_steps}")
    print(f"  LR               : {training_args.learning_rate}")
    print(f"  Scheduler        : {training_args.lr_scheduler_type}")

    result = trainer.train()

    print(f"\n  Training complete.")
    print(f"  Final train loss : {result.training_loss:.4f}")
    print(f"  Runtime          : {result.metrics.get('train_runtime', 0):.0f}s")
    print(f"  Tokens/sec       : {result.metrics.get('train_samples_per_second', 0):.1f}")

    trainer.save_model(output_dir)
    return trainer
```

**Checkpoint:** Loss decreases over the first 50 steps from ~2.5 to ~1.5 for a domain dataset. If loss is flat or rising, check: (a) LR too high (try 1e-4), (b) template mismatch (model never saw these tokens), (c) dataset is too small (< 500 examples is usually insufficient).

## Phase 5 — Perplexity Evaluation

**Why this fifth:** Eval loss from training is on the training distribution. Perplexity on a truly held-out split (not seen during training) measures generalization, not memorization.

```python
import torch
from torch.utils.data import DataLoader
from transformers import DataCollatorForLanguageModeling


def compute_perplexity(
    model,
    tokenizer,
    texts: list[str],
    batch_size: int = 4,
    max_length: int = 512,
    label: str = "model",
) -> float:
    """
    Compute perplexity on a list of plain text strings.

    PPL = exp(-1/N * sum log P(x_i | x_{<i}))

    Lower = better. Typical pre-fine-tune PPL on domain text: 15–80.
    After fine-tuning: 8–25 depending on domain specificity.
    """
    model.eval()

    encodings = tokenizer(
        texts,
        return_tensors="pt",
        padding=True,
        truncation=True,
        max_length=max_length,
    )

    total_loss = 0.0
    total_tokens = 0

    with torch.no_grad():
        for i in range(0, len(texts), batch_size):
            batch_input_ids = encodings["input_ids"][i : i + batch_size].to(model.device)
            batch_attn_mask = encodings["attention_mask"][i : i + batch_size].to(model.device)

            # Labels = input_ids; HuggingFace CausalLM shifts internally
            labels = batch_input_ids.clone()
            labels[batch_attn_mask == 0] = -100   # mask padding tokens from loss

            outputs = model(
                input_ids=batch_input_ids,
                attention_mask=batch_attn_mask,
                labels=labels,
            )
            # outputs.loss is mean NLL per non-masked token
            n_tokens = (labels != -100).sum().item()
            total_loss += outputs.loss.item() * n_tokens
            total_tokens += n_tokens

    avg_nll = total_loss / total_tokens
    ppl = math.exp(avg_nll)
    print(f"  [{label}]  NLL: {avg_nll:.4f}  |  PPL: {ppl:.2f}")
    return ppl


def evaluate_before_after(
    base_model, finetuned_model, tokenizer, val_dataset, n_samples: int = 100
):
    section("Phase 5 — Perplexity Evaluation")
    texts = [ex["text"] for ex in val_dataset.select(range(n_samples))]

    ppl_base = compute_perplexity(base_model, tokenizer, texts, label="base model")
    ppl_ft = compute_perplexity(finetuned_model, tokenizer, texts, label="fine-tuned")

    delta = (ppl_base - ppl_ft) / ppl_base * 100
    print(f"\n  PPL reduction     : {delta:.1f}%")
    print(f"  Interpretation    : ", end="")
    if delta > 30:
        print("strong domain adaptation — base model was underfit on this domain")
    elif delta > 10:
        print("moderate domain adaptation — expected for domain-specific QA")
    elif delta > 0:
        print("mild adaptation — dataset may be small or close to pretraining distribution")
    else:
        print("WARNING: PPL increased — possible overfitting or template mismatch")
    return ppl_base, ppl_ft
```

**Checkpoint:** PPL on domain val set drops by at least 15% (medical QA typically: base PPL ~35, post-LoRA ~20). If PPL increased, the fine-tuned model has overfit or the template differs between train and eval.

## Phase 6 — Adapter Merge and Save

**Why this last:** The PEFT model stores $W_0$ and $\Delta W = \frac{\alpha}{r}BA$ separately. Merging folds $\Delta W$ into $W_0$ permanently, producing a standard HuggingFace model that loads without PEFT and runs at full inference speed.

```python
from peft import PeftModel


def merge_and_save(
    base_model_path: str,
    adapter_path: str,
    output_path: str = "./merged-model",
):
    """
    Load base model in float16, load LoRA adapter, merge, save.

    Why float16 here (not 4bit)?
    Merging requires dequantizing W0 and adding ΔW in full precision.
    The merged model is then in float16, ~13 GB on disk for 7B.
    """
    section("Phase 6 — Adapter Merge + Save")

    print("  Loading base model in float16 for merge...")
    base = AutoModelForCausalLM.from_pretrained(
        base_model_path,
        torch_dtype=torch.float16,
        device_map="auto",
    )
    tokenizer = AutoTokenizer.from_pretrained(base_model_path)

    print(f"  Loading adapter from: {adapter_path}")
    model = PeftModel.from_pretrained(base, adapter_path)

    print("  Merging adapter weights into base model...")
    # merge_and_unload() computes W_final = W0 + (alpha/r) * B @ A for each layer
    model = model.merge_and_unload()

    print(f"  Saving merged model to: {output_path}")
    model.save_pretrained(output_path, safe_serialization=True)  # .safetensors
    tokenizer.save_pretrained(output_path)

    # Verify save
    saved_files = list(os.listdir(output_path))
    print(f"  Saved files: {[f for f in saved_files if not f.startswith('.')]}")
    print("\n  Merged model ready for inference. Load with:")
    print(f"  AutoModelForCausalLM.from_pretrained('{output_path}')")


def quick_inference_test(model_path: str, prompt: str) -> str:
    """Sanity-check the merged model with a single prompt."""
    section("Inference Test")
    tokenizer = AutoTokenizer.from_pretrained(model_path)
    model = AutoModelForCausalLM.from_pretrained(
        model_path,
        torch_dtype=torch.float16,
        device_map="auto",
    )
    inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
    with torch.no_grad():
        out = model.generate(
            **inputs,
            max_new_tokens=256,
            temperature=0.7,
            do_sample=True,
            pad_token_id=tokenizer.eos_token_id,
        )
    response = tokenizer.decode(out[0][inputs["input_ids"].shape[1]:], skip_special_tokens=True)
    print(f"  Prompt   : {prompt[:80]}...")
    print(f"  Response : {response[:300]}")
    return response


def main():
    # Phase 1 — load
    model, tokenizer = load_base_model()

    # Phase 2 — LoRA config
    model = build_lora_model(model, r=16, alpha=32)

    # Phase 3 — dataset
    dataset = load_and_format_dataset(
        dataset_name="medalpaca/medical_meadow_medqa",
        template="chatml",
        train_size=5000,
        val_size=500,
    )

    # Phase 5 — baseline PPL (before training)
    section("Phase 5a — Baseline Perplexity (before training)")
    val_texts = [ex["text"] for ex in dataset["validation"].select(range(100))]
    ppl_before = compute_perplexity(model, tokenizer, val_texts, label="before training")

    # Phase 4 — train
    trainer = train(model, tokenizer, dataset, output_dir="./lora-medical")

    # Phase 5b — post-training PPL
    section("Phase 5b — Post-Training Perplexity")
    ppl_after = compute_perplexity(model, tokenizer, val_texts, label="after training")
    print(f"\n  PPL change: {ppl_before:.2f} → {ppl_after:.2f} ({(ppl_before-ppl_after)/ppl_before*100:.1f}% reduction)")

    # Phase 6 — merge
    merge_and_save(
        base_model_path=MODEL_ID,
        adapter_path="./lora-medical",
        output_path="./merged-medical-model",
    )

    quick_inference_test(
        "./merged-medical-model",
        "<|im_start|>user\nWhat are the first-line treatments for type 2 diabetes?<|im_end|>\n<|im_start|>assistant\n",
    )


if __name__ == "__main__":
    main()
```

**Checkpoint:** Merged model directory contains `model.safetensors`, `config.json`, `tokenizer.json`, and `tokenizer_config.json`. It loads without PEFT installed and generates coherent domain-specific text. If merge fails with a dtype error, add `model = model.to(torch.float32)` before `merge_and_unload()`.

## Debugging Guide

| Symptom | Likely Cause | Fix |
|---|---|---|
| CUDA OOM on load | Base model too large for VRAM | Reduce `bnb_4bit_compute_dtype` to `float16`, or lower `per_device_train_batch_size` to 1 |
| Loss NaN after step 1 | `float16` overflow in norm layers | Switch to `bf16=True`, or add `max_grad_norm=0.3` |
| PPL increases after fine-tuning | Template mismatch or LR too high | Verify train/eval use same template; try LR=5e-5 |
| `merge_and_unload` dtype error | Adapter in bfloat16, base in float16 | Cast both to float32 before merge |
| Training loss flat | Adapter not receiving gradients | Check `model.print_trainable_parameters()` shows > 0 trainable |
| Slow training speed | Gradient checkpointing overhead | Expected 30–40% slowdown vs no checkpointing — this is normal |

4. "Perplexity is $\exp(-\frac{1}{N}\sum \log P(x_i|x_{<i}))$ — the exponential of mean NLL. A 25% reduction on held-out domain text indicates meaningful domain shift without catastrophic forgetting of general capabilities."
5. "Merging the adapter with `merge_and_unload()` folds $\Delta W$ into $W_0$ permanently, eliminating all PEFT overhead at inference time. The merged model loads and runs identically to any standard HuggingFace causal LM."
