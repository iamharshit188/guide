# End-to-End Titanic Survival Pipeline — Step-by-Step Guide

> **Module:** 02 — ML Basics · **Difficulty:** Beginner · **Type:** GUIDED · **Time:** ~2.5 hours

## What You'll Build

A complete scikit-learn Pipeline that loads the Titanic CSV, audits and handles missing values, encodes categoricals with `OrdinalEncoder`/`OneHotEncoder`, trains both a `RandomForestClassifier` and `LogisticRegression`, evaluates each with 5-fold stratified cross-validation, and prints a comparison table with AUC-ROC, precision, recall, and F1. A second table shows top-10 feature importances from Random Forest. The pipeline is sklearn-native: all preprocessing runs inside the pipeline to prevent data leakage.

## The Mental Model

A Pipeline is a single object that chains preprocessing and a model — `fit()` on the pipeline applies every step in sequence to training data; `predict()` applies the same learned transformations to test data. This is the correct abstraction because preprocessing parameters (imputer medians, encoder categories, scaler means) must be learned from training data only. If you impute before splitting, test statistics leak into training, and your evaluation is optimistic.

The Titanic dataset has mixed types: numerical columns (`Age`, `Fare`) with missing values, ordinal columns (`Pclass`), nominal columns (`Sex`, `Embarked`) needing one-hot encoding, and irrelevant columns (`Name`, `Ticket`, `Cabin`) to drop. A `ColumnTransformer` inside the pipeline applies the right transformer to the right columns simultaneously.

AUC-ROC measures discrimination: the probability that the model ranks a positive instance above a negative one. It is threshold-independent, making it useful for comparing classifiers. Cross-entropy (log loss) is the training loss for Logistic Regression:

$$\mathcal{L} = -\frac{1}{N}\sum_{i=1}^{N}\left[y_i \log \hat{y}_i + (1 - y_i)\log(1 - \hat{y}_i)\right]$$

$$\text{AUC} = \int_0^1 \text{TPR}\, d(\text{FPR}), \quad \text{Gini} = \frac{1}{N_T} \sum_{t} \Delta I(t) \cdot n_t$$

## Phase 1 — Load Data and Missing Value Audit

**Why this first:** You cannot build a preprocessing pipeline without knowing which columns have missing values, which types they are, and what fraction is missing. The audit determines every decision in Phase 2.

```python
#!/usr/bin/env python3
"""End-to-end Titanic survival prediction pipeline with sklearn."""

from __future__ import annotations
import numpy as np
import pandas as pd
from pathlib import Path
import urllib.request


def section(title: str) -> None:
    width = 64
    print(f"\n{'=' * width}")
    print(f"  {title}")
    print(f"{'=' * width}")


def load_titanic() -> pd.DataFrame:
    """Download Titanic dataset if not cached, return DataFrame."""
    cache = Path("titanic.csv")
    url = (
        "https://raw.githubusercontent.com/datasciencedojo/"
        "datasets/master/titanic.csv"
    )
    if not cache.exists():
        print(f"Downloading Titanic dataset → {cache}")
        urllib.request.urlretrieve(url, cache)
    return pd.read_csv(cache)


def audit_missing(df: pd.DataFrame) -> None:
    """Print per-column missing value counts and percentages."""
    missing = df.isnull().sum()
    missing = missing[missing > 0].sort_values(ascending=False)
    pct = (missing / len(df) * 100).round(2)
    audit = pd.DataFrame({"Missing": missing, "%": pct, "dtype": df[missing.index].dtypes})
    print(audit.to_string())


def main() -> None:
    rng = np.random.default_rng(42)
    
    section("Phase 1 — Load & Missing Value Audit")
    df = load_titanic()
    print(f"Shape: {df.shape}")
    print(f"\nColumn dtypes:\n{df.dtypes.to_string()}")
    print(f"\nSurvival rate: {df['Survived'].mean():.3f}  ({df['Survived'].sum()} survived / {len(df)} total)")
    
    print("\n--- Missing Values ---")
    audit_missing(df)
    
    # Show value counts for categoricals
    print("\nSex value counts:", df["Sex"].value_counts().to_dict())
    print("Embarked value counts:", df["Embarked"].value_counts().to_dict())
    print("Pclass value counts:", df["Pclass"].value_counts().to_dict())
    
    print("\nAge stats:")
    print(df["Age"].describe().round(2).to_string())
```

**Checkpoint:** Output shows 891 rows, 12 columns. `Age` is missing ~177 values (20%), `Cabin` is missing ~687 (77%), `Embarked` is missing 2. These three columns require different strategies.

## Phase 2 — Build the Preprocessing Pipeline

**Why this second:** Pipeline construction is the architectural decision of the project. Every downstream step depends on getting this right. We drop `Cabin` (too sparse), impute `Age` with median, impute `Embarked` with mode, drop `Name`/`Ticket`/`PassengerId`.

```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler, OneHotEncoder, OrdinalEncoder


def build_preprocessor() -> ColumnTransformer:
    """
    Column groups:
    - numerical: Age, Fare, SibSp, Parch → impute median → scale
    - ordinal:   Pclass → impute most_frequent (rare missing) → ordinal encode
    - nominal:   Sex, Embarked → impute most_frequent → one-hot encode
    - drop:      PassengerId, Name, Ticket, Cabin
    """
    numerical_features = ["Age", "Fare", "SibSp", "Parch"]
    ordinal_features   = ["Pclass"]
    nominal_features   = ["Sex", "Embarked"]
    
    numerical_pipeline = Pipeline([
        ("imputer", SimpleImputer(strategy="median")),
        ("scaler", StandardScaler()),
    ])
    
    ordinal_pipeline = Pipeline([
        ("imputer", SimpleImputer(strategy="most_frequent")),
        # Pclass is 1, 2, 3 — higher is lower class, so order matters
        ("encoder", OrdinalEncoder(categories=[[1, 2, 3]])),
    ])
    
    nominal_pipeline = Pipeline([
        ("imputer", SimpleImputer(strategy="most_frequent")),
        # handle_unknown='ignore' silences error on test-time unseen categories
        ("encoder", OneHotEncoder(handle_unknown="ignore", sparse_output=False)),
    ])
    
    preprocessor = ColumnTransformer(
        transformers=[
            ("num", numerical_pipeline, numerical_features),
            ("ord", ordinal_pipeline, ordinal_features),
            ("nom", nominal_pipeline, nominal_features),
        ],
        remainder="drop",      # drop PassengerId, Name, Ticket, Cabin
    )
    
    return preprocessor


def get_feature_names(preprocessor: ColumnTransformer) -> list[str]:
    """Extract feature names after fit, including one-hot encoded names."""
    names: list[str] = []
    for name, transformer, cols in preprocessor.transformers_:
        if name == "num":
            names.extend(cols)
        elif name == "ord":
            names.extend(cols)
        elif name == "nom":
            encoder = transformer.named_steps["encoder"]
            for i, col in enumerate(cols):
                names.extend([f"{col}_{cat}" for cat in encoder.categories_[i]])
    return names


# Add to main():
def phase2_demo(df: pd.DataFrame) -> tuple:
    section("Phase 2 — Build Preprocessing Pipeline")
    
    TARGET = "Survived"
    X = df.drop(columns=[TARGET])
    y = df[TARGET].values
    
    from sklearn.model_selection import train_test_split
    X_train, X_test, y_train, y_test = train_test_split(
        X, y, test_size=0.2, stratify=y, random_state=42
    )
    print(f"Train: {X_train.shape[0]} rows | Test: {X_test.shape[0]} rows")
    print(f"Train survival rate: {y_train.mean():.3f}")
    print(f"Test  survival rate: {y_test.mean():.3f}")
    
    preprocessor = build_preprocessor()
    
    # Fit only on train, transform both
    X_train_proc = preprocessor.fit_transform(X_train, y_train)
    X_test_proc  = preprocessor.transform(X_test)
    
    feature_names = get_feature_names(preprocessor)
    print(f"\nOutput shape after preprocessing: {X_train_proc.shape}")
    print(f"Feature names ({len(feature_names)}): {feature_names}")
    print(f"No NaNs in processed train: {not np.isnan(X_train_proc).any()}")
    
    return X_train, X_test, y_train, y_test, preprocessor, feature_names
```

**Checkpoint:** `X_train_proc.shape == (712, 10)` (4 numerical + 1 ordinal + 5 one-hot from Sex+Embarked). No NaN values in output. `StandardScaler` parameters are fitted on train only — verify by checking `preprocessor.named_transformers_['num'].named_steps['scaler'].mean_` is computed from train data.

## Phase 3 — Train Models with Stratified Cross-Validation

**Why this third:** Cross-validation gives a more reliable performance estimate than a single train/test split — especially important with 891 rows. Stratified folds preserve the class ratio in each fold.

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import StratifiedKFold, cross_validate


def build_models(preprocessor: ColumnTransformer) -> dict[str, Pipeline]:
    """Return dict of named full pipelines (preprocessor + model)."""
    models = {
        "Random Forest": Pipeline([
            ("preprocessor", preprocessor),
            ("classifier", RandomForestClassifier(
                n_estimators=200,
                max_depth=8,
                min_samples_leaf=4,
                random_state=42,
                n_jobs=-1,
            )),
        ]),
        "Logistic Regression": Pipeline([
            ("preprocessor", preprocessor),
            ("classifier", LogisticRegression(
                C=1.0,
                solver="lbfgs",
                max_iter=1000,
                random_state=42,
            )),
        ]),
    }
    return models


def cross_validate_models(
    models: dict[str, Pipeline],
    X: pd.DataFrame,
    y: np.ndarray,
    n_splits: int = 5,
) -> dict[str, dict[str, float]]:
    """
    Run stratified k-fold CV for each model.
    Returns dict: model_name → {metric: mean_score}.
    """
    cv = StratifiedKFold(n_splits=n_splits, shuffle=True, random_state=42)
    
    scoring = {
        "roc_auc":  "roc_auc",
        "precision": "precision",
        "recall":    "recall",
        "f1":        "f1",
    }
    
    results: dict[str, dict[str, float]] = {}
    
    for name, pipeline in models.items():
        print(f"  Cross-validating: {name} ({n_splits} folds)...")
        cv_scores = cross_validate(
            pipeline, X, y,
            cv=cv,
            scoring=scoring,
            return_train_score=False,
            n_jobs=-1,
        )
        results[name] = {
            metric: cv_scores[f"test_{metric}"].mean()
            for metric in scoring
        }
        for metric, score in results[name].items():
            std = cv_scores[f"test_{metric}"].std()
            print(f"    {metric:12s}: {score:.4f} ± {std:.4f}")
    
    return results
```

**Checkpoint:** Cross-validation prints 5 folds per model. Random Forest AUC should be above 0.84; Logistic Regression AUC should be above 0.82. If AUC is below 0.75 for either model, the preprocessor has a leak (fitted on full data instead of train folds inside CV — which doesn't happen when using Pipelines correctly).

## Phase 4 — Evaluation Table and Feature Importances

**Why this last:** Comparing models on a held-out test set (not the CV folds used to select models) and understanding *which* features drive predictions are the final deliverables.

```python
from sklearn.metrics import (
    roc_auc_score, precision_score, recall_score,
    f1_score, confusion_matrix
)


def print_comparison_table(
    models: dict[str, Pipeline],
    X_train: pd.DataFrame,
    X_test: pd.DataFrame,
    y_train: np.ndarray,
    y_test: np.ndarray,
    cv_results: dict[str, dict[str, float]],
) -> None:
    section("Phase 4 — Model Comparison Table")
    
    header = f"{'Model':22} | {'AUC':>6} | {'Prec':>6} | {'Rec':>6} | {'F1':>6} | Source"
    print(header)
    print("-" * len(header))
    
    for name, pipeline in models.items():
        pipeline.fit(X_train, y_train)
        y_pred = pipeline.predict(X_test)
        y_prob = pipeline.predict_proba(X_test)[:, 1]
        
        auc   = roc_auc_score(y_test, y_prob)
        prec  = precision_score(y_test, y_pred, zero_division=0)
        rec   = recall_score(y_test, y_pred, zero_division=0)
        f1    = f1_score(y_test, y_pred, zero_division=0)
        
        print(f"{name:22} | {auc:.4f} | {prec:.4f} | {rec:.4f} | {f1:.4f} | test set")
        
        cv = cv_results[name]
        print(
            f"{'(cross-val mean)':22} | {cv['roc_auc']:.4f} | "
            f"{cv['precision']:.4f} | {cv['recall']:.4f} | {cv['f1']:.4f} | 5-fold CV"
        )
        
        cm = confusion_matrix(y_test, y_pred)
        print(f"  Confusion matrix: TN={cm[0,0]}, FP={cm[0,1]}, FN={cm[1,0]}, TP={cm[1,1]}")
        print()


def print_feature_importances(
    pipeline: Pipeline,
    feature_names: list[str],
    top_n: int = 10,
) -> None:
    section("Feature Importances (Random Forest)")
    
    rf = pipeline.named_steps["classifier"]
    importances = rf.feature_importances_
    
    # Sort descending
    indices = np.argsort(importances)[::-1]
    
    print(f"{'Rank':>4} | {'Feature':22} | {'Importance':>10} | Bar")
    print("-" * 60)
    
    max_imp = importances[indices[0]]
    for rank, idx in enumerate(indices[:top_n], start=1):
        name = feature_names[idx] if idx < len(feature_names) else f"feat_{idx}"
        imp  = importances[idx]
        bar_len = int(imp / max_imp * 30)
        bar  = "█" * bar_len
        print(f"{rank:>4} | {name:22} | {imp:>10.4f} | {bar}")


# ── Complete main() ────────────────────────────────────────────────────────────

def main() -> None:
    rng = np.random.default_rng(42)
    
    # Phase 1
    section("Phase 1 — Load & Missing Value Audit")
    df = load_titanic()
    print(f"Shape: {df.shape}")
    audit_missing(df)
    
    # Phase 2
    section("Phase 2 — Build Preprocessing Pipeline")
    TARGET = "Survived"
    X = df.drop(columns=[TARGET])
    y = df[TARGET].values
    
    from sklearn.model_selection import train_test_split
    X_train, X_test, y_train, y_test = train_test_split(
        X, y, test_size=0.2, stratify=y, random_state=42
    )
    preprocessor = build_preprocessor()
    feature_names = get_feature_names(preprocessor.fit(X_train))
    print(f"Feature count: {len(feature_names)}")
    
    # Phase 3
    section("Phase 3 — Stratified Cross-Validation (5-fold)")
    models = build_models(build_preprocessor())  # fresh preprocessor per model
    cv_results = cross_validate_models(models, X, y)
    
    # Phase 4
    # Rebuild models with fresh preprocessors for final fit
    final_models = build_models(build_preprocessor())
    print_comparison_table(final_models, X_train, X_test, y_train, y_test, cv_results)
    
    # Feature importances from the Random Forest final model
    rf_pipeline = final_models["Random Forest"]
    rf_pipeline.fit(X_train, y_train)
    proc_fitted = rf_pipeline.named_steps["preprocessor"]
    feature_names = get_feature_names(proc_fitted)
    print_feature_importances(rf_pipeline, feature_names)


if __name__ == "__main__":
    main()
```

**Checkpoint:** Comparison table shows both models with AUC above 0.80. Feature importance table has 10 rows. `Sex_male` and `Sex_female` should appear in the top 5 — sex is the strongest predictor of Titanic survival. `Age` should also appear in top 5. If `Pclass` is not in top 5, verify it was not dropped.

## Debugging Guide

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| `ValueError: could not convert string to float` during fit | Raw DataFrame columns passed to model without preprocessor | Verify pipeline has `preprocessor` step before `classifier` |
| AUC from cross-val is 0.50 (random) | `cross_validate` called with raw X, not a Pipeline — imputer fitted on all data leaks into folds | Wrap preprocessor inside the Pipeline object before passing to `cross_validate` |
| `AttributeError: 'ColumnTransformer' has no attribute 'categories_'` in `get_feature_names` | Calling `get_feature_names` before `fit` | Call only after `preprocessor.fit(X_train)` |
| Feature importances all equal (1/n_features) | Random Forest `max_depth=None` with very few trees — all trees identical due to no randomness | Add `min_samples_leaf=4` and `n_estimators >= 100` |
| `Cabin` column causes shape mismatch | `Cabin` was not listed in any transformer and `remainder` is not `'drop'` | Set `remainder='drop'` in `ColumnTransformer` |
| `Embarked` imputation with mode fails | `SimpleImputer(strategy='most_frequent')` on a column with all nulls returns empty | Assert `df['Embarked'].isnull().sum() < len(df)` before building pipeline |

## What This Project Teaches You to Say in an Interview

1. "A sklearn Pipeline prevents data leakage because `fit_transform` on the pipeline only learns preprocessing parameters (imputer medians, scaler means) from the training fold — never from validation or test data. If you impute before cross-validation, test statistics pollute training estimates and your AUC is artificially inflated."
2. "AUC-ROC measures the probability that a randomly chosen positive is ranked higher than a randomly chosen negative. It is threshold-independent, making it useful for comparing classifiers without committing to a decision threshold. A classifier that always predicts the majority class achieves AUC = 0.5, not 0.8."
3. "Random Forest Gini importance measures the weighted average decrease in Gini impurity across all splits using that feature. It is biased toward high-cardinality features. For unbiased importance, use permutation importance or SHAP, which measure the actual impact on a held-out metric."
4. "Stratified k-fold ensures each fold has the same class ratio as the full dataset. With 38% survival rate, an unstratified fold could randomly get 20% or 60% survival — making loss values incomparable across folds and inflating variance."
5. "`OrdinalEncoder` for `Pclass` encodes 1st class < 2nd class < 3rd class numerically. `OneHotEncoder` for `Sex` creates separate binary features — the model sees no ordinal relationship between 'male' and 'female'. Using `OrdinalEncoder` on nominal features imposes a false ordering that tree models may overfit to."
