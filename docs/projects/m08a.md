# Personal Document Q&A System — Step-by-Step Guide

> **Module:** 08 — RAG & Vector Search · **Difficulty:** Advanced · **Type:** GUIDED · **Time:** ~4 hours

## What You'll Build

A CLI RAG pipeline that ingests PDF and TXT files from a local directory, chunks them with token-aware recursive splitting, embeds chunks with `sentence-transformers`, and stores them in a persistent ChromaDB collection. At query time it retrieves top-5 chunks via cosine similarity, re-ranks with MMR for diversity, injects numbered context blocks into a prompt, calls an LLM (OpenAI GPT-4o or local Ollama), and returns an answer with inline source citations showing filename and page number. Re-indexing is triggered automatically when documents are newer than the last index timestamp.

## The Mental Model

RAG solves the context window problem: an LLM has a finite context window (4K–128K tokens) but a user's document collection can be millions of tokens. Instead of truncating, you retrieve only the $k$ most relevant chunks — small enough to fit in context, large enough to contain the answer. The embedding model maps both query and chunk into the same semantic vector space where cosine similarity measures relevance. The LLM is then used as a reader, not a knowledge store: it synthesizes an answer from retrieved text rather than recalling facts from parameters.

The chunking strategy is load-bearing. Chunks too small (50 tokens) lose context — a sentence about "the defendant" has no meaning without the surrounding paragraph establishing who the defendant is. Chunks too large (1000 tokens) dilute relevance: if the answer is in one sentence but the chunk contains 20 sentences, the embedding averages over all of them and the cosine score drops. The 512-token chunk with 50-token overlap is a practical baseline: overlap ensures sentences at chunk boundaries appear in at least one chunk in full context.

MMR (Maximal Marginal Relevance) trades pure relevance for diversity. If the top-5 chunks by cosine similarity are all from the same page, the LLM sees the same information five ways and misses potentially relevant content from other sections. MMR selects chunk $i$ at each step to maximize $\lambda \cdot \text{sim}(q, c_i) - (1-\lambda) \cdot \max_{j \in \text{selected}} \text{sim}(c_j, c_i)$, balancing relevance to query against redundancy with already-selected chunks.

## Phase 1 — Document Loader

**Why this first:** Downstream chunking, embedding, and retrieval are all downstream of raw text extraction. Get clean (text, metadata) pairs before writing any other logic.

```python
#!/usr/bin/env python3
"""Personal Document Q&A System — RAG pipeline with ChromaDB."""

from __future__ import annotations
import os
import sys
import json
import time
import hashlib
import argparse
from pathlib import Path
from typing import Any

import fitz          # PyMuPDF
import tiktoken


def section(title: str) -> None:
    width = 64
    print(f"\n{'=' * width}")
    print(f"  {title}")
    print(f"{'=' * width}")


# ── Document Loader ───────────────────────────────────────────────────────────

def load_pdf(path: Path) -> list[dict[str, Any]]:
    """
    Extract text page-by-page from a PDF.
    Returns list of {text, source, page} dicts.
    PyMuPDF preserves paragraph boundaries better than pdfminer
    because it respects block-level layout detection.
    """
    doc = fitz.open(str(path))
    pages = []
    for page_num, page in enumerate(doc, start=1):
        text = page.get_text("text").strip()
        if not text:          # skip blank/image-only pages
            continue
        pages.append({
            "text": text,
            "source": path.name,
            "page": page_num,
        })
    doc.close()
    return pages


def load_txt(path: Path) -> list[dict[str, Any]]:
    """
    Load plain text file as a single 'page'.
    Assigns page=1 for metadata consistency.
    """
    text = path.read_text(encoding="utf-8", errors="replace").strip()
    if not text:
        return []
    return [{"text": text, "source": path.name, "page": 1}]


def load_documents(doc_dir: str) -> list[dict[str, Any]]:
    """
    Walk doc_dir for .pdf and .txt files.
    Returns flat list of (text, metadata) dicts.
    """
    section("Phase 1 — Document Loading")
    base = Path(doc_dir)
    if not base.exists():
        raise FileNotFoundError(f"Document directory not found: {doc_dir}")

    pages: list[dict[str, Any]] = []
    for path in sorted(base.iterdir()):
        if path.suffix.lower() == ".pdf":
            result = load_pdf(path)
            print(f"  PDF  {path.name}: {len(result)} pages")
            pages.extend(result)
        elif path.suffix.lower() == ".txt":
            result = load_txt(path)
            print(f"  TXT  {path.name}: {len(result)} pages")
            pages.extend(result)

    print(f"\n  Total page segments: {len(pages)}")
    if not pages:
        raise ValueError(f"No PDF or TXT documents found in {doc_dir}")
    return pages
```

**Checkpoint:** `load_documents("./docs")` prints each file name with page count and returns a list where every dict has keys `text`, `source`, `page`. PDF pages with only images return empty text and are skipped silently. A 10-page PDF yields at most 10 dicts, possibly fewer if some pages are blank.

## Phase 2 — Recursive Chunker

**Why this second:** Chunks are the unit of retrieval. The chunking strategy determines whether the LLM receives complete, coherent context or fragmented sentences.

```python
# ── Recursive Chunker ─────────────────────────────────────────────────────────

TOKENIZER = tiktoken.get_encoding("cl100k_base")    # GPT-4 tokenizer


def count_tokens(text: str) -> int:
    return len(TOKENIZER.encode(text))


def recursive_split(
    text: str,
    chunk_size: int = 512,
    overlap: int = 50,
) -> list[str]:
    """
    Split text hierarchically: paragraphs → sentences → characters.
    Each resulting chunk is at most chunk_size tokens.
    Adjacent chunks share `overlap` tokens at boundaries.

    Why recursive: splitting on paragraphs respects semantic boundaries
    but some paragraphs exceed chunk_size. Sentence splitting is the
    fallback; character splitting is the last resort for unstructured text.
    """
    separators = ["\n\n", "\n", ". ", " ", ""]
    return _split(text, separators, chunk_size, overlap)


def _split(
    text: str,
    separators: list[str],
    chunk_size: int,
    overlap: int,
) -> list[str]:
    if count_tokens(text) <= chunk_size:
        return [text.strip()] if text.strip() else []

    sep = separators[0]
    next_seps = separators[1:]

    if not sep:
        # Character-level fallback: slice by token count
        tokens = TOKENIZER.encode(text)
        chunks = []
        start = 0
        while start < len(tokens):
            end = min(start + chunk_size, len(tokens))
            chunk_text = TOKENIZER.decode(tokens[start:end])
            chunks.append(chunk_text)
            start += chunk_size - overlap
        return chunks

    parts = text.split(sep)
    chunks: list[str] = []
    current = ""

    for part in parts:
        candidate = (current + sep + part).strip() if current else part
        if count_tokens(candidate) <= chunk_size:
            current = candidate
        else:
            if current:
                chunks.append(current.strip())
            # Part itself may exceed chunk_size — recurse with finer separator
            if count_tokens(part) > chunk_size and next_seps:
                sub_chunks = _split(part, next_seps, chunk_size, overlap)
                chunks.extend(sub_chunks)
                current = sub_chunks[-1] if sub_chunks else ""
            else:
                current = part

    if current:
        chunks.append(current.strip())

    # Apply overlap: prepend tail of previous chunk to next chunk
    if overlap > 0 and len(chunks) > 1:
        overlapped: list[str] = [chunks[0]]
        for i in range(1, len(chunks)):
            prev_tokens = TOKENIZER.encode(chunks[i - 1])[-overlap:]
            overlap_text = TOKENIZER.decode(prev_tokens)
            overlapped.append(overlap_text + " " + chunks[i])
        return overlapped

    return chunks


def chunk_documents(
    pages: list[dict[str, Any]],
    chunk_size: int = 512,
    overlap: int = 50,
) -> list[dict[str, Any]]:
    """
    Chunk each page's text and propagate source metadata.
    Returns list of {text, source, page, chunk_id, token_count} dicts.
    """
    section("Phase 2 — Chunking")
    all_chunks: list[dict[str, Any]] = []

    for page in pages:
        splits = recursive_split(page["text"], chunk_size, overlap)
        for i, chunk_text in enumerate(splits):
            token_count = count_tokens(chunk_text)
            all_chunks.append({
                "text": chunk_text,
                "source": page["source"],
                "page": page["page"],
                "chunk_id": f"{page['source']}__p{page['page']}__c{i}",
                "token_count": token_count,
            })

    total_tokens = sum(c["token_count"] for c in all_chunks)
    avg_tokens = total_tokens / len(all_chunks) if all_chunks else 0
    print(f"  Total chunks: {len(all_chunks)}")
    print(f"  Avg tokens per chunk: {avg_tokens:.0f}")
    print(f"  Total tokens: {total_tokens:,}")
    return all_chunks
```

**Checkpoint:** A 10-page PDF with ~2000 tokens per page produces roughly 40–50 chunks at `chunk_size=512`. Every chunk dict has all 5 keys. `max(c["token_count"] for c in chunks) <= 512` — no chunk exceeds the target size. `sum(1 for c in chunks if c["token_count"] < 10) == 0` — no near-empty chunks (whitespace-only pages were filtered in Phase 1).

## Phase 3 — Embedding and ChromaDB Upsert

**Why this third:** Embeddings and the vector store are the retrieval backbone. Getting the upsert right — with content-based IDs to support re-indexing — makes the pipeline idempotent.

```python
# ── Embedding + ChromaDB ──────────────────────────────────────────────────────

import chromadb
from sentence_transformers import SentenceTransformer


def get_or_create_collection(
    persist_dir: str = "./chroma_db",
    collection_name: str = "documents",
) -> tuple[chromadb.Collection, chromadb.PersistentClient]:
    """
    Create or load a persistent ChromaDB collection.
    cosine distance = 1 - cosine_similarity; retrieval uses cosine metric.
    """
    client = chromadb.PersistentClient(path=persist_dir)
    collection = client.get_or_create_collection(
        name=collection_name,
        metadata={"hnsw:space": "cosine"},
    )
    return collection, client


def hash_text(text: str) -> str:
    """Deterministic ID: SHA-256 hex of UTF-8 text, first 16 chars."""
    return hashlib.sha256(text.encode("utf-8")).hexdigest()[:16]


def embed_and_upsert(
    chunks: list[dict[str, Any]],
    collection: chromadb.Collection,
    model_name: str = "all-MiniLM-L6-v2",
    batch_size: int = 64,
) -> None:
    """
    Embed chunks and upsert into ChromaDB.
    Uses hash(text) as the document ID — existing chunks with identical
    content are not re-embedded (ChromaDB upsert is idempotent by ID).

    all-MiniLM-L6-v2: 384-dim, ~22M params, runs on CPU at ~2000 chunks/sec.
    For higher quality at the cost of speed use 'all-mpnet-base-v2' (768-dim).
    """
    section("Phase 3 — Embedding and ChromaDB Upsert")

    model = SentenceTransformer(model_name)
    print(f"  Embedding model: {model_name}")
    print(f"  Embedding dimension: {model.get_sentence_embedding_dimension()}")

    existing_ids = set(collection.get(include=[])["ids"])
    print(f"  Existing docs in collection: {len(existing_ids)}")

    # Filter chunks already in the collection
    new_chunks = [c for c in chunks if hash_text(c["text"]) not in existing_ids]
    print(f"  New chunks to embed: {len(new_chunks)}")

    if not new_chunks:
        print("  Nothing to embed — collection is up to date.")
        return

    for batch_start in range(0, len(new_chunks), batch_size):
        batch = new_chunks[batch_start : batch_start + batch_size]
        texts = [c["text"] for c in batch]
        ids = [hash_text(c["text"]) for c in batch]
        metadatas = [
            {
                "source": c["source"],
                "page": c["page"],
                "chunk_id": c["chunk_id"],
                "token_count": c["token_count"],
            }
            for c in batch
        ]

        embeddings = model.encode(texts, show_progress_bar=False).tolist()

        collection.upsert(
            ids=ids,
            embeddings=embeddings,
            documents=texts,
            metadatas=metadatas,
        )
        done = min(batch_start + batch_size, len(new_chunks))
        print(f"  Upserted {done}/{len(new_chunks)} chunks", end="\r")

    print(f"\n  Upsert complete. Collection size: {collection.count()}")
```

**Checkpoint:** After the first run, `collection.count()` equals the number of chunks. After running again on the same documents, `new_chunks` is empty and "Nothing to embed" prints. Adding a new document causes only that document's chunks to be embedded. Verify with `collection.peek(1)` — the first result should have `embeddings`, `documents`, and `metadatas` populated.

## Phase 4 — Retrieval with MMR Re-Ranking

**Why this fourth:** Cosine retrieval finds the most similar chunks, but diversity re-ranking ensures the LLM receives a spread of information rather than near-duplicate paragraphs.

```python
# ── Retrieval ────────────────────────────────────────────────────────────────

import numpy as np
from sentence_transformers import SentenceTransformer


def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b) + 1e-10))


def mmr_rerank(
    query_embedding: np.ndarray,
    candidate_embeddings: list[np.ndarray],
    candidates: list[dict[str, Any]],
    k: int = 5,
    lambda_param: float = 0.5,
) -> list[dict[str, Any]]:
    """
    Maximal Marginal Relevance selection.

    At each step, select the candidate c_i that maximizes:
        lambda * sim(query, c_i) - (1 - lambda) * max_{c_j in selected} sim(c_j, c_i)

    lambda=1.0  → pure cosine ranking (no diversity)
    lambda=0.5  → balanced relevance and diversity
    lambda=0.0  → pure diversity (ignore query)
    """
    selected_indices: list[int] = []
    selected_embeddings: list[np.ndarray] = []

    for _ in range(min(k, len(candidates))):
        best_idx = -1
        best_score = float("-inf")

        for i, (emb, _) in enumerate(zip(candidate_embeddings, candidates)):
            if i in selected_indices:
                continue

            relevance = cosine_similarity(query_embedding, emb)

            if selected_embeddings:
                redundancy = max(
                    cosine_similarity(emb, sel_emb)
                    for sel_emb in selected_embeddings
                )
            else:
                redundancy = 0.0

            score = lambda_param * relevance - (1 - lambda_param) * redundancy

            if score > best_score:
                best_score = score
                best_idx = i

        if best_idx == -1:
            break

        selected_indices.append(best_idx)
        selected_embeddings.append(candidate_embeddings[best_idx])

    return [candidates[i] for i in selected_indices]


def retrieve(
    query: str,
    collection: chromadb.Collection,
    embed_model: SentenceTransformer,
    n_candidates: int = 20,
    k: int = 5,
    lambda_param: float = 0.5,
) -> list[dict[str, Any]]:
    """
    Two-stage retrieval:
    1. Cosine search top-n_candidates from ChromaDB.
    2. MMR re-rank to select top-k diverse results.
    Returns list of {text, source, page, score} dicts.
    """
    query_emb = embed_model.encode(query)

    results = collection.query(
        query_embeddings=[query_emb.tolist()],
        n_results=min(n_candidates, collection.count()),
        include=["documents", "metadatas", "embeddings", "distances"],
    )

    candidates = []
    candidate_embeddings = []

    docs = results["documents"][0]
    metas = results["metadatas"][0]
    dists = results["distances"][0]
    embs = results["embeddings"][0]

    for doc, meta, dist, emb in zip(docs, metas, dists, embs):
        candidates.append({
            "text": doc,
            "source": meta["source"],
            "page": meta["page"],
            "score": 1.0 - dist,   # ChromaDB cosine distance → similarity
        })
        candidate_embeddings.append(np.array(emb))

    reranked = mmr_rerank(query_emb, candidate_embeddings, candidates, k, lambda_param)
    return reranked
```

**Checkpoint:** `retrieve("What is the capital of France?", ...)` returns exactly `k=5` dicts. Each dict has `source` and `page` populated. Verify diversity: if the document collection has multiple files, the top-5 results should include chunks from at least 2 different sources (not all from one file). Print `[r["source"] for r in results]` to confirm.

## Phase 5 — Prompt Construction, LLM Call, and Citation Extraction

**Why this last:** The context block construction and citation format determine whether the LLM's answer is grounded and auditable.

```python
# ── LLM Integration ───────────────────────────────────────────────────────────

import re


SYSTEM_PROMPT = """You are a document assistant. Answer questions using ONLY the provided context.
Each context block is labeled [N] with its source file and page number.
When you use information from a block, cite it inline as [N].
If the answer is not in the context, say "The provided documents do not contain this information."
Do not use prior knowledge — only the context blocks below."""


def build_context_block(chunks: list[dict[str, Any]]) -> str:
    """
    Format retrieved chunks as a numbered context block.
    [1] source.pdf (page 3)
    <text>
    """
    lines = []
    for i, chunk in enumerate(chunks, start=1):
        lines.append(f"[{i}] {chunk['source']} (page {chunk['page']})")
        lines.append(chunk["text"])
        lines.append("")
    return "\n".join(lines)


def call_openai(
    system: str,
    context: str,
    question: str,
    model: str = "gpt-4o-mini",
) -> str:
    """Call OpenAI API. Requires OPENAI_API_KEY environment variable."""
    from openai import OpenAI
    client = OpenAI()
    response = client.chat.completions.create(
        model=model,
        messages=[
            {"role": "system", "content": system},
            {"role": "user", "content": f"Context:\n{context}\n\nQuestion: {question}"},
        ],
        temperature=0.1,      # low temperature for factual grounding
        max_tokens=1024,
    )
    return response.choices[0].message.content


def call_ollama(
    system: str,
    context: str,
    question: str,
    model: str = "llama3.2",
) -> str:
    """Call local Ollama server. No API key required."""
    import urllib.request
    payload = {
        "model": model,
        "messages": [
            {"role": "system", "content": system},
            {"role": "user", "content": f"Context:\n{context}\n\nQuestion: {question}"},
        ],
        "stream": False,
    }
    data = json.dumps(payload).encode("utf-8")
    req = urllib.request.Request(
        "http://localhost:11434/api/chat",
        data=data,
        headers={"Content-Type": "application/json"},
    )
    with urllib.request.urlopen(req, timeout=60) as resp:
        result = json.loads(resp.read())
    return result["message"]["content"]


def extract_citations(answer: str, chunks: list[dict[str, Any]]) -> list[dict[str, Any]]:
    """
    Parse [N] citation markers from the LLM answer.
    Returns list of cited chunks with source and page.
    """
    cited_indices = set(int(m) for m in re.findall(r"\[(\d+)\]", answer))
    citations = []
    for idx in sorted(cited_indices):
        if 1 <= idx <= len(chunks):
            chunk = chunks[idx - 1]
            citations.append({"ref": idx, "source": chunk["source"], "page": chunk["page"]})
    return citations


def answer_question(
    question: str,
    collection: chromadb.Collection,
    embed_model: SentenceTransformer,
    use_ollama: bool = False,
) -> dict[str, Any]:
    """End-to-end RAG pipeline for a single question."""
    section(f"Query: {question}")

    # Retrieve
    chunks = retrieve(question, collection, embed_model)
    print(f"  Retrieved {len(chunks)} chunks")
    for i, c in enumerate(chunks, 1):
        print(f"    [{i}] {c['source']} p{c['page']}  score={c['score']:.3f}")

    context = build_context_block(chunks)

    # Generate
    t0 = time.time()
    if use_ollama:
        answer = call_ollama(SYSTEM_PROMPT, context, question)
    else:
        answer = call_openai(SYSTEM_PROMPT, context, question)
    elapsed = time.time() - t0

    citations = extract_citations(answer, chunks)

    return {
        "question": question,
        "answer": answer,
        "citations": citations,
        "chunks_used": len(chunks),
        "latency_sec": round(elapsed, 2),
    }


# ── Main ──────────────────────────────────────────────────────────────────────

def main() -> None:
    parser = argparse.ArgumentParser(description="Document Q&A RAG pipeline")
    parser.add_argument("--docs", default="./documents", help="Directory with PDF/TXT files")
    parser.add_argument("--db", default="./chroma_db", help="ChromaDB persistence path")
    parser.add_argument("--reindex", action="store_true", help="Force re-embedding all docs")
    parser.add_argument("--ollama", action="store_true", help="Use Ollama instead of OpenAI")
    parser.add_argument("--query", type=str, default=None, help="Run a single query and exit")
    args = parser.parse_args()

    # Load and chunk documents
    pages = load_documents(args.docs)
    chunks = chunk_documents(pages)

    # Set up vector store
    collection, _ = get_or_create_collection(args.db)

    if args.reindex:
        section("Re-indexing: clearing existing collection")
        collection.delete(ids=collection.get(include=[])["ids"])

    embed_model = SentenceTransformer("all-MiniLM-L6-v2")
    embed_and_upsert(chunks, collection, batch_size=64)

    # Interactive Q&A loop or single query
    if args.query:
        result = answer_question(args.query, collection, embed_model, args.ollama)
        print(f"\nAnswer:\n{result['answer']}")
        print(f"\nCitations:")
        for c in result["citations"]:
            print(f"  [{c['ref']}] {c['source']} — page {c['page']}")
        print(f"\nLatency: {result['latency_sec']}s")
    else:
        section("Interactive Q&A (type 'exit' to quit)")
        while True:
            try:
                question = input("\nQuestion: ").strip()
            except (EOFError, KeyboardInterrupt):
                break
            if question.lower() in ("exit", "quit", "q"):
                break
            if not question:
                continue
            result = answer_question(question, collection, embed_model, args.ollama)
            print(f"\n{result['answer']}")
            if result["citations"]:
                print("\nSources:")
                for c in result["citations"]:
                    print(f"  [{c['ref']}] {c['source']} — page {c['page']}")


if __name__ == "__main__":
    main()
```

**Checkpoint:** Run `python rag_qa.py --docs ./documents --query "What is discussed in chapter 1?"`. The output shows retrieved chunks with source/page labels, followed by the LLM answer with inline `[N]` citation markers, followed by a citation list. Latency with OpenAI GPT-4o-mini should be under 3 seconds. With Ollama on CPU, expect 10–30 seconds. The citation list should reference only chunk numbers that actually appear in the answer text.

## Debugging Guide

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| `collection.count()` never increases | ChromaDB path not persistent; using in-memory client | Use `chromadb.PersistentClient(path="./chroma_db")` not `chromadb.Client()` |
| All retrieved chunks come from one file | No MMR — cosine retrieval returns top-k from most similar section | Verify `mmr_rerank` is called; increase `lambda_param` toward 1.0 for less diversity if that's the issue, or decrease toward 0.0 for more |
| LLM answers "I don't have information" despite relevant docs | Chunks too large — relevant sentence diluted; or wrong embedding model | Lower `chunk_size` to 256; verify `all-MiniLM-L6-v2` is actually loaded (not falling back to a different model) |
| `UnicodeDecodeError` on PDF loading | PyMuPDF returns raw bytes for some PDFs with non-UTF-8 encodings | Add `errors="replace"` to text extraction; use `page.get_text("rawdict")` for structured extraction |
| Answer contains hallucinated page numbers | LLM ignored system prompt and fabricated citations | Add explicit instruction: "Only cite [N] where N appears in the numbered context blocks above. Never cite a number you did not see." |

## What This Project Teaches You to Say in an Interview

1. "Chunk size and overlap are the most impactful hyperparameters in a RAG pipeline. A 512-token chunk with 50-token overlap means every sentence within 50 tokens of a chunk boundary appears in at least one chunk with sufficient surrounding context. Without overlap, a sentence split across a boundary loses its context in both halves."
2. "MMR re-ranking addresses the retrieval clustering problem: the top-k by cosine similarity often clusters around one section of the document. MMR trades off similarity to the query against similarity to already-selected chunks, parameterized by $\lambda$. At $\lambda = 0.5$, half the score comes from relevance, half from novelty."
3. "Content-addressed IDs (hash of chunk text) make upsert idempotent: running the pipeline twice on unchanged documents produces no re-embedding work. If a document changes, its chunks get new hashes and are inserted as new records — old records remain until explicitly deleted."
4. "The system prompt must prohibit prior knowledge. Without it, GPT-4 will answer from parametric memory even when the context is empty or wrong, making citations meaningless. Low temperature (0.1) further reduces hallucination by making the model less likely to generate plausible-sounding fabrications."
5. "ChromaDB stores embeddings in HNSW (Hierarchical Navigable Small World) index, giving approximate nearest-neighbor search at $O(\log N)$ per query. For collections under 100K chunks, exact search is also viable; ChromaDB switches automatically based on collection size."
