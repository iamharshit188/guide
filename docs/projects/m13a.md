# Image-Text Hybrid Search Engine — Step-by-Step Guide

> **Module:** 13 · **Difficulty:** Expert · **Type:** GUIDED · **Time:** ~4 hours

## What You'll Build

A cross-modal search engine using CLIP (ViT-B/32) that lets you query an image collection with plain text ("a dog on a beach") or with another image, retrieving the top-10 most semantically similar results by cosine similarity. All images and queries are embedded into the same 512-dimensional joint space; retrieval is handled by a FAISS flat index. Results print as an ASCII table showing filename, cosine score, and any available caption metadata.

## The Mental Model

CLIP trains a vision encoder and a text encoder jointly on 400M image-text pairs using contrastive learning. The loss pushes matching image-text pairs together and pulls non-matching pairs apart in the same 512-dim embedding space:

$$\mathcal{L} = -\frac{1}{N}\sum_{i=1}^{N} \log \frac{\exp(\text{sim}(v_i, t_i)/\tau)}{\sum_{j=1}^{N} \exp(\text{sim}(v_i, t_j)/\tau)}$$

where $v_i \in \mathbb{R}^{512}$ is the image embedding, $t_i \in \mathbb{R}^{512}$ is the paired text embedding, and $\tau$ is a learned temperature. After training, both encoders map their inputs to the same metric space. Crucially, if you L2-normalize all embeddings to the unit hypersphere, cosine similarity reduces to a dot product:

$$\text{cos}(u, v) = u \cdot v \quad \text{when} \quad \|u\|_2 = \|v\|_2 = 1$$

This means FAISS IndexFlatIP (inner product) on normalized vectors is exactly cosine search — fast, exact, no approximation needed for collections under ~1M items.

## Phase 2 — Load CLIP Model

**Why this first:** The model must be loaded before any embedding can happen. This phase also detects GPU availability so all later phases inherit the correct device.

```python
# ── Phase 2: Load CLIP model ───────────────────────────────────────────────────
from transformers import CLIPModel, CLIPProcessor


def load_clip(model_id: str = MODEL_ID) -> tuple[CLIPModel, CLIPProcessor, torch.device]:
    """Load CLIP model + processor. Returns (model, processor, device)."""
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    print(f"  Device: {device}")

    print(f"  Loading {model_id} ...")
    processor = CLIPProcessor.from_pretrained(model_id)
    model     = CLIPModel.from_pretrained(model_id).to(device)
    model.eval()

    # Count parameters
    n_params = sum(p.numel() for p in model.parameters()) / 1e6
    print(f"  Parameters: {n_params:.1f}M")
    return model, processor, device


def l2_normalize(x: np.ndarray) -> np.ndarray:
    """Row-wise L2 normalization. Input shape: (N, D) or (D,)."""
    if x.ndim == 1:
        return x / (np.linalg.norm(x) + 1e-9)
    norms = np.linalg.norm(x, axis=1, keepdims=True)
    return x / (norms + 1e-9)
```

**Checkpoint:** Model loads in under 60 seconds on CPU, under 10 seconds on GPU. The processor bundles the image preprocessor (resize → center crop → normalize) and the text tokenizer (BPE, max 77 tokens).

## Phase 3 — Batch-Process Images and Build FAISS Index

**Why this first:** The FAISS index is the core data structure. All search queries are meaningless until the index is populated.

```python
# ── Phase 3: Build image embedding index ──────────────────────────────────────
import faiss


def embed_images(
    image_paths: list[Path],
    model: CLIPModel,
    processor: CLIPProcessor,
    device: torch.device,
    batch_size: int = BATCH_SIZE,
) -> np.ndarray:
    """
    Batch-embed images using CLIP vision encoder.
    Returns float32 array of shape (N, 512), L2-normalized.
    """
    all_embeddings: list[np.ndarray] = []

    for start in range(0, len(image_paths), batch_size):
        batch_paths = image_paths[start : start + batch_size]
        images: list[Image.Image] = []

        for p in batch_paths:
            try:
                img = Image.open(p).convert("RGB")
                images.append(img)
            except Exception as e:
                print(f"    Warning: could not open {p.name}: {e}")
                # Substitute a blank image to keep batch alignment
                images.append(Image.new("RGB", (224, 224), color=128))

        # Processor handles: resize to 224×224, center-crop, normalize
        # pixel_values shape: (batch, 3, 224, 224)
        inputs = processor(images=images, return_tensors="pt", padding=True)
        pixel_values = inputs["pixel_values"].to(device)

        with torch.no_grad():
            # get_image_features returns the pooled, projected embedding
            feats = model.get_image_features(pixel_values=pixel_values)

        # Move to CPU, convert to float32 numpy
        feats_np = feats.cpu().float().numpy()          # (batch, 512)
        all_embeddings.append(feats_np)

        processed = min(start + batch_size, len(image_paths))
        print(f"    Embedded {processed}/{len(image_paths)} images", end="\r")

    print()  # newline after \r progress
    embeddings = np.vstack(all_embeddings)              # (N, 512)
    return l2_normalize(embeddings)


def build_faiss_index(embeddings: np.ndarray) -> faiss.IndexFlatIP:
    """
    Build an exact inner-product index.
    Because embeddings are L2-normalized, inner product == cosine similarity.
    """
    dim = embeddings.shape[1]
    index = faiss.IndexFlatIP(dim)          # exact search, dot product
    index.add(embeddings.astype(np.float32))
    return index


def build_and_save_index(
    model: CLIPModel,
    processor: CLIPProcessor,
    device: torch.device,
) -> tuple[faiss.IndexFlatIP, list[str]]:
    """Embed all images in IMAGE_DIR and persist index + metadata."""
    image_paths = sorted(
        p for p in IMAGE_DIR.iterdir()
        if p.suffix.lower() in {".jpg", ".jpeg", ".png", ".webp"}
    )
    if not image_paths:
        sys.exit(f"No images found in {IMAGE_DIR}/")

    print(f"  Embedding {len(image_paths)} images ...")
    embeddings = embed_images(image_paths, model, processor, device)

    print("  Building FAISS index ...")
    index = build_faiss_index(embeddings)

    # Persist for reuse
    np.save(INDEX_FILE, embeddings)
    filenames = [p.name for p in image_paths]
    META_FILE.write_text(json.dumps(filenames))
    print(f"  Saved index ({INDEX_FILE}) and metadata ({META_FILE})")

    return index, filenames
```

**Checkpoint:** After this phase, `clip_index.npy` exists (shape `(N, 512)`, float32) and `clip_meta.json` lists filenames. Cosine similarity between any two rows should be in `[-1, 1]`; two embeddings of the same image should be `≈ 1.0`.

## Phase 4 — Text Query Pipeline

**Why this first:** Text-to-image is the most common search direction. Once it works, the image-to-image pipeline reuses nearly identical code.

```python
# ── Phase 4: Text query pipeline ──────────────────────────────────────────────

def embed_text(
    query: str,
    model: CLIPModel,
    processor: CLIPProcessor,
    device: torch.device,
) -> np.ndarray:
    """
    Embed a text query.
    Returns float32 array of shape (512,), L2-normalized.

    CLIP's text encoder uses BPE tokenization (max 77 tokens, including
    [SOS] and [EOS]). Longer queries are silently truncated — design
    queries under ~50 words.
    """
    inputs = processor(
        text=[query],
        return_tensors="pt",
        padding=True,
        truncation=True,
        max_length=77,
    )
    input_ids      = inputs["input_ids"].to(device)
    attention_mask = inputs["attention_mask"].to(device)

    with torch.no_grad():
        feats = model.get_text_features(
            input_ids=input_ids,
            attention_mask=attention_mask,
        )

    feats_np = feats.cpu().float().numpy()[0]       # (512,)
    return l2_normalize(feats_np)


def search(
    query_vec: np.ndarray,
    index: faiss.IndexFlatIP,
    filenames: list[str],
    k: int = TOP_K,
) -> list[tuple[str, float]]:
    """
    Query the FAISS index.
    Returns list of (filename, cosine_score) sorted by score descending.
    """
    q = query_vec.reshape(1, -1).astype(np.float32)
    scores, indices = index.search(q, k)            # scores shape: (1, k)
    results = [
        (filenames[idx], float(scores[0][rank]))
        for rank, idx in enumerate(indices[0])
        if idx != -1
    ]
    return results
```

**Checkpoint:** `embed_text("a dog on a beach", ...)` should return a vector of shape `(512,)` with `np.linalg.norm(vec) ≈ 1.0`. Searching against an index populated with `dog_beach.jpg` should return it in rank 1 or 2.

## Phase 5 — Image Query Pipeline

**Why this first:** Image-to-image search reuses the vision encoder but skips the text tokenizer entirely. This phase validates the encoder is the shared component.

```python
# ── Phase 5: Image query pipeline ─────────────────────────────────────────────

def embed_query_image(
    image_path: str,
    model: CLIPModel,
    processor: CLIPProcessor,
    device: torch.device,
) -> np.ndarray:
    """
    Embed a query image using CLIP vision encoder.
    Returns float32 array of shape (512,), L2-normalized.

    The query image goes through the same ViT-B/32 pipeline as indexed images:
    resize shortest edge to 224, center-crop to 224×224,
    normalize with ImageNet mean/std.
    """
    img = Image.open(image_path).convert("RGB")
    inputs = processor(images=[img], return_tensors="pt")
    pixel_values = inputs["pixel_values"].to(device)

    with torch.no_grad():
        feats = model.get_image_features(pixel_values=pixel_values)

    feats_np = feats.cpu().float().numpy()[0]       # (512,)
    return l2_normalize(feats_np)
```

**Checkpoint:** Querying with `dog_beach.jpg` as the query image should return `dog_beach.jpg` itself at rank 0 with score `≈ 1.0` (it is in the index). The second result should be visually or semantically similar.

## Phase 6 — Results Display and Interactive Loop

**Why this first:** Tying everything into an interactive loop surfaces the entire system's behavior — latency, edge cases, score ranges.

```python
# ── Phase 6: Results display and interactive loop ─────────────────────────────

def print_results(
    results: list[tuple[str, float]],
    query: str,
    query_type: str = "text",
) -> None:
    """Print results as an ASCII table."""
    header = f"  Query ({query_type}): \"{query}\""
    print(f"\n{header}")
    print(f"  {'─' * 60}")
    print(f"  {'Rank':<6}{'Filename':<30}{'Score':>8}  Caption")
    print(f"  {'─' * 60}")
    for rank, (fname, score) in enumerate(results, 1):
        caption = CAPTIONS.get(fname, "—")[:35]
        print(f"  {rank:<6}{fname:<30}{score:>8.4f}  {caption}")
    print(f"  {'─' * 60}")


def load_or_build_index(
    model: CLIPModel,
    processor: CLIPProcessor,
    device: torch.device,
) -> tuple[faiss.IndexFlatIP, list[str]]:
    """Load cached index if available; build otherwise."""
    if INDEX_FILE.exists() and META_FILE.exists():
        print("  Loading cached index ...")
        embeddings = np.load(INDEX_FILE).astype(np.float32)
        filenames  = json.loads(META_FILE.read_text())
        index = faiss.IndexFlatIP(embeddings.shape[1])
        index.add(embeddings)
        print(f"  Loaded index: {embeddings.shape[0]} vectors, dim={embeddings.shape[1]}")
        return index, filenames
    return build_and_save_index(model, processor, device)


def interactive_loop(
    model: CLIPModel,
    processor: CLIPProcessor,
    device: torch.device,
    index: faiss.IndexFlatIP,
    filenames: list[str],
) -> None:
    """REPL: text query, image query, or quit."""
    print("\n  Commands:")
    print("    t: <query>   — text-to-image search")
    print("    i: <path>    — image-to-image search")
    print("    q            — quit")

    while True:
        try:
            raw = input("\n> ").strip()
        except (EOFError, KeyboardInterrupt):
            print("\n  Bye.")
            break

        if not raw or raw.lower() == "q":
            break

        if raw.lower().startswith("t:"):
            query = raw[2:].strip()
            if not query:
                print("  Empty query.")
                continue
            t0 = time.perf_counter()
            vec = embed_text(query, model, processor, device)
            results = search(vec, index, filenames)
            elapsed = (time.perf_counter() - t0) * 1000
            print_results(results, query, query_type="text")
            print(f"  Latency: {elapsed:.1f} ms")

        elif raw.lower().startswith("i:"):
            path = raw[2:].strip()
            if not Path(path).exists():
                print(f"  File not found: {path}")
                continue
            t0 = time.perf_counter()
            vec = embed_query_image(path, model, processor, device)
            results = search(vec, index, filenames)
            elapsed = (time.perf_counter() - t0) * 1000
            print_results(results, path, query_type="image")
            print(f"  Latency: {elapsed:.1f} ms")

        else:
            print("  Unknown command. Use 't: <text>' or 'i: <path>'.")


# ── Entry point ────────────────────────────────────────────────────────────────
def main() -> None:
    section("Phase 1 — Bootstrap Image Collection")
    bootstrap_images()

    section("Phase 2 — Load CLIP Model")
    model, processor, device = load_clip()

    section("Phase 3 — Build / Load FAISS Index")
    index, filenames = load_or_build_index(model, processor, device)

    section("Phase 6 — Interactive Search Loop")
    interactive_loop(model, processor, device, index, filenames)


if __name__ == "__main__":
    main()
```

**Checkpoint:** Full system run:
1. First run builds the index (~30s CPU, ~5s GPU).
2. `t: a dog running on sand` returns `dog_beach.jpg` in top-3.
3. `i: images/mountain_snow.jpg` returns visually similar landscape images.
4. Latency on CPU: 200–600 ms per query (dominated by text encoder forward pass). On GPU: < 50 ms.

## Debugging Guide

| Symptom | Likely Cause | Fix |
|---|---|---|
| `RuntimeError: CUDA out of memory` | Batch too large for GPU | Reduce `BATCH_SIZE` to 4 or 8 |
| All results have score `≈ 0.0` | Embeddings not normalized | Verify `l2_normalize` is called before `index.add` and before `search` |
| Text query returns obviously wrong images | CLIP truncated a long query | Keep queries under 50 words; CLIP's context window is 77 BPE tokens |
| `faiss.IndexFlatIP` wrong scores | Query vector not float32 | Cast with `.astype(np.float32)` before `index.search` |
| Image load fails silently | EXIF rotation not applied | Use `ImageOps.exif_transpose(img)` from Pillow before passing to processor |

## What This Project Teaches You to Say in an Interview

1. "CLIP uses InfoNCE contrastive loss over $N^2$ pairs per batch — the diagonal of the similarity matrix holds positives; off-diagonal entries are in-batch negatives. This is why larger batch sizes materially improve CLIP training."
2. "L2-normalizing embeddings before indexing means inner product equals cosine similarity. I use `IndexFlatIP` rather than `IndexFlatL2` because dot product is one instruction cheaper per dimension than squared Euclidean distance."
3. "CLIP's vision encoder is a ViT-B/32: the image is split into 32×32 patches, each patch is linearly projected to $d=768$, and a `[CLS]` token aggregates global context via 12 layers of self-attention. The `[CLS]` output is then projected to 512-dim via a learned linear head."
4. "Zero-shot retrieval works because the text and image encoders share a metric space — 'a golden retriever' and the pixel pattern of a golden retriever both map near the same point. This generalizes to unseen categories with no fine-tuning."
5. "FAISS `IndexFlatIP` is exact nearest-neighbor. For production with > 1M vectors I would switch to `IndexIVFFlat` or `IndexHNSWFlat` to trade a small recall penalty for sub-linear query time."
