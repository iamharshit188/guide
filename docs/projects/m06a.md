# Word Analogy Explorer — Step-by-Step Guide
> **Module:** 06 — GenAI Core · **Difficulty:** Intermediate · **Type:** GUIDED · **Time:** ~6 hours

## What You'll Build

A CLI tool that trains a Word2Vec skip-gram model from scratch using only NumPy on a tokenized text corpus. Given an analogy query like `king - man + woman`, it returns the top-10 nearest words by cosine similarity in the learned embedding space. Supports nearest-neighbor lookup for any word, shows live training loss every 1000 steps, and outputs a formatted similarity table.

## The Mental Model

Word2Vec skip-gram learns embeddings by predicting context words from a center word. The model has two weight matrices: $\mathbf{W}_{in} \in \mathbb{R}^{|V| \times d}$ (center word embeddings) and $\mathbf{W}_{out} \in \mathbb{R}^{|V| \times d}$ (context word embeddings). For a center word $w$ and true context word $c$, the model maximizes $\sigma(\mathbf{v}_c^T \mathbf{v}_w)$ while minimizing $\sigma(\mathbf{v}_{n_k}^T \mathbf{v}_w)$ for $K$ noise-sampled words $n_k$.

The noise distribution is the unigram frequency raised to the power 0.75 — this flattens the distribution so common words like "the" are sampled less often than their raw frequency would dictate, giving rare words a fighting chance as negative samples. Without this, your model learns excellent representations of stop words and terrible ones for everything else.

After training, only $\mathbf{W}_{in}$ is used. The analogy `A - B + C = ?` works because the embedding space encodes relational structure as vector offsets: $\mathbf{v}_{king} - \mathbf{v}_{man} \approx \mathbf{v}_{queen} - \mathbf{v}_{woman}$. Cosine similarity (not Euclidean distance) is used for search because embedding norms are arbitrary — direction encodes meaning, magnitude does not.

## Phase 1 — Corpus Load, Vocab Build, and Negative Sampling Table

**Why this first:** Every downstream operation depends on the vocabulary index. The negative sampling table must be built from corpus frequencies before training begins.

```python
import numpy as np
import re
import collections
from pathlib import Path

RNG = np.random.default_rng(42)


def section(title: str) -> None:
    print(f"\n{'='*60}\n{title}\n{'='*60}")


def load_corpus(path: str) -> list[str]:
    """Read, lowercase, tokenize on whitespace and punctuation."""
    text = Path(path).read_text(encoding="utf-8").lower()
    tokens = re.findall(r"[a-z']+", text)
    return tokens


def build_vocab(
    tokens: list[str], min_freq: int = 5
) -> tuple[dict, dict, list]:
    """
    Returns:
        word2idx : token -> int
        idx2word : int -> token
        freqs    : list of counts aligned with word2idx ordering
    """
    counter = collections.Counter(tokens)
    vocab = {w: c for w, c in counter.items() if c >= min_freq}
    word2idx = {w: i for i, w in enumerate(sorted(vocab))}
    idx2word = {i: w for w, i in word2idx.items()}
    freqs = [vocab[idx2word[i]] for i in range(len(word2idx))]
    return word2idx, idx2word, freqs


def build_negative_table(
    freqs: list[int], table_size: int = 10_000_000
) -> np.ndarray:
    """
    Unigram^0.75 distribution for negative sampling.
    table_size entries, each is a word index. Sample with RNG.integers.
    """
    freqs_arr = np.array(freqs, dtype=np.float64)
    powered = freqs_arr ** 0.75
    probs = powered / powered.sum()
    table = RNG.choice(len(freqs), size=table_size, p=probs).astype(np.int32)
    return table


def encode_corpus(tokens: list[str], word2idx: dict) -> np.ndarray:
    """Convert token list to int array, dropping OOV tokens."""
    ids = [word2idx[t] for t in tokens if t in word2idx]
    return np.array(ids, dtype=np.int32)


if __name__ == "__main__":
    section("Phase 1 — Vocab Build")
    tokens = load_corpus("wiki2.txt")
    word2idx, idx2word, freqs = build_vocab(tokens, min_freq=5)
    neg_table = build_negative_table(freqs)
    corpus = encode_corpus(tokens, word2idx)
    V = len(word2idx)
    print(f"Tokens : {len(tokens):,}  |  Vocab : {V:,}  |  Corpus IDs : {len(corpus):,}")
    print(f"Neg-table sample: {[idx2word[i] for i in neg_table[:8]]}")
    # Checkpoint: V ~20,000-30,000 for WikiText-2 at min_freq=5
```

**Checkpoint:** `V` should be in the range 20,000–30,000 for WikiText-2. `neg_table[:10]` should include a mix of common and mid-frequency words — not dominated by "the", "a", "of".

## Phase 2 — Weight Initialization and Forward Pass

**Why this first:** Validate the sigmoid and dot product math before plugging into the training loop. A sign error in the loss here poisons 6 hours of training.

```python
def init_weights(V: int, d: int = 100) -> tuple[np.ndarray, np.ndarray]:
    """
    W_in  : center word embeddings,  shape (V, d)
    W_out : context word embeddings, shape (V, d)
    Uniform ±(0.5/d) initialization keeps initial activations small.
    """
    scale = 0.5 / d
    W_in  = RNG.uniform(-scale, scale, size=(V, d)).astype(np.float32)
    W_out = RNG.uniform(-scale, scale, size=(V, d)).astype(np.float32)
    return W_in, W_out


def sigmoid(x: np.ndarray) -> np.ndarray:
    x = np.clip(x, -20.0, 20.0)   # numerically stable
    return 1.0 / (1.0 + np.exp(-x))


def forward_loss(
    center_idx: int,
    context_idx: int,
    neg_indices: np.ndarray,
    W_in: np.ndarray,
    W_out: np.ndarray,
) -> tuple[float, np.ndarray, np.ndarray, np.ndarray]:
    """
    Negative sampling objective:
      L = -log σ(v_c · v_w) - Σ_k log σ(-v_{n_k} · v_w)

    Returns:
        loss     : scalar
        grad_vw  : gradient w.r.t. W_in[center_idx]        shape (d,)
        grad_vc  : gradient w.r.t. W_out[context_idx]      shape (d,)
        grad_neg : gradients w.r.t. W_out[neg_indices]     shape (K, d)
    """
    v_w   = W_in[center_idx]         # (d,)
    v_c   = W_out[context_idx]       # (d,)
    v_neg = W_out[neg_indices]       # (K, d)

    # Positive pair
    pos_score = sigmoid(float(v_c @ v_w))
    loss_pos  = -np.log(pos_score + 1e-8)

    # Negative pairs
    neg_scores = sigmoid(-(v_neg @ v_w))   # (K,)
    loss_neg   = -np.log(neg_scores + 1e-8).sum()

    loss = float(loss_pos + loss_neg)

    # Gradient of L w.r.t. v_w:
    #   from positive: (σ - 1) · v_c
    #   from negatives: Σ_k (1 - σ(-v_{n_k}·v_w)) · (-v_{n_k})
    grad_vw  = (pos_score - 1.0) * v_c
    grad_vw += ((1.0 - neg_scores)[:, None] * (-v_neg)).sum(axis=0)

    # Gradient of L w.r.t. v_c: (σ - 1) · v_w
    grad_vc  = (pos_score - 1.0) * v_w

    # Gradient of L w.r.t. v_{n_k}: (1 - σ(-v_{n_k}·v_w)) · (-v_w)
    grad_neg = (1.0 - neg_scores)[:, None] * (-v_w[None, :])

    return loss, grad_vw, grad_vc, grad_neg


if __name__ == "__main__":
    section("Phase 2 — Forward Pass Check")
    W_in, W_out = init_weights(V=100, d=50)
    loss, gvw, gvc, gneg = forward_loss(0, 1, np.array([5, 7, 3]), W_in, W_out)
    print(f"Initial loss : {loss:.4f}  (expect ~2.0–4.0 for K=3)")
    print(f"grad_vw shape: {gvw.shape}  grad_neg shape: {gneg.shape}")
```

**Checkpoint:** Initial loss with $K=5$ negatives should be approximately $-\log(0.5) \times 6 \approx 4.16$ — all sigmoid outputs near 0.5 at random init. Gradient shapes must match embedding dimension $d$.

## Phase 3 — Training Loop with SGD and LR Decay

**Why this now:** The (center, context) pair generator and negative sampler must be correct before spending compute on training.

```python
def generate_pairs(corpus: np.ndarray, window: int = 5):
    """Yield (center_idx, context_idx) pairs via sliding window."""
    n = len(corpus)
    for i in range(n):
        center = int(corpus[i])
        left   = max(0, i - window)
        right  = min(n, i + window + 1)
        for j in range(left, right):
            if j != i:
                yield center, int(corpus[j])


def sample_negatives(
    neg_table: np.ndarray,
    center: int,
    context: int,
    K: int = 5,
) -> np.ndarray:
    """Draw K negatives from the table, excluding center and context."""
    negs: list[int] = []
    table_len = len(neg_table)
    while len(negs) < K:
        idx = int(neg_table[RNG.integers(table_len)])
        if idx != center and idx != context:
            negs.append(idx)
    return np.array(negs, dtype=np.int32)


def train(
    corpus: np.ndarray,
    V: int,
    neg_table: np.ndarray,
    d: int = 100,
    window: int = 5,
    K: int = 5,
    lr_init: float = 0.025,
    epochs: int = 3,
    log_every: int = 1000,
) -> tuple[np.ndarray, np.ndarray]:
    W_in, W_out = init_weights(V, d)
    total_pairs = len(corpus) * 2 * window * epochs   # approximate
    step = 0
    running_loss = 0.0

    for epoch in range(epochs):
        for center, context in generate_pairs(corpus, window):
            # Linear decay: lr → 0.0001 × lr_init at end of training
            lr = max(lr_init * (1 - step / total_pairs), lr_init * 0.0001)

            neg_indices = sample_negatives(neg_table, center, context, K)
            loss, grad_vw, grad_vc, grad_neg = forward_loss(
                center, context, neg_indices, W_in, W_out
            )

            # SGD updates
            W_in[center]       -= lr * grad_vw
            W_out[context]     -= lr * grad_vc
            W_out[neg_indices] -= lr * grad_neg

            running_loss += loss
            step += 1

            if step % log_every == 0:
                avg = running_loss / log_every
                print(
                    f"Step {step:>8,}  epoch {epoch + 1}/{epochs}"
                    f"  lr {lr:.5f}  loss {avg:.4f}"
                )
                running_loss = 0.0

    return W_in, W_out


if __name__ == "__main__":
    section("Phase 3 — Training")
    W_in, W_out = train(corpus, V, neg_table, d=100, epochs=3)
    np.save("embeddings_in.npy", W_in)
    print("Saved embeddings_in.npy")
```

**Checkpoint:** Loss at step 1,000 should be approximately 3.5–4.5. By step 50,000 it should be below 2.0. By end of epoch 1, below 1.5. If loss is flat or rising, check gradient signs — the most common bug is a missing negation in `grad_vw`.

## Phase 4 — Analogy Query Engine

**Why this now:** Validates that the embedding space has learned geometric structure. If `king - man + woman` doesn't return `queen` in top-5, the model needs more epochs — diagnose before building the CLI.

```python
def normalize_rows(M: np.ndarray) -> np.ndarray:
    """L2-normalize each row. Returns unit-vector matrix."""
    norms = np.linalg.norm(M, axis=1, keepdims=True)
    norms = np.where(norms == 0, 1.0, norms)
    return M / norms


def analogy_query(
    word_a: str,
    word_b: str,
    word_c: str,
    W_norm: np.ndarray,
    word2idx: dict,
    idx2word: dict,
    top_k: int = 10,
) -> list[tuple[str, float]]:
    """
    Compute v_A - v_B + v_C, return top-k nearest by cosine similarity.
    Excludes the three query words from results.
    Example: analogy_query("king", "man", "woman", ...) → [("queen", 0.72), ...]
    """
    missing = [w for w in [word_a, word_b, word_c] if w not in word2idx]
    if missing:
        raise KeyError(f"Words not in vocabulary: {missing}")

    query_vec = (
        W_norm[word2idx[word_a]]
        - W_norm[word2idx[word_b]]
        + W_norm[word2idx[word_c]]
    )
    query_vec /= np.linalg.norm(query_vec) + 1e-8

    scores = W_norm @ query_vec   # (V,) — cosine sims

    # Exclude query words
    for w in [word_a, word_b, word_c]:
        scores[word2idx[w]] = -2.0

    top_indices = np.argpartition(scores, -top_k)[-top_k:]
    top_indices = top_indices[np.argsort(scores[top_indices])[::-1]]
    return [(idx2word[i], float(scores[i])) for i in top_indices]


def print_results(results: list[tuple[str, float]], query_str: str) -> None:
    print(f"\nQuery: {query_str}")
    print(f"{'Rank':<6}{'Word':<20}{'Similarity':>12}")
    print("-" * 38)
    for rank, (word, sim) in enumerate(results, 1):
        print(f"{rank:<6}{word:<20}{sim:>12.4f}")


if __name__ == "__main__":
    section("Phase 4 — Analogy Queries")
    W_norm = normalize_rows(W_in)

    results = analogy_query("king", "man", "woman", W_norm, word2idx, idx2word)
    print_results(results, "king - man + woman = ?")

    results = analogy_query("paris", "france", "germany", W_norm, word2idx, idx2word)
    print_results(results, "paris - france + germany = ?")
```

**Checkpoint:** With 3 epochs on WikiText-2 and $d=100$: `king - man + woman` should return `queen` in top-3. Capital-country analogies (`paris - france + germany → berlin`) require a larger corpus — acceptable if it appears in top-10.

## Phase 5 — Nearest Neighbor Search and CLI

**Why this last:** Nearest-neighbor is the simplest validation that individual word embeddings are meaningful, and it completes the tool's interactive interface.

```python
import sys


def nearest_neighbors(
    word: str,
    W_norm: np.ndarray,
    word2idx: dict,
    idx2word: dict,
    top_k: int = 10,
) -> list[tuple[str, float]]:
    if word not in word2idx:
        raise KeyError(f"'{word}' not in vocabulary")
    idx = word2idx[word]
    scores = W_norm @ W_norm[idx]
    scores[idx] = -2.0   # exclude self
    top_indices = np.argpartition(scores, -top_k)[-top_k:]
    top_indices = top_indices[np.argsort(scores[top_indices])[::-1]]
    return [(idx2word[i], float(scores[i])) for i in top_indices]


def run_cli(W_in: np.ndarray, word2idx: dict, idx2word: dict) -> None:
    W_norm = normalize_rows(W_in)
    print("\nWord Analogy Explorer")
    print("  nn <word>             — nearest neighbors")
    print("  analogy <A> <B> <C>   — compute A - B + C")
    print("  quit\n")

    while True:
        try:
            line = input(">>> ").strip()
        except (EOFError, KeyboardInterrupt):
            break

        if not line or line == "quit":
            break

        parts = line.split()
        cmd = parts[0]
        try:
            if cmd == "nn" and len(parts) == 2:
                results = nearest_neighbors(parts[1], W_norm, word2idx, idx2word)
                print_results(results, f"nearest({parts[1]})")
            elif cmd == "analogy" and len(parts) == 4:
                results = analogy_query(
                    parts[1], parts[2], parts[3], W_norm, word2idx, idx2word
                )
                print_results(results, f"{parts[1]} - {parts[2]} + {parts[3]} = ?")
            else:
                print("Unknown command.")
        except KeyError as e:
            print(f"Error: {e}")


def main() -> None:
    section("Loading")
    W_in = np.load("embeddings_in.npy")
    tokens = load_corpus("wiki2.txt")
    word2idx, idx2word, _ = build_vocab(tokens, min_freq=5)
    print(f"Embeddings shape: {W_in.shape}  |  Vocab: {len(word2idx):,}")
    run_cli(W_in, word2idx, idx2word)


if __name__ == "__main__":
    main()
```

**Checkpoint:** `nn king` returns royalty-adjacent words: `queen`, `prince`, `throne`, `monarch`. `nn python` returns programming languages or snake-adjacent terms depending on corpus domain — both are correct behavior.

## Debugging Guide

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| Loss flat at ~4.0 after 50K steps | LR too small or gradient signs flipped | Check `(pos_score - 1)` not `(1 - pos_score)` in `grad_vw`; verify LR > 1e-4 |
| All nearest neighbors are stop words | Negative table dominated by high-freq words | Confirm you raised freqs to power 0.75 before computing probabilities |
| `KeyError` on valid-looking word | Word below min_freq=5 threshold | Lower min_freq or add the word's variant to tokenizer regex |
| Analogy results look random after training | Corpus too small or too few epochs | Need at least 5M tokens; 3 epochs minimum on WikiText-2 |
| OOM during negative table build | table_size=10M × int32 = 40MB | Reduce table_size to 1M; sampling is still valid |

2. "The unigram^0.75 noise distribution prevents high-frequency words from dominating negative samples — without it, 'the' and 'a' are sampled ~30% of the time and the model learns to push them away from everything, corrupting stop-word embeddings."
3. "Analogy queries work because relational meaning is encoded as vector offsets in the embedding space — an emergent consequence of the co-occurrence structure in the corpus, not something we explicitly supervised."
4. "I use only $\mathbf{W}_{in}$ for inference because the out-embedding matrix is optimized for discrimination during training, not for capturing semantic proximity — averaging in and out embeddings helps in some settings but rarely meaningfully."
5. "Cosine similarity, not Euclidean distance, is correct for this search because embedding norms are arbitrary — a word appearing 10,000 times has a larger-magnitude embedding than one appearing 100 times, but the angle between them encodes semantic relationship."
