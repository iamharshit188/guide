## What You'll Build

A CLI tool that trains a Word2Vec skip-gram model from scratch using NumPy on a raw text corpus. It accepts analogy queries (`king - man + woman = ?`) and returns top-10 completions by cosine similarity. After training, word embeddings encode semantic and syntactic relationships; the arithmetic reflects genuine structure in the embedding space, not lookup magic.

## The Mental Model

Skip-gram trains a word to predict its context. Given center word "bank", the model must assign high probability to nearby words ("river", "account", "deposit"). The model learns this by maintaining two embedding matrices: $\mathbf{W}_{in} \in \mathbb{R}^{V \times D}$ (center word embeddings) and $\mathbf{W}_{out} \in \mathbb{R}^{V \times D}$ (context word embeddings). During training, the center word embedding is updated to be close to its real context embeddings and far from randomly sampled "noise" words — this is negative sampling.

Negative sampling avoids the $O(V)$ softmax normalization over the entire vocabulary. Instead of computing $\frac{\exp(\mathbf{v}_c^T \mathbf{v}_w)}{\sum_{j=1}^V \exp(\mathbf{v}_j^T \mathbf{v}_w)}$, we frame it as $K+1$ binary classifications: is this (center, word) pair a real pair or a noise pair? This is $O(K)$ per training step, not $O(V)$.

The skip-gram loss with $K$ negative samples is:
$$\mathcal{L} = -\log \sigma(\mathbf{v}_c^T \mathbf{v}_w) - \sum_{k=1}^K \mathbb{E}_{n_k \sim P_n}\left[\log \sigma(-\mathbf{v}_{n_k}^T \mathbf{v}_w)\right]$$

where $\mathbf{v}_w$ is the center word embedding, $\mathbf{v}_c$ is a real context embedding, $\mathbf{v}_{n_k}$ are negative sample embeddings, and $P_n \propto f(w)^{0.75}$ is the noise distribution (unigram frequency raised to the 3/4 power, which down-weights common words like "the").

Analogies work because the embedding space is structured: the direction from "man" to "woman" encodes "gender change." Applying the same direction to "king" lands near "queen." This is not a built-in feature — it emerges from the co-occurrence statistics of a large corpus.

## Phase 1 — Corpus Tokenization, Vocabulary, and Noise Table

**Why this first:** All downstream operations — embedding lookup, negative sampling, training — depend on word→integer indices and the noise distribution. Get the indexing right before adding any math.

```python
import numpy as np
import re
from collections import Counter
import struct

def section(title):
    print(f"\n{'='*60}")
    print(f"  {title}")
    print(f"{'='*60}")

def load_corpus(path: str) -> list[str]:
    """Read raw text, lowercase, split on non-alpha characters."""
    with open(path, 'r', encoding='utf-8') as f:
        text = f.read().lower()
    # keep only alphabetic tokens
    tokens = re.findall(r'[a-z]+', text)
    return tokens

def build_vocab(tokens: list[str], min_freq: int = 5) -> tuple[dict, dict, list]:
    """
    Returns:
        word2idx: word -> int index (sorted by descending frequency)
        idx2word: int -> word
        freqs: list of counts aligned with word2idx
    """
    counter = Counter(tokens)
    # filter rare words
    vocab = [(w, c) for w, c in counter.items() if c >= min_freq]
    vocab.sort(key=lambda x: -x[1])  # sort by descending frequency

    word2idx = {w: i for i, (w, _) in enumerate(vocab)}
    idx2word = [w for w, _ in vocab]
    freqs = [c for _, c in vocab]
    return word2idx, idx2word, freqs

def build_noise_table(freqs: list[int], table_size: int = 100_000_000) -> np.ndarray:
    """
    Unigram^0.75 noise distribution.
    Pre-builds a lookup table: draw a random index -> get a word index.
    Avoids calling np.random.choice with p= on every step (too slow).
    """
    freq_arr = np.array(freqs, dtype=np.float64)
    freq_arr = freq_arr ** 0.75
    freq_arr /= freq_arr.sum()
    counts = (freq_arr * table_size).astype(np.int64)

    table = np.empty(counts.sum(), dtype=np.int32)
    ptr = 0
    for word_idx, count in enumerate(counts):
        table[ptr:ptr + count] = word_idx
        ptr += count
    return table

def encode_corpus(tokens: list[str], word2idx: dict) -> np.ndarray:
    """Convert token list to integer array, dropping OOV words."""
    indices = [word2idx[t] for t in tokens if t in word2idx]
    return np.array(indices, dtype=np.int32)

if __name__ == "__main__":
    section("Phase 1: Tokenization and Vocabulary")
    rng = np.random.default_rng(42)

    tokens = load_corpus("text8")
    print(f"Total tokens: {len(tokens):,}")

    word2idx, idx2word, freqs = build_vocab(tokens, min_freq=5)
    V = len(word2idx)
    print(f"Vocabulary size (min_freq=5): {V:,}")
    print(f"Top-10 words: {idx2word[:10]}")

    noise_table = build_noise_table(freqs)
    print(f"Noise table size: {len(noise_table):,}")
    print(f"Sample negatives (first 5): {[idx2word[i] for i in noise_table[:5]]}")

    corpus = encode_corpus(tokens, word2idx)
    print(f"Encoded corpus length: {len(corpus):,}")
```

**Checkpoint:** `V` is approximately 71,000 for text8 with `min_freq=5`. Top-10 words are function words ("the", "of", "and", ...). Noise table has 100M entries. Rare words like "aardvark" are absent from the vocabulary.

## Phase 2 — Embedding Initialization and Forward Pass

**Why this first:** The forward pass defines what the model computes. Once it's correct, the backward pass just differentiates it.

```python
def init_embeddings(V: int, D: int, rng: np.random.Generator) -> tuple[np.ndarray, np.ndarray]:
    """
    Xavier (Glorot) uniform initialization.
    W_in: center word embeddings (what we keep after training)
    W_out: context word embeddings (discarded after training)
    """
    limit = np.sqrt(6.0 / (V + D))
    W_in  = rng.uniform(-limit, limit, (V, D)).astype(np.float32)
    W_out = rng.uniform(-limit, limit, (V, D)).astype(np.float32)
    return W_in, W_out

def sigmoid(x: np.ndarray) -> np.ndarray:
    # numerically stable sigmoid
    return np.where(x >= 0, 1 / (1 + np.exp(-x)), np.exp(x) / (1 + np.exp(x)))

def forward(center_idx: int, context_idx: int, neg_indices: np.ndarray,
            W_in: np.ndarray, W_out: np.ndarray) -> tuple[float, np.ndarray, np.ndarray]:
    """
    Compute skip-gram negative sampling loss and gradients.

    Loss = -log sigma(v_c . v_w) - sum_k log sigma(-v_nk . v_w)

    Returns:
        loss: scalar float
        grad_w: gradient w.r.t. center word embedding W_in[center_idx]
        grad_ctx: gradient w.r.t. context embedding W_out[context_idx]  (shape: D,)
        grad_neg: gradient w.r.t. negative embeddings W_out[neg_indices] (shape: K, D)
    """
    v_w  = W_in[center_idx]           # shape: (D,)
    v_c  = W_out[context_idx]         # shape: (D,)
    v_nk = W_out[neg_indices]         # shape: (K, D)

    # positive pair score
    pos_score = sigmoid(np.dot(v_c, v_w))        # scalar

    # negative pair scores
    neg_scores = sigmoid(-v_nk @ v_w)             # shape: (K,)

    # loss (negate for minimization)
    loss = -np.log(pos_score + 1e-7) - np.sum(np.log(neg_scores + 1e-7))

    # gradients via chain rule
    # d_loss / d(v_c . v_w) * d(v_c . v_w) / d(v_w) = (sigma - 1) * v_c
    err_pos = pos_score - 1.0           # scalar
    # d_loss / d(-v_nk . v_w) * d(-v_nk . v_w) / d(v_w) = (1 - sigma) * (-v_nk) summed
    err_neg = 1.0 - neg_scores          # shape: (K,)   (positive = reduce, so gradient pulls v_w away)

    grad_w   = err_pos * v_c + (-err_neg) @ v_nk  # (D,)
    grad_ctx = err_pos * v_w                        # (D,)
    grad_neg = np.outer(-err_neg, v_w)              # (K, D)

    return loss, grad_w, grad_ctx, grad_neg

if __name__ == "__main__":
    section("Phase 2: Embeddings and Forward Pass")
    D = 100  # embedding dimension
    V_test = 1000

    W_in_test, W_out_test = init_embeddings(V_test, D, rng)
    print(f"W_in shape: {W_in_test.shape}, dtype: {W_in_test.dtype}")
    print(f"Xavier range: [{W_in_test.min():.4f}, {W_in_test.max():.4f}]")

    # test forward pass
    loss, gw, gc, gn = forward(0, 1, np.array([2, 3, 4]), W_in_test, W_out_test)
    print(f"Initial loss (should be near log(2) ≈ 0.693): {loss:.4f}")
    print(f"Gradient shapes: center={gw.shape}, ctx={gc.shape}, neg={gn.shape}")
```

**Checkpoint:** Initial loss ≈ $\log(2) \approx 0.693$ (sigmoid of random dot products ≈ 0.5, so $-\log(0.5) \approx 0.693$). If loss is 0 or 10, initialization is wrong. Gradient shapes: `(D,)`, `(D,)`, `(K, D)`.

## Phase 3 — Training Loop with SGD and Learning Rate Decay

**Why this first:** The training loop is the integration point — it generates (center, context) pairs, samples negatives from the noise table, calls forward, and applies SGD updates.

```python
def get_pairs(corpus: np.ndarray, window: int, i: int) -> list[tuple[int, int]]:
    """
    Generate (center, context) pairs for position i.
    Window size is drawn uniformly from [1, window] (Word2Vec convention).
    """
    half_w = rng.integers(1, window + 1)
    pairs = []
    start = max(0, i - half_w)
    end   = min(len(corpus), i + half_w + 1)
    center = int(corpus[i])
    for j in range(start, end):
        if j != i:
            pairs.append((center, int(corpus[j])))
    return pairs

def sample_negatives(noise_table: np.ndarray, K: int, exclude: set,
                     rng: np.random.Generator) -> np.ndarray:
    """Sample K negative word indices, avoiding words in exclude set."""
    negs = []
    while len(negs) < K:
        candidates = rng.integers(0, len(noise_table), size=K * 2)
        for c in candidates:
            w = int(noise_table[c])
            if w not in exclude and len(negs) < K:
                negs.append(w)
    return np.array(negs, dtype=np.int32)

def train(corpus: np.ndarray, W_in: np.ndarray, W_out: np.ndarray,
          noise_table: np.ndarray, rng: np.random.Generator,
          window: int = 5, K: int = 5, lr_start: float = 0.025,
          lr_min: float = 0.0001, n_epochs: int = 1,
          log_every: int = 1000) -> tuple[np.ndarray, np.ndarray]:
    """
    Train skip-gram with negative sampling.
    Uses SGD with linear learning rate decay (Word2Vec default).
    """
    total_steps = len(corpus) * n_epochs
    step = 0

    for epoch in range(n_epochs):
        total_loss = 0.0
        pairs_seen = 0

        for i in range(len(corpus)):
            # linear LR decay: lr decreases from lr_start to lr_min
            lr = max(lr_min, lr_start * (1.0 - step / total_steps))

            pairs = get_pairs(corpus, window, i)
            for center_idx, ctx_idx in pairs:
                neg_idx = sample_negatives(
                    noise_table, K,
                    exclude={center_idx, ctx_idx},
                    rng=rng
                )

                loss, grad_w, grad_ctx, grad_neg = forward(
                    center_idx, ctx_idx, neg_idx, W_in, W_out
                )

                # SGD update
                W_in[center_idx]  -= lr * grad_w
                W_out[ctx_idx]    -= lr * grad_ctx
                W_out[neg_idx]    -= lr * grad_neg

                total_loss += loss
                pairs_seen += 1

            step += 1

            if i > 0 and i % log_every == 0:
                avg_loss = total_loss / max(pairs_seen, 1)
                pct = 100.0 * step / total_steps
                print(f"  Epoch {epoch+1} | {pct:.1f}% | step {step:,} | avg_loss {avg_loss:.4f} | lr {lr:.6f}")
                total_loss = 0.0
                pairs_seen = 0

    return W_in, W_out

if __name__ == "__main__":
    section("Phase 3: Training")
    D = 100
    K = 5      # negative samples
    WINDOW = 5

    W_in, W_out = init_embeddings(V, D, rng)
    print(f"Training on {len(corpus):,} tokens, vocab size {V:,}")
    print(f"Embedding dim: {D}, Window: {WINDOW}, Negatives: {K}")
    print(f"This will take ~20 min on CPU for text8 (1 epoch)\n")

    W_in, W_out = train(
        corpus, W_in, W_out, noise_table, rng,
        window=WINDOW, K=K, lr_start=0.025, lr_min=0.0001,
        n_epochs=3, log_every=100_000
    )

    # save embeddings
    np.save("embeddings_in.npy", W_in)
    np.save("embeddings_out.npy", W_out)
    print("Saved embeddings_in.npy and embeddings_out.npy")
```

**Checkpoint:** Loss should decrease from ~3–4 (with K=5 negatives: $-\log(0.5) \times 6 \approx 4.16$) to ~1.5–2.0 after 1 epoch on text8. If loss increases, check that gradients are being subtracted (not added) and that `lr` is positive.

## Phase 4 — Analogy Queries via Embedding Arithmetic

**Why this first:** This is the core deliverable. Once embeddings are trained, normalize them once and all queries are just cosine similarity searches.

```python
def normalize(W: np.ndarray) -> np.ndarray:
    """L2-normalize each row. Returns unit-norm embedding matrix."""
    norms = np.linalg.norm(W, axis=1, keepdims=True)
    norms = np.where(norms == 0, 1.0, norms)  # avoid division by zero
    return W / norms

def analogy_query(a: str, b: str, c: str, W_norm: np.ndarray,
                  word2idx: dict, idx2word: list,
                  top_k: int = 10) -> list[tuple[str, float]]:
    """
    a - b + c = ?
    Example: king - man + woman = queen

    Method: compute query vector q = v_c - v_a + v_b,
            normalize q, find top-k by cosine similarity.
            Exclude a, b, c from results.
    """
    for word in [a, b, c]:
        if word not in word2idx:
            raise ValueError(f"'{word}' not in vocabulary")

    v_a = W_norm[word2idx[a]]
    v_b = W_norm[word2idx[b]]
    v_c = W_norm[word2idx[c]]

    query = v_c - v_a + v_b
    query /= (np.linalg.norm(query) + 1e-8)

    # cosine similarity with all words (dot product of unit vectors)
    sims = W_norm @ query  # shape: (V,)

    # exclude the query words
    exclude = {word2idx[a], word2idx[b], word2idx[c]}
    for idx in exclude:
        sims[idx] = -np.inf

    top_indices = np.argpartition(sims, -top_k)[-top_k:]
    top_indices = top_indices[np.argsort(sims[top_indices])[::-1]]

    return [(idx2word[i], float(sims[i])) for i in top_indices]

def print_analogy_results(a: str, b: str, c: str, results: list[tuple[str, float]]):
    print(f"\n  Query: {a} - {b} + {c} = ?")
    print(f"  {'Rank':<6} {'Word':<20} {'Cosine Sim':<12}")
    print(f"  {'-'*38}")
    for rank, (word, score) in enumerate(results, 1):
        print(f"  {rank:<6} {word:<20} {score:.4f}")

if __name__ == "__main__":
    section("Phase 4: Analogy Queries")

    W_in = np.load("embeddings_in.npy")
    W_norm = normalize(W_in)
    print(f"Normalized embedding matrix: {W_norm.shape}")

    queries = [
        ("king",   "man",    "woman"),
        ("paris",  "france", "italy"),
        ("bigger", "big",    "small"),
        ("walked", "walk",   "run"),
    ]

    for a, b, c in queries:
        try:
            results = analogy_query(a, b, c, W_norm, word2idx, idx2word, top_k=10)
            print_analogy_results(a, b, c, results)
        except ValueError as e:
            print(f"  Skipped: {e}")
```

**Checkpoint:** After 3 epochs on text8, `king - man + woman` should return "queen" in the top-3 (rank 1 is common but not guaranteed at D=100). `paris - france + italy` should return "rome". If the top result is always the query word itself, you forgot to exclude the query words from the result set.

## Phase 5 — Nearest Neighbor Search and Pretty-Print CLI

**Why this final:** Nearest neighbor search is a degenerate analogy (query = $\mathbf{0} + \mathbf{v}_w$). Wrap both into a clean CLI that accepts user input.

```python
def nearest_neighbors(word: str, W_norm: np.ndarray, word2idx: dict,
                       idx2word: list, top_k: int = 10) -> list[tuple[str, float]]:
    """Return top-k words by cosine similarity to the given word."""
    if word not in word2idx:
        raise ValueError(f"'{word}' not in vocabulary")

    query = W_norm[word2idx[word]]
    sims  = W_norm @ query

    sims[word2idx[word]] = -np.inf  # exclude self

    top_indices = np.argpartition(sims, -top_k)[-top_k:]
    top_indices = top_indices[np.argsort(sims[top_indices])[::-1]]
    return [(idx2word[i], float(sims[i])) for i in top_indices]

def print_neighbors(word: str, results: list[tuple[str, float]]):
    print(f"\n  Nearest neighbors of '{word}':")
    print(f"  {'Rank':<6} {'Word':<20} {'Cosine Sim':<12}")
    print(f"  {'-'*38}")
    for rank, (w, s) in enumerate(results, 1):
        print(f"  {rank:<6} {w:<20} {s:.4f}")

def run_cli(W_norm: np.ndarray, word2idx: dict, idx2word: list):
    print("\n  Word Analogy Explorer")
    print("  Commands:")
    print("    analogy: a - b + c   (e.g., 'king - man + woman')")
    print("    nn: word             (e.g., 'nn: python')")
    print("    quit")

    while True:
        raw = input("\n> ").strip()
        if raw.lower() in ('quit', 'exit', 'q'):
            break

        if raw.lower().startswith('nn:'):
            word = raw[3:].strip().lower()
            try:
                results = nearest_neighbors(word, W_norm, word2idx, idx2word)
                print_neighbors(word, results)
            except ValueError as e:
                print(f"  Error: {e}")

        elif '-' in raw and '+' in raw:
            # parse "a - b + c"
            try:
                parts = re.split(r'[-+]', raw)
                a, b, c = [p.strip().lower() for p in parts]
                results = analogy_query(a, b, c, W_norm, word2idx, idx2word)
                print_analogy_results(a, b, c, results)
            except (ValueError, IndexError) as e:
                print(f"  Error: {e}. Format: 'a - b + c'")

        else:
            print("  Unknown command. Use 'a - b + c' or 'nn: word'")

if __name__ == "__main__":
    section("Phase 5: CLI")

    W_in   = np.load("embeddings_in.npy")
    W_norm = normalize(W_in)

    # quick smoke test
    try:
        nn_results = nearest_neighbors("computer", W_norm, word2idx, idx2word)
        print_neighbors("computer", nn_results)
    except ValueError as e:
        print(f"  'computer' not in vocab: {e}")

    run_cli(W_norm, word2idx, idx2word)
```

**Checkpoint:** `nn: computer` should return words like "software", "hardware", "system", "network" in the top-10. If results are all function words, the model hasn't trained long enough or the vocabulary is too small (increase `min_freq=5` to include more content words).

## Debugging Guide

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| Loss stays flat after 10K steps | Learning rate too small or too large | Check `lr_start=0.025`; if flat from start, check gradient signs — should subtract `lr * grad` |
| All analogies return "the", "of", "and" | Did not exclude high-frequency words from results | These words dominate because they appear in all contexts; add subsampling: skip word $w$ in training with probability $p = 1 - \sqrt{t/f(w)}$ where $t=10^{-5}$ |
| `king - man + woman` returns "king" | Did not exclude query words from similarity results | Set `sims[word2idx[a]] = sims[word2idx[b]] = sims[word2idx[c]] = -np.inf` |
| NaN loss after a few thousand steps | Learning rate too large for some word pair | Reduce `lr_start` to 0.01; check for zero-norm embeddings after update |
| Nearest neighbor returns the query word itself | Forgot to exclude `word2idx[word]` from the result set | `sims[word2idx[word]] = -np.inf` before `argpartition` |

## What This Project Teaches You to Say in an Interview

1. **Skip-gram negative sampling bypasses the $O(V)$ softmax** by reformulating prediction as $K+1$ binary classifications. The noise distribution $P_n \propto f(w)^{0.75}$ down-weights common words, so you don't waste negative samples on "the" and "of."

2. **Xavier initialization sets embedding variance to $\frac{2}{V+D}$**, keeping initial dot products in a range where sigmoid gradients are non-vanishing. Random normal with $\sigma=1$ would give dot products with variance $D$, saturating the sigmoid at initialization.

3. **Analogy arithmetic works because the embedding space is linear for semantic relationships** — the displacement from "man" to "woman" is approximately constant across gender-paired nouns. This is an emergent property of the co-occurrence statistics, not a designed feature.

4. **`np.argpartition` is $O(V)$** while full sort is $O(V \log V)$. For top-10 from a 71K vocabulary, argpartition then sort of 10 elements is faster than full sort by approximately $\log(71000)/\log(10) \approx 4.8\times$.

5. **The center embedding matrix $\mathbf{W}_{in}$ is what you keep** — the context matrix $\mathbf{W}_{out}$ is discarded. Some implementations average them; the original Word2Vec paper uses only $\mathbf{W}_{in}$. The choice matters for downstream tasks: averaged embeddings slightly improve performance on some benchmarks.
